用户需要根据环境规划，依次登录每台服务器，为服务器配置 hostname。

== 命名规范

服务器名称中应包含 zone id，逻辑机柜号，逻辑机位号。 +
**格式**：zone id + 逻辑机柜号 + 逻辑机位号 +
**示例**：testcloud1r01n00 

具体说明如下：
[cols='1,4'] 
|===
| 命名	|  说明

| testcloud1	| zone id，测试 1 区
| r01	| 代表逻辑机柜号，r01 为第一个 rack，第二个 rack 为 r02，后面的机柜命名依此类推，每个 rack 规划 10 台机器。
| n00	| 代表逻辑机位号，n00 为每个 rack 的第一台机器，第二台为 n01，后面的节点命名依此类推，每个 rack 最大机位号为 n09。
|===

include::../../../../../_components/attention.adoc[]
请务必按照上述命名规范为每台服务器配置 hostname，否则可能导致安装失败。

include::../../../../../_components/table_footer.adoc[]

== 前提条件

已根据环境规划，规划服务器名称。

== 操作步骤

. 依次登录每台物理服务器节点。
. 执行以下命令，修改/etc/hostname中的主机名。
+
[source]
----
hostnamectl set-hostname <hostname>
----
+
或
+
[source]
----
vi /etc/hostname
----
