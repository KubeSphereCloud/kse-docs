---
title: "收集硬件信息"
weight: 30
---

企业云平台部署前，还需要收集已准备好的硬件环境、操作系统环境的各项信息。

需严格检查所有收集到的信息，确保与实际环境一致。

. 登录 FirstBox。
+
[source]
----
ssh root@<firstbox ip address>
----
+
参数说明如下：
+
**<firstbox ip address>**：安装 FirstBox 时配置的管理 IP。
. 执行以下命令，在 FirstBox 的 /root 目录下创建并编辑 ip_list 文件。
+
[source]
----
touch /root/ip_list
vim /root/ip_list
----
+
在 ip_list 文件中写入部署企业云平台的服务器节点的管理网 IP。每一行一个 IP 地址。
+
示例：
+
[source]
----
172.31.60.195
172.31.60.196
172.31.60.197
172.31.60.198 
172.31.60.199
----
. 执行以下命令，将 ip_list 中节点地址添加到 installer 库中。
+
[source]
----
/pitrix/cli/add-nodes.py -I /root/ip_list
----
+
若回显信息类似如下，表示添加成功。
+
image::/images/ep_offline/installation/deploy_collect_server_info_01.png[,80%]
. 执行以下命令，查看新添加节点是否正确。
+
[source]
----
/pitrix/cli/describe-nodes.py -s new
----
+
若回显信息中 **recv** 字段包含了所有已添加的节点（此时节点信息中只包括 node_id 和更新时间），且返回类似如下信息，表示添加正确。
+
[source]
----
  "ret_code": 0,
  "total_count": 5
----
. 配置 FirstBox 与其他物理服务器节点的免密连接。
+

include::../../../../_components/note.adoc[]
添加节点后，系统需要 SSH 连接到 ip_list 中各地址去收集硬件环境信息。

* **若服务器安装操作系统时用的默认用户名密码 yop/zhu1241jie，可以跳过此步骤**。
* 若服务器不是默认的用户名和密码，用户需要手动建立所有服务器节点与 FirstBox 的 SSH 免密访问通道。

include::../../../../_components/table_footer.adoc[]
+
执行以下命令，通过脚本将密钥下发到所有物理节点。
+
[source]
----
/pitrix/node/collect/establish_ssh.sh /root/ip_list [SSH_PORT] [USER] [PASSWD]
----
+
**示例**：
+
[source]
----
/pitrix/node/collect/establish_ssh.sh /root/ip_list 22 yop zhu1241jietest
----
+
**参数说明**：
+
[cols='1,3'] 
|===
| 参数 | 说明 

| **SSH_PORT** | 端口号，默认为 22。

| **USER** +
**PASSWD** 
| 安装操作系统时设置的用户名和密码，例如 yop、zhu1241jietest。
|===
+
回显信息如下，表示操作成功。
+
image::/images/ep_offline/installation/deploy_collect_server_info_02.png[]
+

include::../../../../_components/note.adoc[]
若命令执行失败，请查看日志/pitrix/log/node/establish_ssh.log，检查报错原因并修复。

若仍不能解决，请联系技术支持。

include::../../../../_components/table_footer.adoc[]

. 执行以下命令，格式化待部署节点上用于做数据存储的磁盘。
+
若待收集节点数据磁盘有脏数据，可能会导致收集硬件信息失败，建议收集前格式化一下待部署节点数据磁盘。
+
**若全部是新磁盘，或确定磁盘数据已被清理，可以跳过此步骤。**
+
[source]
----
/pitrix/bin/erase_data_disks.sh -i <hyper_ip> -d <disk_name>
----
+
**示例**：
+
[source]
----
/pitrix/bin/erase_data_disks.sh -i /root/hyper_ip_list -d nvme0n1
----
+
**参数说明**：

 ** **-i 参数**：待格式化的磁盘所在的物理节点 IP。
+
若多个节点的盘符一致，可以创建并编辑 hyper_ip_list 文件。放入待部署节点的 IP，每一行一个 Hyper 节点的 IP 地址。
+
[source]
----
vim /root/hyper_ip_list
----
+
若各个节点的盘符不一致，则只能逐一指定节点 IP 进行格式化。
 ** **-d 参数**：待格式化的磁盘盘符，需根据实际情况进行指定，例如 nvme0n1。
 ** 若需查询该命令其他各参数及说明，可以执行如下命令。
+
[source]
----
/pitrix/bin/erase_data_disks.sh -h
----

+

include::../../../../_components/note.adoc[]
若命令执行失败，请查看日志/pitrix/log/node/erase_data_disks.log，检查报错原因并修复。

若仍不能解决，请联系技术支持。

include::../../../../_components/table_footer.adoc[]

. 执行以下命令，收集硬件信息。
+
[source]
----
/pitrix/cli/collect-nodes.py -I /root/ip_list
----
+

include::../../../../_components/attention.adoc[]
* 安装企业云平台前，该收集命令可以重复执行以修正硬件信息。
* 安装企业云平台后，请勿再执行收集命令。

include::../../../../_components/table_footer.adoc[]
+
回显信息如下，表示收集任务下发成功。
+
image::/images/ep_offline/installation/deploy_collect_server_info_03.png[,80%]

. 执行以下命令，查看收集节点任务当前的状态。
+
[source]
----
/pitrix/cli/get-collect-nodes-status.py
----
+
回显信息如下，状态为 `successful` 时，表示收集任务执行成功。
+
image::/images/ep_offline/installation/deploy_collect_server_info_04.png[,80%]
+

include::../../../../_components/note.adoc[]
若状态为 step0，表示任务正在执行中，请耐心等待，稍后重新执行命令查看状态。

若状态异常，请查看相关日志，检查原因并修复。若仍不能解决，请联系技术支持。

日志路径：

* 节点简略日志：执行命令 `/pitrix/cli/get-collect-nodes-log.py`，获取节点简略日志。
* 节点详细日志文件：/pitrix/log/node/collect.log。

include::../../../../_components/table_footer.adoc[]

. 校验硬件信息。
 ** 执行以下命令，获取所有节点的详细信息。
+
[source]
----
/pitrix/cli/describe-nodes.py
----
** 执行以下命令，获取指定节点的详细信息。
+
[source]
----
/pitrix/cli/describe-nodes.py -i <node_ip>
----
+
检查获取信息中的如下内容：

   *** 是否包含所有待部署节点。
   *** **检查所有节点的硬件信息，包括节点名称、磁盘数量、磁盘大小、磁盘类型、 CPU、内存大小是否与物理配置一致。**
+

include::../../../../_components/note.adoc[]
* 若硬件配置不符合预期，可以对硬件进行调整，调整后请重新执行步骤 6 再次收集节点信息。
* 若收集到了多余的磁盘或其他原因需要调整硬件信息，可以执行命令 ``/pitrix/cli/modify-node-attributes.py`` 对硬件信息进行调整，详细操作请参见link:../../appendix/command[附录 > 常用命令 > 修改硬件信息]。请注意，通过命令修改硬件信息后，若重新执行步骤 6 再次收集节点信息，会覆盖掉通过命令修改的内容。

include::../../../../_components/table_footer.adoc[]

   *** 执行以下命令，校验 hostname。
+
[source]
----
/pitrix/cli/describe-nodes.py | grep hostname
----
