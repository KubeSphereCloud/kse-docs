该文件用于配置企业云平台基本信息、网络信息和需要部署的服务信息。

== 操作步骤

. 登录 FirstBox。
+
[source]
----
ssh root@<firstbox ip address>
----
+
参数说明如下：
+
**<firstbox ip address>**：安装 FirstBox 时配置的管理 IP。
. 执行以下命令，从模版创建配置文件，文件名为variables.ZONE_ID.yaml。
+
[source]
----
cp -f /pitrix/config/templates/variables.template.yaml /pitrix/config/variables.ZONE_ID.yaml
----
示例：
+
[source]
----
cp -f /pitrix/config/templates/variables.template.yaml /pitrix/config/variables.testcloud1.yaml
----
+

include::../../../../../_components/note.adoc[]
文件名中的 ZONE_ID 为可用区 ID，请根据环境规划中的信息进行替换。

示例中以 variables.testcloud1.yaml 为例（即 ZONE_ID 为 testcloud1）。

若有多个 ZONE，需要对应创建多个配置文件。

include::../../../../../_components/table_footer.adoc[]

. 执行以下命令，修改配置文件variables.ZONE_ID.yaml。
+
[source]
----
vi /pitrix/config/variables.ZONE_ID.yaml
----
+
示例：
+
[source]
----
vi /pitrix/config/variables.testcloud1.yaml
----
. 按照参数说明，按需为待部署的 ZONE 填入相关信息。
+
variables.ZONE_ID.yaml 中包括如下配置：

 ** **common 配置（必选）**：配置企业云平台基础信息。
 ** **base 配置（必选）**：配置企业云平台安装模式。
 ** **高级配置**
  *** 环境配置（必选）：多 ZONE 部署时需要配置。
  *** 网络配置（必选）：若用户需要系统自动控制分配管理虚拟机地址、floating ip 等信息时，需要配置。
  *** NTP&DNS：配置 NTP 服务器和 DNS 服务器。
  *** 可选服务/功能配置（可选）：根据实际需求进行配置。默认安装 BOSS 和 CloudSat。
包括：IPv6、BM、VDI（云桌面）、BOSS、Warehouse、Eipctl、Rsserver、Cronus、Influxdb、IAM（统一身份认证）、HPC（超性能计算）、API 网关（API Gateway）、QKE（容器平台）、SMC（迁移中心）、Cloudsat（云监控）、NeonSAN、对象存储。
 ** **服务配置（可选）**：若用户需要手动控制分配管理虚拟机地址、floating ip 等信息时，需要配置。

== common 配置

**配置示例**

[source]
----
common:
  ssh_port: 22
  firstbox_address: ''
  cloud_type: 'private'
  domain: 'democloud.com'
  region_id: 'testcloud'
  console_id: 'democloud'
  console_name: 'democloud'
  site_protocol: 'http'
  is_poc: 0
----

**参数说明**

[cols='1,3']
|===
| 参数 | 说明

| ssh_port | SSH 使用的端口，默认为 22。无需修改。
| firstbox_address | FirstBox 节点 IP，系统自动获取，不需要手动填写。
| cloud_type | 云类型，支持public、private、express 三种类型，默认为 private。 +
私有云部署时请选择private类型。
| domain | 域名，用于访问 Console 等 Web 环境。请根据环境规划进行配置。
| region_id  a| 区域 ID，资源的区域。请根据环境规划进行配置。

include::../../../../../_components/note_in_table.adoc[]
* 区域通过地理位置划分，不同区域间内网隔离，不能互相访问。
* 一个区域可以包括多个可用区（可通过 zone_id 和 zone_name 进行配置），每个可用区都相互独立但网络互通，同一可用区内的主机网络延时更小。

include::../../../../../_components/footer_in_table.adoc[]

| console_id | Console ID，Console 的唯一标识。请根据环境规划进行配置。
| console_name | Console 名称。请根据环境规划进行配置。
| site_protocol a| Web 使用的网络协议，支持 http、https，默认为 http。请根据环境规划进行配置。

include::../../../../../_components/note_in_table.adoc[]
如果使用 https 协议，还需要上传 SSL 证书。详细操作请参见link:../../../appendix/https[配置 HTTPS 支持]。

include::../../../../../_components/footer_in_table.adoc[]

| is_poc | 是否用于 POC，1 表示是，0 表示否。默认为 0。 +
配置为 1，可以获得更高的性能。
|===

== base 配置

**配置示例**

[source]
----
base:
    deploy_mode: 'standard'
----

**参数说明**

**deploy_mode**：安装模式，支持 standard、fusion 模式，默认为 standard。

* standard：标准部署。KS 节点与 Hyper 节点分别独立部署。资源足够时，建议选择标准部署模式。
* fusion：融合部署。KS 节点与 Hyper 节点部署在同一物理服务器上。

== 高级配置

=== 环境配置

请根据环境规划进行如下配置。

**配置示例**：

[source]
----
hyper_mtu: 1500
zone_name: '测试一区'
zone_code: 1
is_global_zone: 1
is_sub_zone: 1
is_vlan_mode: 0
sdn_version: 2
----

**参数说明**：

[cols='1,3']
|===
| 参数 | 说明

| hyper_mtu | 计算节点 mtu，默认为 1500。若要在虚拟机中运行隧道或使用 h3cvg，需要设置为 1550。
| zone_name | 可用区中文名称。请根据环境规划进行配置。
| zone_code | 多 ZONE 环境时配置。取值范围为 1~999，默认为 1。
| is_global_zone a| 是否部署为主 ZONE，默认为 1。 +
如果只有 1 个 ZONE，is_global_zone 和 is_sub_zone 保持默认值即可。多 ZONE 部署时需要有一个主 ZONE。

* 如果是主 ZONE，此参数配置为 1。
* 如果是子 ZONE ，此参数配置为 0。
| is_sub_zone a| 是否部署为子 ZONE，默认为 1。

* 如果是主 ZONE ，此参数可以配置为 1。即主 ZONE 也可以作为子 ZONE。
* 如果是子 ZONE ，此参数必须配置为 1。
| is_vlan_mode | 是否配置 VLAN 模式，默认为 0，不配置 VLAN。如果使用 VLAN 模式，此参数需要配置为 1。
| sdn_version | SDN 版本，默认为 2，使用 SDN2.0 版本。支持 SDN1.0、SDN2.0 版本。 +
如果 SDN 模式使用 SDN2.0 版本，需要配置为 2。
|===

=== 网络配置

若用户需要手动控制分配管理虚拟机地址、floating ip 等信息。请根据环境规划进行网络配置。

* 若用户希望系统**自动**为各服务管理角色分配IP地址，请配置带 pool 的参数，系统将根据 pool 地址池，自动分配 IP 地址。
+

include::../../../../../_components/attention.adoc[]
* 所有 pools 配置时需配置掩码，例如 10.10.0.1/24。
* 所有 pools 参数的类型为字符串，参数值需要加引号。

include::../../../../../_components/table_footer.adoc[]

* 若用户需要**手动**为各服务管理角色分配IP地址，无需配置如下带 pool 的参数，请执行<<_服务配置>>，手动为各服务管理角色配置 mgmt_network_address。
+

include::../../../../../_components/note.adoc[]
* 若为融合模式（fusion），仅需指定 zoocassa 角色的相关配置即可。
* 如果同时配置了 pool 参数和各服务管理角色的 mgmt_network_address，以 mgmt_network_address 配置为准。

include::../../../../../_components/table_footer.adoc[]

**配置示例-生产环境**：

mgmt_network_pools: ['10.10.0.1/24-10.10.0.253/24']
vm_mgmt_network_pools: ['10.10.0.211/24-10.10.0.229/24']
ks_floating_ip_pools: ['10.10.0.231/24-10.10.0.239/24']

vm_base_network_start: '10.20.1.0/24'
vm_private_networks:
- '172.16.0.0/16'
- '172.18.0.0/16'
network_separated: 0
user_networks: []
user_network_interface: ''
----

include::../../../../../_components/note.adoc[]
* mgmt_network_pools、vm_mgmt_network_pools、ks_floating_ip_pools 配置说明：
 ** 这三个地址池配置完成后，无需进行<<_服务配置>>。
 ** 在生产环境中，vm_mgmt_network_pools、ks_floating_ip_pools 请配置为管理网络mgmt_network_pools的一个子网段。

* vm_base_network、vm_private_networks 无特殊需求使用默认值即可。
* mgmt_network_pools（管理网络）、vm_base_network（基础网络）、vm_private_networks（VPC 私有网络）、eip_network_pools（EIP 网络）这四个地址池请勿重叠。

=== 服务开关配置

该模块为 IPv6、BM（弹性裸金属服务器）、VDI（云桌面）、BOSS（云计算运维管理平台）、Warehouse（计费分析服务）、Eipctl、Rsserver、Cronus、Influxdb、IAM（统一身份认证）、HPC（超性能计算）、API网关（API Gateway）、QKE（容器平台）、SMC（迁移中心）、Cloudsat（云监控）、NeonSAN、对象存储等服务组件的开关。

include::../../../../../_components/attention.adoc[]
默认安装 BOSS、CloudSat，其他功能默认不安装。请根据实际情况进行配置。

* 若不需要安装相关组件，请配置对应参数为 0。

**参数说明**：

[cols='1,2']
|===
| 参数名称 | 参数说明

| dnsmaster_floating_ip
| 手动配置 Dnsmaster 服务浮动IP地址。

dnsmaster:

    - '10.10.0.9'
