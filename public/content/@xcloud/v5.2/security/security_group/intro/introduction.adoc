通过策略配置，安全组能极大地提高一个内部网络或云服务器的安全性，并过滤不安全的服务，从而降低内部网络或云服务器的风险。

* 云平台系统为每个用户提供了一个缺省安全组(ID 之后带有星标)，默认打开 22 端口。
* 初始状态下，每个安全组都不包含任何规则，即任何端口都是封闭的，您需要建立规则以打开相应的端口。
* 添加安全组规则时，您必须按照要求指定通信端口或端口范围，之后安全组根据策略决定是否转发业务数据到云服务器。
* 您可以借助 **IP/端口集合**功能把具有相同特征的一组 IP 或者一组端口设置成为 **IP/端口集合**，并且在安全组规则中进行添加，实现批量管理功能。
* 安全组可以绑定的实例类型有：云服务器、VPC 网络、负载均衡器、NAT 网关、AppCenter 应用集群。
