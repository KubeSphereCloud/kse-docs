---
title: "创建转发策略"
weight: 2
---

. 登录{product_console_name}。
. 在顶部导航栏，选择**资源** > **网络** > **负载均衡器**，选择**转发策略**页签，进入**转发策略**列表页面。该页面显示当前平台内已有的转发策略的``名称/ID``，``优先级``、``匹配方式``以及``创建时间``等基本信息。
+
image::/images/express/console/profile_create_1.png[]

. 点击**创建转发策略**，在弹出的**创建转发策略**窗口中，输入各项参数，点击**创建**。
+
image::/images/express/console/profile_create_2.png[,60%]
+
参数说明
+
[options="header",cols="1,3a"]
|===
|参数|详细说明
|名称
|用户自定义，便于用户浏览与搜索。

|匹配方式
|支持选择匹配所有规则或匹配任意规则。

* 匹配所有规则：匹配所有规则的请求才会被转发，即该策略下，规则之间为与/AND 关系。
* 匹配任意规则：匹配任意一条转发规则的请求都将被转发，即该策略下，规则之间为或/OR 关系。

|优先级
|转发策略优先级。 +
取值范围：0 ~ 99。值越小，优先级越高。同等优先级下，转发策略会随机使用。多条策略绑定到后端时，先适用优先级高的。

|健康检查
|设置后端服务器的健康检查方式及健康检查选项。支持HTTP 或TCP。

|设置健康检查方式
|* 检查间隔，范围 2 ～ 60 秒；
* 超时时间，范围 5 ～ 300 秒；
* 不健康阈值，范围 2 ～ 10 次；
* 健康阈值，范围 2 ～ 10 次。
|===

. 返回转发策略列表，新创建的转发策略已显示在列，点击其名称，进入相应的详情页面。
+
image::/images/express/console/profile_create_3.png[]

. 在转发策略的详情页面，选择**规则**页签，点击**添加**。
+
image::/images/express/console/profile_create_4.png[]
. 在弹出的**添加转发规则**的窗口中，输入名称、规则类型、规则内容相关参数，点击**添加**即可。

+

include::../../../../../../../_components/note.adoc[]
平台支持按域名转发、按 URL 转发、按源地址转发和按 HTTP Header 转发四种规则类型。

include::../../../../../../../_components/table_footer.adoc[]

+
image::/images/express/console/profile_create_5.png[,60%]
+
[options="header",cols="1,2,2a,1"]
|===
|规则类型|适用场景|规则内容说明|取值示例
|按域名转发
|为多个域名提供负载均衡服务，后端服务器根据域名分组管理。
|* 支持精确域名及泛域名。
* 取值为正则表达式。
* 多个域名之间通过空格分割，表示匹配任意一条即可。
|www.example.com 或 ^example.com$

|按 URL 转发
|统一服务入口下，通过不同的 URL 区分不同的子服务，后端服务器根据 URL 分组管理。
|* 取值为正则表达式。
* 多个 URL 可通过空格分割，表示匹配任意一条即可。
| ^/static 或 .css$

|按源地址转发
|将重要/特殊客户网段的请求分配给专用服务器。
|支持 IP 地址或网段。
|1.1.1.1 或 1.1.1.0/24

|按 HTTP Header 转发
|针对不同的浏览器或不同终端提供差异化的服务
|* 取值为正则表达式。
* 多个值可通过空格分割，表示匹配任意一条即可。
|无
|===


. （可选）转发规则创建完成后，规则页签内出现**重写规则**列表，点击重写规则下的**添加**。在弹出添加重写规则窗口中，填写各项参数，点击**添加**即可。
+
image::/images/express/console/profile_create_6.png[,60%]
+
参数说明
+
[options="header",cols="1,2a"]
|===
|参数|说明
|名称
|用户自定义，便于搜索与浏览。

|规则类型
|目前仅支持按照路径重写。

|规则内容
| 重写规则固定表达式：`%[path,regsub(x,y,i)]`
规则说明：

* 除 x, y 可修改外，其余部分不能修改。
* 表达式中不能含有空格，x, y 的取值中也不能有空格。
* path 为固定值，不可修改，%[path]表示获取请求的路径。
* regsub(x,y,i) 表示对求值结果做转换，将路径中的第一个匹配到 x 替换成 y，i 表示不区分大小写。

include::../../../../../../../_components/note_in_table.adoc[]
\http://myhomer.vip/test 希望经过 URL 重写后，可以变成 \http://myhomer.vip/tset， +
则可以添加重写规则：%[path,regsub(test,tset,i)]。

include::../../../../../../../_components/footer_in_table.adoc[]

|优先级
|决定规则执行顺序，取值范围 0 ~ 100，数值越小优先级越高。优先级相同，则按照规则添加的顺序依次执行。
|===

. 转发规则和重写规则添加完成之后，转发策略创建成功。
