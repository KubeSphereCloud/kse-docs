== 创建流程

创建可用的负载均衡器基本流程为：**创建负载均衡器** > **创建监听器** > **添加后端服务器**。


== 步骤 1：创建负载均衡器

. 登录{product_console_name}。
. 在顶部导航栏，选择**资源** > **网络** > **负载均衡器**，默认进入**负载均衡器**列表页面。
+
image::/images/express/console/lb_create_1.png[]

. 点击**创建负载均衡器**，在弹出的**创建负载均衡器**的窗口中，根据需求输入各项参数，点击创建即可。

+
image::/images/express/console/lb_create_2.png[,60%]

+
参数说明
+
[options="header",cols="1,3a"]
|===
|参数	|详细说明
|名称
|用户自定义，便于用户浏览与搜索。

|最大连接数
|并发规格，支持选择 5000、200000、40000 以及 100000 的最大连接数。

include::../../../../../../../_components/note_in_table.adoc[]
此处的并发规格适用于非 HTTPS 场景的监听器。对于 HTTPS 监听器，由于需要在负载均衡器上做证书卸载，耗费更多的 CPU，因此 HTTPS 监听器的并发规格为负载均衡器规格的一半。

include::../../../../../../../_components/footer_in_table.adoc[]

|节点数
|至少需要 2 个节点。

|基础网络
|需选择目前平台内已创建完成的基础网络。
|===

. 返回列表页面，新创建的负载均衡器已显示在列，状态为``活跃``，表示已创建成功。


== 步骤 2：创建监听器

负载均衡器创建后，需为其配置监听器。监听器负责监听负载均衡器上的请求，并根据负载均衡策略分发流量到后端服务器。

=== 操作步骤

. 进入**负载均衡器**列表页面。点击指定负载均衡器的名称，进入其详情页面。
+
image::/images/express/console/lb_create_3.png[]

. 选择**监听器**页签，点击**创建监听器**。在弹出的**创建监听器**的窗口中输入相应参数，点击**创建**即可。根据不同的监听协议，其参数配置不尽相同，详见link:#_参数说明[参数说明]。

+
image::/images/express/console/lb_create_4.png[,60%]

. 监听创建好后，点击**应用修改**，更新负载均衡器，使其生效。

.7+|高级设置

|选择禁用或启用。

|健康检查

* 负载均衡器监听协议：通过 X-Forwarded-Proto 头字段获取负载均衡器的监听协议。

|超时时间
|单位：秒，支持范围 10 ～ 86400 秒。

|隧道超时时间
|单位：秒，支持范围 10 ～ 86400 秒。

|附加选项
|*	取消 URL 校验：让负载均衡器接受不符合编码规范的 URL，例如包含未编码中文符的 URL 等。
* 获取客户端 IP：将客户端 IP 直接传送给私有网络中的后段，启用后，负载均衡器对后段完全透明。
+
开启此项，负载均衡器对于后端完全透明，后端云服务器 TCP 连接得到的源地址是客户端的 IP，而不是负载均衡器的 IP。开启透明代理会导致内网请求失败，请在只提供公网服务（绑定了公网 IP）时开启。

* 数据压缩：使用 gzip 算法压缩文本数据，以减少网络流量。
* 启用 HTTP 重定向：将此监听器的 HTTP 请求重定向至 HTTPS 监听器。**启用后，此监听器不支持添加后端服务器。**
* 禁用 HTTP 头字段Proxy：安全起见，某些后端为 CGI 的 Web 应用需要在用户请求中禁用该字段来防止注入攻击。

== 步骤 3：添加后端

创建监听器后，用户需要添加云服务器作为后端服务器。监听器使用用户已配置的协议和端口检查来自客户端的连接请求，并根据用户定义的分配策略将请求转发到后端服务器，由后端服务器进行处理。

. 在指定负载均衡器的详情页面，**监听器创建成功**后，点击**添加后端**。
+
image::/images/express/console/lb_create_5.png[]

. 在弹出的**添加后端**窗口中输入相应参数，点击**添加**即可。
+
image::/images/express/console/lb_create_6.png[,50%]
+
参数说明
+
[options="header",cols="1,1a,3a"]
|===
|配置项	|参数	|详细说明
.5+|基础设置
|名称
|用户自定义，便于用户浏览与搜索。

|基础网络
|选择当前平台内已存在的基础网络。

|主机
|已选定的基础网络中，已绑定的主机。

|端口
|支持范围 20 ~ 65535。

|后端主备模式
|后端主机的工作模式。

* 工作后端：正常提供服务的后端。
* 备用后端：正常情况下，不提供服务。当所有工作后端都处于非健康状态时，备用后端提供服务。

.2+|高级设置
|转发策略
|选择平台内已创建的转发策略。

|权重
|数值越大，权重越高。
|===

. 点击**应用修改**，更新负载均衡器配置。
