== 场景描述

创建转发策略之后，用户需要将相应的转发策略与负载均衡器的后端服务器进行绑定，才能使得匹配规则的请求转发到所绑定的后端服务器。

== 前提条件

* 已创建 HTTP 或 HTTPS 监听协议的监听器。
* 已经创建完成转发策略。

== 操作步骤

. 登录{product_console_name_en}。
. 在顶部导航栏，选择**资源** > **网络** > **负载均衡器**，进入**负载均衡器**列表页面。点击指定的负载均衡器名称，进入其相应的详情页面，选择**监听器**页签。

. 选择已创建完成的监听协议为 HTTP 或 HTTPS 的监听器，对其**添加后端**，在已经添加完成的后端服务器列表中，点击**转发策略**下的绑定标识。
+
image::/images/express/console/profile_bind_1.png[]
 
. 在弹出的**选择一个转发策略**的窗口中，勾选待绑定的转发策略，点击**确定**即可。
+
image::/images/express/console/profile_bind_2.png[,60%]

. 绑定成功后，点击负载均衡器详情页面的**应有修改**，使其生效即可。
