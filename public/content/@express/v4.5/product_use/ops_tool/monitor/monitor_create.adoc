== 功能描述

监控告警功能是基于虚拟资源层面的监控数据，针对平台中的时序数据进行监控，例如虚拟机的内存利用率、CPU 利用率等。平台目前支持的监控对象为虚拟机，支持用户自定义告警规则和监控周期，监控告警有助于用户及时了解资源使用情况和处理突发事件。


== 操作步骤

. 登录{product_console_name_en}。
. 在顶部导航栏，选择**运维工具** > *监控告警*，进入**监控告警**列表页面。
+
image::/images/express/console/monitor_create_1.png[create_monitor_1]

. 点击**创建监控告警**，在弹出的创建监控告警窗口中的**设置告警参数**页面，设置告警对象、监控周期以及告警规则，点击**下一步**。
+

include::../../../../../_components/note.adoc[]
云平台支持设置监控对象**满足任何一条告警规则便触发告警消息**或者**满足全部告警规则才触发告警信息**。具体方法可参考link:../monitor_condition_modify[设置告警状态触发条件]相关内容。

include::../../../../../_components/table_footer.adoc[]
+
image::/images/express/console/monitor_create_2.png[create_monitor_2]
+
*参数说明*
+
[options="header",cols="1,3"]
|===
|参数|说明
|告警对象
|目前仅支持对主机进行告警。

|监控周期
|针对各项告警规则的监控周期，支持 1 分钟或 5 分钟。

|设置告警规则
|支持设置针对 CPU 利用率、内存利用率、磁盘使用量、硬盘 INONE、内网出流量、内网进流量的告警规则。
|===

. 在创建监控告警窗口中的**设置告警方式**页面，设置各项参数。
+
image::/images/express/console/monitor_create_3.png[create_monitor_3]
+
*参数说明*
+
[options="header",cols="1,3"]
|===
|参数|说明
|告警名称
|用户自定义，便于搜索与浏览。

|发送通知
|是否发送告警通知，若选择是，则需配置发送条件、发送通知次数以及通知列表相关参数。

|满足以下条件发送通知
|可单独或全部勾选``资源发生警告时``、``资源恢复正常时``两种条件下发送告警通知。

|持续告警时，连续发送通知次数
|支持设置 1 ～ 5 次。

|通知列表
|接收告警通知的邮箱列表。若无可选列表，需点击右侧创建通知列表进行创建。
|===

. 在创建监控告警窗口中的**设置告警方式**页面，点击**通知列表**右侧的**创建通知列表**，在弹出的**创建通知列表窗口**中，设置各项参数后点击**提交**即可。
+

include::../../../../../_components/note.adoc[]
* 可单独或同时选择邮件通知或 Webhook 通知。
* 邮件通知需对邮箱进行验证。点击邮箱地址右侧的确认符号，进行邮箱验证。
* Webhook 地址填写接收端的 URL 地址，需输入以 http:// 或 https:// 开头的 URL。

include::../../../../../_components/table_footer.adoc[]

+
image::/images/express/console/monitor_create_4.png[create_monitor_3]

. 返回**创建监控告警**页面，点击**创建**即可。
