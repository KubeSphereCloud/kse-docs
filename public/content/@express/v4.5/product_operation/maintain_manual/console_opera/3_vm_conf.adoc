== 运维场景

在云易捷 Console 控制平台使用过程中，若虚拟机创建时所支持的最大 CPU 和内存等配置无法满足实际生产需求时，可参考以下步骤手动进行修改。


== 注意事项

虚拟机的 CPU 或内存最大配置，不能超过物理服务器实际配置的 1/3。

== 操作步骤

. 登录至当前环境的 FirstBox 节点。
+
[source,shell]
----
ssh yop@xx.xx.xx.xx
----
+
include::../../../../../_components/note.adoc[]
* xx.xx.xx.xx 为环境中 FirstBox 节点 IP，需根据实际情况进行修改。
* 用户名密码：yop/zhu1241jie。
include::../../../../../_components/table_footer.adoc[]

. 执行如下命令，输入当前主机密码，切换至 root 用户。
+
[source,shell]
----
sudo -i
----

. 修改 ``server.yaml`` 配置文件中 CPU 与内存的对应关系，如下所示，增加 CPU 和内存上限即可。
+
[source,shell]
----
vi /pitrix/conf/variables/global/server.yaml.<regin_id>
----
+
[source,yaml]
----
... ...
      valid_cpu_cores: '1, 2, 4, 8, 12, 16, 24, 32'
      valid_cpu_memory_pairs:
        !!int 1: '512, 1024, 2048, 4096, 8192, 12288, 16384, 24576, 32768, 40960,
          49152, 65536'
        !!int 2: '512, 1024, 2048, 4096, 6144, 8192, 12288, 16384, 24576, 32768, 40960,
          49152, 65536'
        !!int 4: '512, 1024, 2048, 4096, 6144, 8192, 12288, 16384, 24576, 32768, 40960,
          49152, 65536'
        !!int 8: '512, 1024, 2048, 4096, 6144, 8192, 12288, 16384, 24576, 32768, 40960,
          49152, 65536'
        !!int 12: '512, 1024, 2048, 4096, 6144, 8192, 12288, 16384, 24576, 32768,
          40960, 49152, 65536'
        !!int 16: '512, 1024, 2048, 4096, 6144, 8192, 12288, 16384, 24576, 32768,
          40960, 49152, 65536'
        !!int 20: '512, 1024, 2048, 4096, 6144, 8192, 12288, 16384, 24576, 32768,
          40960, 49152, 65536'
        !!int 24: '512, 1024, 2048, 4096, 6144, 8192, 12288, 16384, 24576, 32768,
          40960, 49152, 65536'
      valid_memory_size: '512, 1024, 2048, 4096, 6144, 8192, 12288, 16384, 24576,
        32768, 40960, 49152, 65536'
... ...
----
+
include::../../../../../_components/note.adoc[]
<regin_id> 需根据实际情况进行修改。
include::../../../../../_components/table_footer.adoc[]

. 执行如下命令，查看集群中的 KS 节点信息。
+
[source,shell]
----
cat /etc/hosts | grep ks
----

. 逐一登录 <zone_id>-ks0 和 <zone_id>-ks1 节点，修改前端配置文件。

.. 进入配置文件所在目录。
+
[source,shell]
----
cd /pitrix/lib/pitrix-webexpress/server/conf/
----

.. 查看并复制 ``config.yaml`` 配置文件中的以下内容。
+
[source,shell]
----
cat config.yaml
----
+
[source,yaml]
----
......
  cpu_memory_map:
    1: [1, 2, 4]
    2: [2, 4, 6, 8, 12, 16]
    4: [4, 6, 8, 12, 16, 32]
    8: [8, 12, 16, 24, 32, 64]
    12: [12, 24, 48]
    16: [16, 32, 64]
    32: [16, 32, 64, 128]
    64: [16, 32, 64, 128]
    128: [32, 64, 128, 256]
......
----

.. 将 ``config.yaml`` 复制的内容，添加至 ``loal_config.yaml`` 配置文件中，需要根据实际需要进行修改，删除不需要的 ``cpu_memory_map`` 配置。
+
[source,shell]
----
vi local_config.yaml

----
+
[source,yaml]
----
......
  cpu_memory_map:
    1: [1, 2, 4]
    2: [2, 4, 6, 8, 12, 16]
    4: [4, 6, 8, 12, 16, 32]
    8: [8, 12, 16, 24, 32, 64]
    12: [12, 24, 48]
    16: [16, 32, 64]
......
----

.. 依次执行如下命令，重启 express 和 apache 使配置生效。
+
[source,shell]
----
pm2 restart express
----
+
[source,shell]
----
service httpd restart
----
