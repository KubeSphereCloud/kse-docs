== 集群 IP 数量及占用说明

* 物理节点 N 个即需要 N 个 IP 地址。
* FirstBox 节点：即集群维护节点1 个。
* 集群管理节点（ks0、ks1、ks2）：各 1 个 IP，共 3 个。
* BOSS 服务：1 个。
* Zookeeper 和 Cassandra 服务 1 个。
* Webservice 服务 1 个。
* 管理服务数据库 1 个。
* DNS 管理服务 1 个。
* Console 平台访问代理和 API server 调用：1 个。
* 可预留 10 个 IP 地址，作为 v2v、数据管理平台、纳管 VMware 等部署产品组件需要。

== 服务器名称规划

在云易捷安装部属过程中，各个节点服务器的命名规范如下：

* **命名规则**：区域 ID + 逻辑机柜号 + 逻辑机位号，如 ``bjedu1r01n00``。
* 不建议采用大写字母， -、_ 、等特殊字符。

**名称示例说明**：

* bjedu1：为区域 ID，表示北京培训 1 区。
* r01：逻辑机柜号，r01 表示为第一个 rack，第二个 rack 为 r02，后面的机柜命名以此类推，其中每个 rack 中规划 10 台机器。
* n00：逻辑机位号，n00 表示每一个 rack 中的第一台机器，第二台机器名为 n01，后面节点的命名方式以此类推，其中每个 rack 的最大机位号为 n09。

== 服务器 IP 规划示例

云易捷的安装部署过程中，网卡一般使用两个双网口的万兆网卡，上下排列。

* 第一个网卡的双网口做 Bond 作为管理/业务网络端口，以 TRUNK 的方式连接到交换机上。
* 第二个网卡的两个网口作为存储网络，以 ACCESS 方式分别连接在两个交换机上。
