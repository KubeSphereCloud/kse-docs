title: "Configure External Persistent Storage"
weight: 04
---


This section explains how to configure an external persistent storage system for the KubeSphere Enterprise cluster in a production environment. A persistent storage system is used to create volumes for storing application data on the KubeSphere Enterprise platform. If an external persistent storage system is not configured, KubeSphere Enterprise will default to using the local storage system of the cluster nodes. If your KubeSphere Enterprise cluster does not require external persistent storage, you can skip this section.

// Note
include::../../../../../_ks_components-en/admonitions/note.adoc[]

The configuration of external persistent storage for KubeSphere Enterprise is only supported in scenarios where Kubernetes and KubeSphere Enterprise are installed together. If you install KubeSphere Enterprise on an existing Kubernetes cluster, KubeSphere Enterprise will use the existing persistent storage configuration of the Kubernetes cluster.
