资源是{ks_product_abbr_full_both}集群运行工作负载所需的实体对象。每执行一次创建操作，就可能产生对应的资源，如节点、虚拟机、网络等。在{ks_product_both}中，资源主要分为物理资源和虚拟资源两种类型。物理资源是指从物理维度上定义的 IT 基础设施设备，如服务器节点、物理网络和存储设备等。虚拟资源是逻辑层面的资源，是经过虚拟化技术改造的、用户使用方便的资源，如 VPC 网络、虚拟机、磁盘、快照等。

本文介绍{ks_product_both}集群中的各类资源类型。

[discrete]
endif::[]
== 物理资源link:[id=c69404ad5b24466cb4cba1f3f264e468]

下表列举了使用{ks_product_both}过程中涉及的常见物理资源类型。

[cols="1a,3a"]
|===
|资源 |描述

|物理节点
|{ks_product_right}集群中的基础服务器，在节点上可以创建虚拟机。{ks_product_right}提供以下两种节点类型：

* 控制节点：控制平面节点，专用于管理和控制集群中的任务流。

* 工作节点：用于运行工作负载的节点。

|物理 CPU
|物理节点上实际插入的 CPU 数量。

|CPU 核数
|一块物理 CPU 上能处理数据的芯片组数量。

|物理内存
|物理节点上物理内存条提供的内存空间。

|物理磁盘
|集群存储设备上的磁盘。

|物理子网
|物理网络的单位，一个物理网络可划分为若干个子网。
|===

== 虚拟资源link:[id=50467fdb8e2b4f5ba10acfdc1c556235]

下表列举了使用{ks_product_both}过程中涉及的常见虚拟资源类型。

[cols="1a,3a"]
|===
|资源 |描述

|项目
|用于对资源进行分组管理。您可以将用户分配到项目中，并指定用户的角色以控制用户在项目中的权限。

|虚拟机
|虚拟机创建在{ks_product_both}集群节点上，可以用于运行工作负载。

|虚拟机模板
|虚拟机的一种主副本，包含了虚拟机的具体配置。

|镜像
|虚拟机的一种标准交付物，用于打包虚拟机相关配置及其依赖的环境。

|磁盘
|为虚拟机提供的块级别的虚拟存储资源，以文件夹的形式存在于宿主机上。

|弹性 IP
|提供独立的公网 IP 资源，包括公网 IP 地址。

|VPC 网络 
|云上私有网络，不同的 VPC 之间逻辑上完全隔离。

|虚拟子网
|VPC 内的 IP 地址范围。

|SSH 密钥
|通过加密算法生成的一对密钥，由公钥和私钥组成。

|安全组
|一种虚拟防火墙，用于控制与其关联资源的入流量和出流量。
|===
