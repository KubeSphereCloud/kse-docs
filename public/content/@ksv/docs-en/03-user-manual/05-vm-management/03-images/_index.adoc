Images are a standard delivery format of VMs. The configuration and dependencies of VMs can be encapsulated into images.  Different images can contain different OSs, such as Fedora, CentOS, and Ubuntu. {ks_product_abbr_full} provides the following types of images: system images, private images, and shared images. The following table compares the images of different types.

[cols='1a,3a']
|===
|Type |Description

|System image
|The official image provided by {ks_product} or the image uploaded by user **admin**.  Other users can use system images to create VMs.

|Private image
|The image created by replicating an existing image or uploading from a local environment.  You can use, share, replicate, or delete a private image only if the image is created on your own.

|Shared image
|The image that can be used by other users.
|===
