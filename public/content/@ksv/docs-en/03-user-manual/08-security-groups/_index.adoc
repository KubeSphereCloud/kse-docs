Security groups are virtual firewalls that are used to control inbound and outbound traffic of associated resources. This secures the resources. Each VM must be added to at least one security group.

You can add traffic rules for each security group. The rules for inbound and outbound traffic are separate.

The **Security Groups** module is available to all users.  User **admin** can view and manage all security groups in all projects. The following table compares the permissions granted to non-admin users of different project roles.

[cols='1a,3a']
|===
|Project role |Permission

|Administrator
|View and manage all security groups in the projects to which they belong.

|Operator
|View and manage all security groups in the projects to which they belong.

|Viewer
|View only all security groups in the projects to which they belong.
|===
