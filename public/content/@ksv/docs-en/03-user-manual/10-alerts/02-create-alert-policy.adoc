== Procedurelink:[id=6f969d23e8514f209560b5e15132a6b3]

include::../../../_custom-en/alerts/alert-steps.adoc[]

. On the **Alert Policies** tab, click **Create** in the upper-right corner.
. In the dialog box that appears, configure basic information for the alert policy.
+
--
[cols='1a,4a']
|===
|Parameter |Description

|Name
|The custom name of the alert policy.

include::../../../_custom-en/security-groups/naming-convention.adoc[]

|Alias
|The alias of the alert policy.

include::../../../_custom-en/alerts/alias-naming-convention.adoc[]

|Duration (mins)
|The duration of the alert.

An alert is fired if the duration has been reached.

|Severity
|The severity of the alert. Valid values:

include::../../../_custom-en/alerts/alert-severity-levels.adoc[]

|Description
|The description of the alert policy.

include::../../../_custom-en/networks/description-convention.adoc[]
|===
--
. Click **Next** and configure the alert rule for the policy.
+
--
[cols='1a,3a']
|===
|Parameter |Description

|Target Type
|The type of targets for monitoring.  {ks_product} can monitor nodes and VMs.

|Alert Rule
|The alert rule of the alert policy. The supported alert rules vary based on the target type.

* For nodes, you can configure a rule based on the following metrics: CPU usage, available memory, memory usage, available local disk size, local disk size, local disk reads, local disk writes, disk read throughput, and disk write throughput.
* For VMs, you can configure a rule based on the following metrics: CPU usage, memory usage, local disk reads, local disk writes, outbound rate, and inbound rate.

|Monitoring Target
|The name of the node or VM for monitoring.  You can monitor one or more targets.
|===
--
. Click **OK**.  The alert policy appears in the alert policy list after it is created.
