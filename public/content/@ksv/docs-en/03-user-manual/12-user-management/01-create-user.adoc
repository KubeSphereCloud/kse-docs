== Procedurelink:[id=c1e6567eea124f888a17ceb2895b8edf]

include::../../../_custom-en/users/user-steps.adoc[]

. On the ** Local Users** page, click **Create** in the upper-right corner.
. In the dialog box that appears, configure the user information.
+
--
[cols='1a,3a']
|===
|Parameter |Description

|Username
|The name of the user.

include::../../../_custom-en/security-groups/naming-convention.adoc[]

|Email
|The email address of the user.

The email address can be used to log in to the {ks_product} web console.

|Password
|The password of the user.

include::../../../_custom-en/users/password-rule.adoc[]

|Confirm Password
|Enter your password again to confirm.

|Project
|The project to which the user belongs.

If you do not specify a project when you create the user, the user is not allowed to view or perform operations on any resource.  You can add the user to a project after it is created. For more information, see link:../../04-projects-management/05-add-project-member/[Add a member to a project] and link:../../04-projects-management/07-remove-project-member/[Remove a member from a project].

|Role
|The role of the user in the project.

|Description
|The description of the user.

include::../../../_custom-en/networks/description-convention.adoc[]
|===
--
. Click **OK**.  The user appears in the user list after it is created.
