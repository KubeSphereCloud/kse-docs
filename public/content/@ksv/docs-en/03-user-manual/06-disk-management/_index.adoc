In {ks_product_abbr_full}, disks are block-level virtual resources for storage. Each disk is a filesystem on a server. For use purposes, {ks_product_abbr_full} provides system disks and data disks. The following content lists the differences between system disks and data disks:

* System disk: a virtual disk that contains an OS. A system disk is created each time you create a VM. The lifecyle of a system disk is the same as that of the VM.
* Data disk: a disk for storage. You can create a data disk when or after a VM is created.

The **Disks** module is available to all users.  User **admin** can view and manage all disks in all projects. The following table compares the permissions granted to non-admin users of different project roles.

[cols='1a,3a']
|===
|Project role |Permission

|Administrator
|View and manage all disks in the projects to which they belong.

|Operator
|View and manage all disks in the projects to which they belong.

|Viewer
|View only all disks in the projects to which they belong.
|===
