本小节主要介绍如何在 K8s 集群内部连接 RadonDB MongoDB 数据库。


== 前提条件link:[id=b6cfb1f0961d41e3bc685d5d1dce2f19]

* 请确保已获取{ks_product_both}平台登录用户帐号和密码，且已获取平台操作管理权限。

* 请确保已安装 RadonDB MongoDB 应用，且应用状态处于**运行中**。


== 操作步骤link:[id=1f45850851b14786afe260e39d98e68f]

=== 查看账号信息link:[id=1fb75c883f684e69b050437c94d84470]

include::../../../../_custom/projectManagement/projectManagement-oper-openProjectPage.adoc[]

. 在左侧导航栏选择**配置** > **保密字典**。

. 在**保密字典**页面的搜索框中输入 **admin-root** 并按**回车键**。

. 在搜索结果中找到新建的 MongoDB 应用对应的字典，点击字典的名称进入其详情页面。

. 右侧的**数据**区域显示用户名和密码。
+
密码默认显示为密文，可通过右上角的图标进行切换。
+

=== 访问数据库link:[id=30ac75b3462d4cc7a0d3afeae3bb3165]

. 在 K8s 集群内部创建一个 Pod，该 Pod 作为 MongoDB 客户端。

.. 准备一个 client.yaml 文件。
+
--
文件内容如下，其中 namespace 需要与 MongoDB 应用的 namespace（企业空间）相同。
include::../../../../../_ks_components/code/bash.adoc[]
apiVersion: v1
kind: Pod
metadata:
  name: mongoclient
  namespace: default
spec: 
  containers: 
  - name: mongo
    image: radondb/mongodb:4.2.19
include::../../../../../_ks_components/code/codeEnd.adoc[]
--

.. 执行以下命令，创建 Pod。
+
--
include::../../../../../_ks_components/code/bash.adoc[]

kubectl apply -f client.yaml

include::../../../../../_ks_components/code/codeEnd.adoc[]
--

. 进入创建的 Pod，执行以下命令访问 MongoDB。
+
--
include::../../../../../_ks_components/code/bash.adoc[]

kubectl exec -it mongoclient -n <namespace> -- mongo mongodb://<username>:<password>@<ip:port>/?authSource=admin

include::../../../../../_ks_components/code/codeEnd.adoc[]
--
+
--
[%header,cols="1a,2a,2a"]
|===
|参数 |描述 |示例

|namespace
|RadonDB MongoDB 应用的企业空间。 |default

|username
|RadonDB MongoDB 数据库用户名。 |root

|password
|RadonDB MongoDB 数据库密码。 |root_pwd

|ip:port
|RadonDB MongoDB 的访问地址。 |radondb-ukuhdz-svc.p1:27017
|===
