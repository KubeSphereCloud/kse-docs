== 操作步骤

. 以具有pass:a,q[{ks_permission}]权限的用户登录{ks_product_left} Web 控制台并进入您的集群。
. 在左侧导航栏选择pass:a,q[{ks_navigation}]。

+
--
[%header,cols="1a,4a"]
|===
|功能区 |描述

|名称、标签和描述
|
显示当前集群的名称、标签和描述。

* 成员集群的名称由用户添加成员集群时设置。

* 集群名称右侧显示当前集群的标签。

|基本信息
|显示当前集群的提供商、Kubernetes 版本、{ks_product_right}版本和对企业空间的可见性。

|概览
|查看当前集群的资源状态，如项目、容器组、部署、有状态副本集、守护进程集、任务、定时任务、持久卷、服务、应用路由等的数量。

|工具
|显示 kubectl 命令行工具，和 kubeconfig 配置文件。

// |资源用量
// |显示所有节点的 CPU、内存、磁盘的总量和实时用量，以及集群允许创建的容器组总数和已创建的容器组数量。默认情况下每个节点最多允许创建 110 个容器组。

// |Kubernetes 状态
// |显示集群的每秒 API 请求次数、API 请求延迟、容器组调度次数和容器组调度失败次数。

// |节点
// |显示当前集群中特定资源用量最高的 5 个节点。您可以从下拉列表中选择按 CPU 用户、CPU 平均负载、内存用量、磁盘用量、Inode 用量或容器组用量排行。
|===
