基于青云云平台部署，可以充分利用云平台网络和存储的服务能力，为用户提供整体解决方案。

image::/images/ks-qkcp/zh/technical-white-papers/422.png[]

* **代理节点**：如 KubeSphere 部署在基础网络，需预先创建一个域名代理节点，保障 KubeSphere 能够访问到青云云平台的 API 服务。
* **控制节点（master）**：其上提供了 Kubernetes 所需的控制面板组件，提供了资源调度、控制等能力，为达到高可用，一般建议为三个节点。
* **工作节点（worker）**：控制节点会动态调度 Pod 到工作节点，工作节点上运行着承载服务能力的容器。
* **etcd 节点**：保存 Kubernetes 集群信息，为达到高可用，一般建议三个节点。etcd 可以部署于独立的专门用于 etcd 运行环境的集群，也可部署在 KubeSphere 集群所用的节点或者以 Pod 方式部署在 Kubernetes 集群中，Pod 方式的性能要低于直接部署在物理机上。
* **镜像仓库节点**：集中保存平台所需的 docker 镜像，为达到高可用，需要两台主节点、一台数据库节点、一台缓存节点、一台日志节点。镜像仓库也可以 Pod 的方式部署在 Kubernetes 集群中。
* **监控/日志/告警**：可将相关服务部署于集群中，也可对接外部的企业已有的统一的监控、日志告警系统，同时 QingCloud 自己的统一监控、日志、告警服务也在开发中，相关参考方案会在之后提供。
