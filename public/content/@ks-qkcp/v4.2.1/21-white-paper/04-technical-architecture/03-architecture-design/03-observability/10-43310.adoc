== 核心功能

在某些情况下，用户需要建立其特有的可观测性标准，例如自定义监控指标和监控形式。此外，他们还需要手动将常用工具集成至云端，如用于 Kubernetes 监控的事实标准工具 Prometheus。换言之，自定义功能已成为行业上的必要需求，不仅需要各类云原生应用提供云上业务支撑，同时也需要细粒度全监控功能，以提前检测出任何可能对业务造成影响的问题。

=== 架构设计

image::/images/ks-qkcp/zh/technical-white-papers/43310.png[]

KubeSphere 自定义监控的架构设计以 **Dashboard as Code** 为核心理念，整体采用 Kubernetes 原生架构，通过 CRD（Custom Resource Definition）定义监控仪表盘与面板模板，实现声明式可视化管理。系统由前端、后端服务和监控数据层三部分组成：前端提供图形化配置界面，支持用户以拖拽或 YAML 编辑方式创建仪表盘；后端以 **API 定义为核心**，提供统一的接口用于管理仪表盘，并支持 **Grafana 仪表盘导入**（将 Grafana JSON 模板转换为 CR 对象），以便兼容现有监控资产；监控数据层由 **Prometheus** 提供统一的数据源，用于指标采集与查询。

在运行机制上，用户定义的自定义监控对象（如 `Dashboard` CR）将转化为可展示的渲染模板，前端界面据此生成图表布局与 PromQL 查询逻辑，形成 **CR → 模板 → 可视化渲染** 的闭环。架构整体支持 GitOps 与版本化管理，使仪表盘定义可代码化、可追溯。
