== 核心功能

Kubernetes Event 是对集群内资源对象状态变化、系统行为或错误信息的实时记录。作为一种特殊的资源对象，它具备以下特性：

* **详细记录事件信息**：包括首次与最近发生时间、出现次数、事件类型（Normal/Warning）、涉及对象、来源组件及触发原因。
* **面向资源对象关联与定位**：每个工作负载、Pod、Node 等资源均可关联事件，便于快速诊断问题来源。
* **帮助运维排障**：提供运行时错误和状态变化的追踪依据，是排查集群稳定性与应用可靠性问题的重要数据源。

Kubernetes 默认将事件数据存储在 Etcd 中，并为了控制性能及存储开销，仅保留最近约 1 小时的记录。因此，平台通常需要将事件导出并汇聚到更持久的存储中，以支持长期查询与分析。

== 架构设计

image::/images/ks-qkcp/zh/technical-white-papers/4334.png[]

KubeSphere 事件系统通过独立的事件采集与存储架构，突破 Etcd 存储限制，实现更完整的事件生命周期管理。

**主要组件与能力**

[%header,cols="1a,4a"]
|===
|组件	|作用
|kube-events	|监听 Kubernetes Event，并统一导出、转换与增强
|事件处理与聚合模块	|提供事件分类、去重、批量上报、可靠传输等能力
|事件持久化与索引存储	|支持存储至时序数据库或搜索引擎以便持续查询
|可视化展示与告警联动	|UI 聚合展示 + 与告警系统（如 Alertmanager）协作
|===
