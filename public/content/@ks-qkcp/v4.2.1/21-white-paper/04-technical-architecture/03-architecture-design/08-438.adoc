== 核心功能

IPPool 可以帮助集群管理员、企业空间管理员、项目管理员对当前的集群、企业空间、项目所使用的 Pod IP 进行规划；IPPool 需要在启用 calico 插件的模式下启动；支持网络策略可视化管理。

主要功能：

* 创建企业空间时, 可以勾选是否添加“默认的企业空间网络策略”；
* 企业空间管理员可以删除“默认的企业空间网络策略”；
* “企业空间网络策略”会自动为每个项目生成“项目网络策略”, 自动生成的策略只能查看, 不能修改和删除；
* 项目管理员可以为自己有权限的项目配置“项目网络策略”。

== 架构设计

1. 集群内使用的 CoreDNS 既是权威 DNS 也是递归 DNS，上游的 DNS 可自定义。（在集群内部，可以使用 `ks-apiserver.kubesphere-system.svc.cluster.local` 来访问 `kubesphere-system` namespace 下的 `ks-apiserver` 这个 service）。
2. 集群所使用的 domain 可以在安装时自定义，默认使用 `cluster.local`。
3. 如果底层网络已经互通，建议作为数据中心 DNS 上 ZONE 的延伸。
4. 在 K8s 集群内，使用 https://github.com/kubernetes-sigs/external-dns 可以向 DNS 服务商暴露 K8s 集群中的 service 和 ingress，这是一种主动 push 的模型。
