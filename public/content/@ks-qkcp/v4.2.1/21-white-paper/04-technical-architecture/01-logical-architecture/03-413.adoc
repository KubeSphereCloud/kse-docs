---
title: "核心平台层"
weight: 03
---

核心平台层作为 KubeSphere 的业务逻辑与平台能力核心，基于 Kubernetes 构建统一治理与服务框架，通过 LuBan 微内核提供可扩展、可插拔的系统形态。该层面向企业级场景，提供多维度管理能力、可靠的安全体系以及可持续演进的组件化生态，主要由以下功能模块构成：

[discrete]
=== 1. 认证与鉴权（Authentication & Authorization）

* 支持多种身份认证体系，包括 LDAP、OAuth2、OIDC 等企业级集成方式。
* 提供基于角色的访问控制（RBAC），确保不同用户、团队的权限边界清晰可控。
* 满足企业合规性需求，实现细粒度、安全的权限管理策略。

通过统一身份体系与权限模型，为平台的多租户与服务治理提供安全基础。

[discrete]
=== 2. 多集群管理（Multi-Cluster Management）

* 支持多个 Kubernetes 集群统一注册、纳管与拓扑视图展示。
* 提供跨集群的资源调度、策略配置、监控告警等能力。
* 支持多云、混合云与边缘环境，实现资源统一协同。

有效解决大规模分布式集群环境下的可操作性与可观测性难题。

[discrete]
=== 3. Kubernetes 资源管理（K8s Resource Management）

* 统一管理集群中的核心原生资源（Pod、Service、Ingress、ConfigMap 等）。
* 提供可视化面板与直观操作流程，降低 Kubernetes 使用门槛。
* 支持 API、CRD 扩展，实现原生一致性与更高自治能力。

让企业用户在不牺牲 Kubernetes 原生能力的前提下，更高效地运营集群资源。

[discrete]
=== 4. 多租户管理（Multi-Tenancy Management）

* 基于 Workspace/Project/User/Role 的层级化租户模型。
* 支持资源配额、网络策略与安全隔离，保障租户间边界与性能稳定。
* 适应大型组织结构，支持灵活团队协同与权限分配。

为云原生平台提供真正企业级的组织治理能力。

[discrete]
=== 5. 扩展组件管理（Extension Management）

* 基于 LuBan 微内核，提供 UI 和 API 的可热插拔机制。
* 支持官方与自定义扩展组件，满足业务多样化与创新需求。
* 提供统一的安装、注册、升级、配置与生命周期管理能力。
* 支持第三方后端 API 和前端项目无缝嵌入，实现即插即用。

该模块是 KubeSphere 平台生态扩展与差异化能力的核心驱动力。
