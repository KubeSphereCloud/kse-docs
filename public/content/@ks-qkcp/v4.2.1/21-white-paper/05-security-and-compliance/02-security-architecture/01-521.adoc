---

KubeSphere 在云原生架构基础上，提供从基础设施到应用层的全链路安全防护体系。通过集成 Kubernetes 原生安全能力与平台级增强机制，构建涵盖供应链、运行时、数据、审计与运维的多层次安全防护闭环，保障企业容器平台的安全、合规与可控。

== 供应链安全

供应链安全是保障软件交付可信的首要环节。KubeSphere 通过以方式构建安全的供应链体系：

* 制品来源可信：支持与私有镜像仓库集成，实现镜像签名与漏洞扫描。
* CI/CD 安全管控：在流水线中引入安全扫描节点，对代码依赖、镜像组件进行安全检测。
* 镜像签名与验证：支持 Cosign、Notary v2 等镜像签名机制，确保运行时仅加载可信镜像。
* 依赖与包管理安全：在构建过程中检测开源依赖漏洞与许可证风险，降低供应链攻击风险。

== 镜像安全

镜像安全聚焦于容器运行前的安全防护：

* 镜像漏洞扫描：集成 Trivy、Clair 等扫描工具，识别基础镜像及组件漏洞。
* 镜像基线与加固：提供企业级镜像基线模板，建议仅保留必要工具与权限，减少攻击面。
* 镜像签名与拉取策略：支持基于镜像签名的拉取策略，仅允许运行经过验证的镜像。
* 镜像仓库访问控制：通过 RBAC 与命名空间隔离，限制镜像访问范围，防止未经授权的使用。

== 运行时安全

运行时安全关注容器在执行阶段的防护能力：

* 安全策略管控：支持 Pod Security Admission、Gatekeeper，限制特权容器、宿主机挂载等高危操作。
* 网络隔离：通过 NetworkPolicy 控制服务间通信，实现最小化网络访问原则。
* 静态数据加密：支持 etcd 数据加密、存储卷加密（StorageClass 加密参数）。
* 传输加密：全平台 API、组件通信均采用 TLS 加密。
* 密钥与证书管理：支持集成外部 KMS，集中管理 Secrets。
* 多租户安全隔离：基于 KubeSphere 多租户架构，实现命名空间、网络、存储、权限的多层隔离。

== 基础环境安全

平台底层的主机、Kubernetes 组件与操作系统安全加固是全链路防护的根基：

* 节点安全加固：使用 CIS Benchmark 规范进行节点配置检查与加固。
* 组件安全配置：kube-apiserver、etcd、kubelet 等关键组件均支持 TLS 双向认证与最小权限配置。
* 集群体检报告：自动检测节点状态、配置合规性与安全策略执行情况，提供可视化报告与安全评分，辅助管理员快速定位问题。
* 密钥轮换与审计：提供证书定期轮换机制，防止过期与泄露风险。

== 审计与日志

审计能力是安全合规与事件追踪的重要手段：

* 操作审计：记录用户在平台、项目、集群层面的操作行为。
* API 审计日志：启用 Kubernetes API 审计日志并集中展示。
