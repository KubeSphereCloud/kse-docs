本节介绍如何将满足条件的 HTTP 请求路径替换为指定路径。

故障注入期间，通过 GET 方法访问注入 Pod 的根路径 `/` 时，请求路径将被替换为 `/50x.html`。

== 操作步骤
. 登录 KubeSphere Web 控制台，进入目标集群，在左侧导航栏点击 *混沌工程*。
. 点击 *创建*，选择故障类型为 *HTTPChaos*，配置以下参数后点击 *创建*。

[%header,cols="1a,4a"]
|===
|参数 |描述

|目标（target）
|选择 `Request`，对请求生效。

|端口（port）
|监听端口，例如 `80`。

|请求方法（method）
|触发故障的 HTTP 方法，例如 `GET`。

|替换路径（replace.path）
|替换后的请求路径，例如 `50x.html`。

|持续时间（duration）
|故障持续时间，例如 `10m`。

|生效范围（selector）
|选择故障注入的目标命名空间和 Pod。
|===


验证故障效果：正常情况下访问 Pod IP 返回 `index.html` 页面；故障注入期间将返回 `50x.html` 页面内容。
