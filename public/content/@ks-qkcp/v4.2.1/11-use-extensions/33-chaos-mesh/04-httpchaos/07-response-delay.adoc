本节介绍如何为满足条件的 HTTP 响应注入延迟。

故障注入期间，响应码为 404 的请求（如访问不存在的路径）将延迟 5 秒返回，响应码为 200 的请求不受影响。

== 操作步骤
. 登录 KubeSphere Web 控制台，进入目标集群，在左侧导航栏点击 *混沌工程*。
. 点击 *创建*，选择故障类型为 *HTTPChaos*，配置以下参数后点击 *创建*。

[%header,cols="1a,4a"]
|===
|参数 |描述

|目标（target）
|选择 `Response`，对响应生效。

|端口（port）
|监听端口，例如 `80`。

|请求方法（method）
|触发故障的 HTTP 方法，例如 `GET`。

|响应码（code）
|触发故障的 HTTP 响应码，例如 `404`。

|延迟时间（delay）
|注入的延迟值，例如 `5s`。

|持续时间（duration）
|故障持续时间，例如 `10m`。

|生效范围（selector）
|选择故障注入的目标命名空间和 Pod。
|===


验证故障效果：

[source,shell]
----
curl "{{podIP}}"        # 正常返回，无延迟
curl "{{podIP}}/xxx"    # 故障注入期间延迟约 5 秒后返回 404
----
