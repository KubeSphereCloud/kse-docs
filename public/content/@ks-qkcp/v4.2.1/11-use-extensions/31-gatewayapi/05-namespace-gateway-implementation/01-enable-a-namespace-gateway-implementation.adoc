本节介绍如何启用项目层级的 Gateway API 网关实现。

## 前提条件

- KubeSphere 平台需要安装并启用 **Gateway API 网关** 扩展组件。

- 您需要加入一个项目并在项目中具有 **项目 Gateway API 网关管理** 权限。

## 操作步骤

1. 以具有 **项目 Gateway API 网关管理** 权限的用户登录 KubeSphere Web 控制台并进入您的项目。

2. 在左侧导航栏选择 **Gateway API 网关 > 网关实现**。

3. 在 **项目网关实现** 页签，选择一个项目，点击**启用**。

4. 在弹出的对话框，设置网关实现的参数，然后点击**确定**。

[%header,cols="1,4"]
|===
| 参数 | 描述

| 容器组副本数量
| 期望的容器组副本数量。设置容器组副本数限制时，该配置项不生效。

| 实际运行容器组副本数限制
| 限制实际运行的容器组副本的最小值和最大值，以确保网关性能并防止资源耗尽。基于 HPA 实现自动伸缩。

| 访问模式
a|
网关实现服务的外部访问模式：

* *NodePort*：将节点端口映射到服务端口，可通过节点 IP 和端口访问服务。
* *LoadBalancer*：在 NodePort 服务的基础上分配外部 IP，并绑定外部负载均衡器。
+
外部负载均衡器监听服务对应的节点端口，从而可以通过外部 IP 地址访问服务。
+
该模式依赖负载均衡器插件和基础设施环境支持，并需要设置相关注解。有关更多信息，请联系您的基础设施环境提供商。

| 负载均衡器提供商
| 网关实现服务的负载均衡器提供商。仅在访问模式为 LoadBalancer 时可用。选择负载均衡器提供商后，系统会在设置网关实现服务注解时提供注解提示。

| 注解
| 网关实现服务的注解。选择负载均衡器提供商后，点击键文本框时，系统会在下拉列表中显示注解提示。仅在访问模式为 LoadBalancer 时可用。有关更多信息，请联系您的基础设施环境提供商。

| CPU 预留
| 为网关实现负载容器预留的 CPU 资源（单位：核）。

| CPU 上限
| 网关实现负载容器可使用的 CPU 资源上限（单位：核）。

| 内存预留
| 为网关实现负载容器预留的内存资源（单位：MiB）。

| 内存上限
| 网关实现负载容器可使用的内存资源上限（单位：MiB）。

| 自动创建网关
| 自动创建一个项目网关，便于快速创建 Gateway API 应用路由并绑定到该网关。

|===
