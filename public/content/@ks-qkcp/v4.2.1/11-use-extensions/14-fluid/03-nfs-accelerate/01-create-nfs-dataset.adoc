本节介绍如何创建 NFS 数据集，用于连接 NFS。


== 前提条件

* KubeSphere 平台需要安装并启用 **Fluid** 和 **Fluid Console** 扩展组件。

:relfileprefix: ../../../../
* {empty}
include::../../../../_custom/clusterManagement/clusterManagement-prer-requiredPermission_new.adoc[]

:relfileprefix: ./

* 已配置完成 NFS 存储系统，且已获取 NFS 的挂载路径。


== 操作步骤

include::../../../../_custom/clusterManagement/clusterManagement-oper-openClusterPage.adoc[]

+

include::../../../../../_ks_components/oper-navigate.adoc[]

+

include::../../../../../_ks_components/oper-clickCreate.adoc[]

. 在**基本信息**页签，设置**数据集**的基本信息，然后点击**下一步**。
+
--

[%header,cols="1a,4a"]
|===
|参数 |描述

|名称
|数据集的名称。

|别名
|数据集的别名。不同数据集的别名可以相同。

|项目
|数据集所属的项目。

|描述
|数据集的描述信息。描述可包含任意字符，最长 256 个字符。
|===
--

. 在**数据源**页签，配置数据源信息，然后点击**下一步**。
+
--
[%header,cols="1a,4a"]
|===
|参数 |操作

|数据源类型
|选择 NFS。
// 支持选择 S3、NFS、PVC、JuiceFS 等类型的数据源。

|名称
|输入自定义名称。

|读写模式
|选择对数据源的读写模式：**只读**或**读写**。

|数据源
|输入 NFS 数据源的挂载路径。

|自动创建 ThinRuntime
|建议勾选该选项。

系统会为 NFS 数据源自动创建运行时 (ThinRuntime)，运行时会同时生成连接 NFS 的 PVC，供后续 NFS 加速使用。可在**存储 > 持久卷声明**页面查看该 PVC。
|===
--

. 在**高级设置**页签，点击**创建**。
+
创建完成的数据集将显示在数据集列表中。
