== KubeSphere 企业版

{ks_product_right}平台的资源管理采用集群和租户两种视角，其中租户视角更贴合企业实际架构。平台分为企业空间和项目两级，用户可根据实际行政架构灵活定义，同时各租户的配额支持通过页面配置，无需编写复杂 YAML 文件，极大简化了资源管理流程。在权限管理方面，平台提供细粒度权限控制，既有内置角色也支持自定义角色配置，不同角色的用户可灵活关联租户，全面保障资源使用的安全性。

== OpenShift
支持 namespace 级别租户隔离，内置管理员和开发者两个管理入口。支持自定义角色、项目配额（通过 YAML 或命令行进行配置），支持租户网络隔离。

== Rancher
支持可视化的用户、租户管理，租户下支持管理多个 namespace。支持网络隔离，支持项目配额，支持可视化角色、成员管理。
