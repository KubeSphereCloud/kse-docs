== KubeSphere 企业版

自{ks_product_left} v4.0 起，KubeSphere 引入了扩展机制并推出全新的 LuBan 架构，该架构基于 Kubernetes 构建，采用微内核 + 扩展组件模式：系统核心（KubeSphere Core）仅保留运行所必需的基础功能，而各独立业务模块则以热插拔的扩展组件形式存在。所有功能组件及第三方组件均可无缝集成到控制台中并独立维护版本，用户可在系统运行时动态安装、卸载、启用或禁用扩展组件，从而实现即插即用的灵活云原生操作系统。

image:/images/ks-qkcp/zh/competitiveness-analysis/4.0-architecture.png[]

=== 架构特性

KubeSphere LuBan 架构的主要特性在于其插件式核心框架，支持组件独立开发、部署与升级，实现系统功能的灵活扩展同时保持核心稳定；同时提供开放基础 UI 组件库，实现完全开放与可定制，满足多样化设计和功能需求；具备前后端热更新与修复功能，确保实时更新、提高开发效率与用户体验，保障系统高可用性；此外，开放性扩展中心鼓励生态共建，支持第三方组件无缝集成。详见：link:https://kubesphere.io/zh/docs/v4.1/01-intro/02-architecture/[{ks_product_right}架构介绍]。

== OpenShift

OpenShift 的技术架构基于原生 Kubernetes，并通过企业级扩展打造出一个安全高效的容器平台。它集成了CI/CD 与开发工具，实施了包括 OAuth、RBAC、网络策略和多租户隔离在内的完善安全措施，同时具备自动化管理功能，如自动扩容与自愈能力。此外，OpenShift 支持混合云和多集群部署，实现统一管理；其与红帽生态系统（如 RHEL 和 RHCOS）的深度整合确保了面向企业环境的专业支持和高效运维。

image:/images/ks-qkcp/zh/competitiveness-analysis/openshift2.png[]

=== 架构特性

OpenShift 底层绑定红帽自身产品 RHCOS 与 RHEL 作为底座操作系统，绑定自身产品一方面降低了整体的维护成本，但也降低了产品方案扩展的灵活性。详见：link:https://docs.redhat.com/en/documentation/openshift_container_platform/4.17/html/architecture/index[OpenShift Container Platform 4.17 官方文档]。

== Rancher

Rancher Server 由认证代理（Authentication Proxy）、Rancher API Server、集群控制器（Cluster Controller）、etcd 节点和集群 Agent（Cluster Agent） 组成。除了集群 Agent 以外，其他组件都部署在 Rancher Server 中。该架构设计方式在节点规模较大的场景下可能会产生性能瓶颈。

image:/images/ks-qkcp/zh/competitiveness-analysis/rancher2.png[]

=== 架构特性

Rancher Server 管控 Rancher 部署的 RKE 集群，架构与 Kubernetes 自身弱耦合，但可能产生性能瓶颈且因为没有利用 Kubernetes 调度系统的能力会存在稳定性问题。详见：link:https://ranchermanager.docs.rancher.com/reference-guides/rancher-manager-architecture[Rancher 架构介绍]。
