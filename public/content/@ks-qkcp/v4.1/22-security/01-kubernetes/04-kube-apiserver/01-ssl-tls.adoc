---
title: "SSL/TLS 加密套件"
weight: 01
---

. 修改宿主机 /etc/kubernetes/manifests/kube-apiserver.yaml 配置文件，在镜像的启动命令中添加参数。
+
[,bash]
----
- --tls-cipher-suites=TLS_ECDHE_ECDSA_WITH_AES_256_GCM_SHA384,TLS_ECDHE_RSA_WITH_AES_256_GCM_SHA384,TLS_ECDHE_ECDSA_WITH_AES_128_GCM_SHA256,TLS_ECDHE_RSA_WITH_AES_128_GCM_SHA256
----

. 验证加密套。
+
[,bash]
----
# 方法 1
openssl s_client -connect ${kube-apiserver 服务 IP 地址}
# openssl s_client -connect 192.168.200.2.6443

# 方法 2 主要客户端使用的同一个加密套字符串格式有差异
# 服务端 TLS_ECDHE_RSA_WITH_AES_128_GCM_SHA256
# 客户端 ECDHE-RSA-AES128-GCM-SHA256

# 验证设置的加密套，正确返回接口结果
curl \
--cacert /etc/kubernetes/pki/ca.crt \
--cert /etc/kubernetes/pki/apiserver.crt \
--key /etc/kubernetes/pki/apiserver.key \
-L https://192.168.200.2:6443 \
--ciphers ECDHE-RSA-AES128-GCM-SHA256
# 验证未设置的加密套，返回异常 “curl: (59) failed setting cipher list“
curl \
--cacert /etc/kubernetes/pki/ca.crt \
--cert /etc/kubernetes/pki/apiserver.crt \
--key /etc/kubernetes/pki/apiserver.key \
-L https://192.168.200.2:6443 \
--ciphers ECDHE-ECDSA-CHACHA20-POLY1305-SHA256
----
