本节介绍在如何在保持当前的 Kubernetes 版本不变的情况下将{ks_product_both}版本升级到 v3.5.0。如果您需要升级 Kubernetes 版本，请参阅link:../02-upgrade-kubernetes-and-kubesphere/[升级 Kubernetes 和{ks_product_left}]。

ifeval::["{file_output_type}" == "html"]
如果您的集群节点无法连接互联网，升级过程中需要使用开源工具 KubeKey 创建私有镜像服务。有关 KubeKey 的更多信息，请访问 link:https://github.com/kubesphere/kubekey[GitHub KubeKey 仓库]。
endif::[]

ifeval::["{file_output_type}" == "pdf"]
升级过程中将使用开源工具 KubeKey 创建私有镜像服务。有关 KubeKey 的更多信息，请访问 link:https://github.com/kubesphere/kubekey[GitHub KubeKey 仓库]。
endif::[]


== 前提条件link:[id=2b15ea4529ee42d0a77b84ad00dcf180]

* 您需要联系{ks_product_both}的交付服务专家获取{ks_product_left} v3.5.0 安装包。如实施现场条件允许在线拉取公开仓库，具备在线安装条件，可联系 KubeSphere 产品团队获取在线安装文件及相关信息。

* 请确保当前{ks_product_both}版本为 v3.4.x。如果您的{ks_product_both}版本低于 v3.4.x，请先将{ks_product_both}版本升级到 v3.4.x。

* 请确保当前 Kubernetes 版本为 v1.21.x、v1.22.x、v1.23.x, v1.24.x, v1.25.x 和 v1.26.x。

== 开始升级link:[id=83037a0ce04a42e8a3192744a89d2565]

include::../../../_custom/installationAndUpgrade/installationAndUpgrade-oper-editCcKsInstaller_new.adoc[]

// Bash
include::../../../../_ks_components/code/bash.adoc[]

kubectl edit cc -n kubesphere-system

include::../../../../_ks_components/code/codeEnd.adoc[]

将 **local_registry** 参数设置为私有镜像服务的地址 **dockerhub.kubekey.local**，然后保存文件。

// YAML
include::../../../../_ks_components/code/yaml.adoc[]

spec:
  persistence:
    storageClass: ""
  authentication:
    jwtSecret: ""
  local_registry: dockerhub.kubekey.local # 配置私有镜像仓库地址
  namespace_override: "" # 指定镜像namespace,当镜像全部上传到一个仓库的namespace中时使用
  etcd:
    monitoring: true      
    endpointIps: 192.168.0.2     # 配置 etcd 节点地址，多个 ip 使用逗号分隔（192.168.0.3,192.168.0.4,192.168.0.5）
    port: 2379             
    tlsEnable: true
include::../../../../_ks_components/code/codeEnd.adoc[]
--

. 执行以下命令开始升级{ks_product_left}：
