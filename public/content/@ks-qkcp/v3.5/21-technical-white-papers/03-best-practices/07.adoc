---
title: 容器入侵检测
weight: 07
---

基于容器的不变性和单一性，通过机器自学习的方式，生成行为基线规则对异常行为进行检测，深度感知容器内部各类事件。结合进程行为，利用多维度数据关联分析，对入侵威胁进行持续监控发现，能够自动感知失陷容器并做出响应，可全面覆盖各类已知威胁场景。主要包括如下内容：

基于事件驱动机制，采集容器内部的网络、文件、进程和系统调用等多维度数据，对容器运行时事件深度感知；

采用智能数据分析，自动生成镜像的行为基线，充分利用业务应用微服务化的特性，持续监控分析容器中潜在的行为风险；

在持续监测行为风险的基础上，采用数据关联分析和行为模型分析快速判断入侵威胁，覆盖反弹 shell 和提权等多种威胁场景；

对于失陷容器，提供告警、隔离等多种灵活处置手段，方便安全运维人员及时介入与处置。

[%header,cols="1a,4a"]
|===
|入侵事件
|具体内容

|反弹 shell 检测
|监控容器内进程启动并获取进程的标准输入、标准输出、标准错误，实时发现容器内反弹 shell，告警信息记录目标 IP、目标端口、进程树、发现时间等信息。

|本地提权
|实时发现容器内异常的进程权限提升行为。

|恶意命令执行
|实时监控容器内的进程启动行为，通过异常的进程启动链发现入侵痕迹。

|网络攻击
|实时检测 DDOS 攻击、SQL 注入等常见网络攻击。

|容器逃逸
|容器逃逸的主流因素可分为用户配置不当（用户层）、容器引擎自身漏洞（服务层）、宿主机内核漏洞（系统层）。用户层（特权、敏感挂载、敏感权限）通过检测容器元数据判断是否存在逃逸风险，服务层和系统层通过检测进程的文本特征来判断是否为逃逸行为。

当前 Docker 容器逃逸的主流因素大致可以分为三类：

- 用户层：用户配置不当导致逃逸。
- 服务层：Docker 容器应用自身漏洞导致逃逸。
- 系统层：Linux 内核漏洞导致逃逸。
|===
