容器环境中，容器安全产品会自动针对容器进行分类，并创建工作负载。

工作负载中总共包含三种策略模式：学习、监控和保护。默认情况下，工作负载处于学习模式中，主动学习容器相关行为，并创建相应运行基线。切换至监视模式后，容器安全产品会实时监测运行时违反安全策略的情况，并产生相关告警。在监视模式下，不会自动创建规则，针对误报，可以手动进行加白。切换至保护模式后，容器安全产品会拒绝工作负载中任何违规行为。需要注意的是，用户自定义创建的工作负载没有保护模式。

=== 基于 Pod 安全标准和准入控制器的集群安全策略

Kubernetes Pod 安全性标准（Security Standard） 为 Pod 定义不同的隔离级别。这些标准能够以一种清晰、一致的方式定义如何限制 Pod 行为。

Kubernetes 提供了一个内置的 Pod Security 准入控制器来执行 Pod 安全标准 （Pod Security Standard）。 创建 Pod 时在名字空间级别应用这些 Pod 安全限制。

Kubernetes 允许通过准入控制器 webhooks 将策略决策与 API 服务器的内部工作分离，每当创建、更新或删除资源时都会执行这些决策。
