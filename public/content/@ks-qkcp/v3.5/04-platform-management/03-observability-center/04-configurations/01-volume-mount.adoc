本节介绍如何在{ks_product}安装前及安装完成后，为 Whizard 可观测中心配置本地持久卷。

== 在{ks_product}安装前配置

. 登录集群节点，执行以下命令编辑 cluster-configuration.yaml 配置文件：
+
--
// bash
[,bash]
----
vi cluster-configuration.yaml
----
--

. 将 **spec:monitoring:whizard:server**  下的 storageClass 设置为对应的存储类，volumeSize 参数修改为合适的存储大小，然后保存⽂件。
+
--
// YAML
[,yaml]
----
spec:
  monitoring:
    whizard:
      server:
        storageClass: ""
        volumeSize: 20Gi
----
--


== 在{ks_product}安装后配置

=== 前提条件

您需要在{ks_product_both}平台具有 **platform-admin** 角色。

=== 操作步骤

. 以具有 **platform-admin** 角色的用户登录{ks_product_left} Web 控制台。

. 将光标悬停在页面右下角的image:/images/ks-qkcp/zh/icons/hammer.svg[hammer,18,18] 图标，然后在菜单中选择 **kubectl**。

. 在 **kubectl** 对话框，执行以下命令编辑集群配置文件：
+
--
// bash
[,bash]
----
kubectl edit cc ks-installer -n kubesphere-system
----
--

. 在集群配置文件中，将 **spec:monitoring:whizard:server**  下的 storageClass 设置为对应的存储类，volumeSize 参数修改为合适的存储大小，然后保存⽂件。

+
--
// YAML
[,yaml]
----
spec:
  monitoring:
    whizard:
      server:
        storageClass: ""
        volumeSize: 20Gi
----
--
