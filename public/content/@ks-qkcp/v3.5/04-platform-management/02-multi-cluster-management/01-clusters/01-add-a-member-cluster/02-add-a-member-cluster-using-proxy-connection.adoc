本节介绍如何通过代理连接添加成员集群。

{ks_product_right}提供了代理服务 **tower**。如果主集群无法访问成员集群的 Kubernetes API 服务，您可以设置成员集群连接到主集群对外暴露的 **tower** 服务从而添加成员集群。

include::../../../../../_custom/platformManagement/multiclusterManagement/clusters/clusters-prer-platformAdminHostMemeber.adoc[]

. 在 **kubectl** 对话框，执行以下命令创建 **agent.yaml** 文件并在文件中输入从主集群获取的配置信息：
+
--
// Bash
include::../../../../../../_ks_components/code/bash.adoc[]

vi agent.yaml

include::../../../../../../_ks_components/code/codeEnd.adoc[]
--

. 在 **kubectl** 对话框执行以下命令创建 tower agent：
+
--
// Bash
include::../../../../../../_ks_components/code/bash.adoc[]

kubectl create -f agent.yaml

include::../../../../../../_ks_components/code/codeEnd.adoc[]

成员集群将自动通过 tower agent 连接主集群的 **tower** 服务并加入主集群上运行的{ks_product_both}平台。您可以在主集群{ks_product_left} Web 控制台的**平台管理 > 集群管理**页面查看成员集群。
