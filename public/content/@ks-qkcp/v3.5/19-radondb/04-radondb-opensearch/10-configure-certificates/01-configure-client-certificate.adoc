. 在 **securityconfig.yaml** 中增加 **clientcert_auth_domain** 配置。

+
--

include::../../../../../_ks_components/code/codeEnd.adoc[]

== 生成客户端证书link:[id=084470f44bae45758d25ba9e20ccedd1]

. 获取 CA 证书。集群自生成的 CA 证书名称为 <cluster-name>-ca。

+
--
// Bash
include::../../../../../_ks_components/code/bash.adoc[]

root@master1:/home/ubuntu/opensearch# kubectl -n opensearch get secret os-001-ca -oyaml
echo <ca.crt 内容> | base64 -d > root-ca.pem 
echo <ca.key 内容> | base64 -d > root-ca-key.pem

include::../../../../../_ks_components/code/codeEnd.adoc[]
--

. 签发客户端证书。
+
--
// Bash
include::../../../../../_ks_components/code/bash.adoc[]

openssl genrsa -out client-key-temp.pem 2048
openssl pkcs8 -inform PEM -outform PEM -in client-key-temp.pem -topk8 -nocrypt -v1 PBE-SHA1-3DES -out client-key.pem 
openssl req -new -key client-key.pem -subj "/C=CA/ST=ONTARIO/L=TORONTO/O=ORG/OU=UNIT/CN=CLIENT" -out client.csr 
openssl x509 -req -in client.csr -CA root-ca.pem -CAkey root-ca-key.pem -CAcreateserial -sha256 -out client.pem -days 730

include::../../../../../_ks_components/code/codeEnd.adoc[]

其中 **-subj** 后的内容可以自定义。
--

. 获取客户端证书的专有名称（DN）。
+
--
// Bash
include::../../../../../_ks_components/code/bash.adoc[]

root@master1:/home/ubuntu/opensearch/certs# openssl x509 -subject -nameopt RFC2253 -noout -in client.pem
subject=CN=CLIENT,OU=UNIT,O=ORG,L=TORONTO,ST=ONTARIO,C=CA

include::../../../../../_ks_components/code/codeEnd.adoc[]
--

. 在 OpenSearch Dashboards 将在上一步获得的映射给用户。
+
--
image::/images/ks-qkcp/zh/radondb-management/os-dashboards.png[]
--

. 使用证书访问集群。
+
--
// Bash
include::../../../../../_ks_components/code/bash.adoc[]

root@i-abmm2sr0:~/test# curl -k --cert client.pem --key client-key.pem https://172.22.9.17:30201/_cat/nodes?pretty

include::../../../../../_ks_components/code/codeEnd.adoc[]

其中：

* **-k**： 忽略服务端证书验证。

* **--cert**： 指定客户端证书。

* **--key**： 指定客户端证书的 key。

* **172.22.9.17**： 任意节点 IP。

* **30201**：9200 暴露在集群外部的端口号。

--


== 多用户访问集群link:[id=4d04202ecb234910b94837e217d7568b]


用户使用客户端证书绑定的 role 来获取相应的权限，根据实际情况，不同用户可配置不同的 role，同时生成对应的证书访问集群。

下面举例如何将生成的证书绑定在 all_access 和 dashboard_read_only 两个 role 上。关于如何生成证书，请参阅link:#084470f44bae45758d25ba9e20ccedd1[生成客户端证书]。

. 进入 OpenSearch Dashboards 管理页面，在左侧导航栏选择 **Security** > **Roles**。

. 在右侧的 **Roles** 页面，点击 **Create role**，输入角色名称并设置权限，点击 **Submit**。
+
--
image::/images/ks-qkcp/zh/radondb-management/multiuser-access.png[]
--

. 测试绑定是否成功。

.. 使用 all_access 证书访问集群，如果出现以下信息，则代表访问集群成功。
+
--
image::/images/ks-qkcp/zh/radondb-management/all-access-success.png[]
--

.. 使用 dashboard_read_only 证书访问 Dashboard，如果出现以下信息，则代表访问集群成功。
+
--
image::/images/ks-qkcp/zh/radondb-management/dashboard-only-success.png[]
--

.. 使用 dashboard_read_only 证书访问集群。如果出现报错，则符合预期。
+
--
image::/images/ks-qkcp/zh/radondb-management/dashboard-only-fail.png[]
--
