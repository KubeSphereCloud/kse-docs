== 客户场景

在高校的应用场景中，需要解决的关键问题是从学校到学院到班级再到学生的多级资源管理，且不同学院之间可能存在资源共享情况。针对各个班级来讲，同一学院下的班级将共用学院的资源配额，老师作为班级管理员可为班长和其他学生自定义相应的组织策略，使其可根据相应的权限创建和使用资源。

== 客户痛点

* 不同学院的学生对于不同资源的权限需求不尽相同，需合理对学院总体资源进行规划，并针对不同学院特性分配相应资源配额。

* 老师作为部门管理员角色，需对不同班级职务的学生设定不同的组织策略以满足师生协作管理的场景。

== 解决方案


image::/images/ep_offline/vdc_pwp/app_scene_educate.png[]

* 基于学校现有的组织架构，提供资源的多级管理功能，资源从学校层面分配给各学院后，再由学院根据不同班级的资源需求情况进行分配后，由各班级老师管理。
* 采用**混合模式**的资源管理模式，将其中部分资源给配置某一学院独立使用，剩余资源由其他学院共用。
* 自定义组织策略，并关联至具有特殊职务的学生，达到师生协同管理资源的目的。
* 自定义资源审批流程，规范资源申请和发放。



== 客户收益

* 自定义组织策略关联成员角色，满足师生协同管理需求。
* 流程管理功能，有效的规范了资源申请流程，避免资源浪费。
