== 客户场景

某一银行客户在实际生产中运用到了 2 套云环境，其一为生产环境（即生产云），其二为开发测试环境（即开发测试云）。在开发测试云环境中，处于成本考虑，相关的应用开发部门的成员在创建和使用云资源时需走线下审批流程，再由 IT 部门运维人员为其创建资源，供申请者使用。


== 客户痛点

云资源创建过程繁琐，资源申请流程缓慢，影响资源交付效率。

== 解决方案

image::/images/ep_offline/vdc_pwp/app_scene_bank.png[]

* 利用企业空间多级资源管理功能，按照某银行的实际组织架构模式，在企业空间内创建应用开发部门及其下级部门，同时为各部门创建成员并关联部门管理员、部门开发成员等角色。

* 采用**分配模式**的资源管理方法，IT 部门为各个应用开发部门分配相应的资源额度之后，应用开发部在配额范围内自主式创建和使用资源，有效控制 IT 成本。

* 各部门管理员可对其部门内资源使用情况以及消费明细进行监测，及时发现资源使用瓶颈或闲置资源，从而合理规划部门资源。


== 客户收益

* 精简资源交付流程，将线下的流程管理改为线上部门配额管理。

* 权限下放，将资源的创建和使用权利直接交给使用者，助力云资源自助服务，快速敏捷交付。 

* 采用分配模式的资源管理方式，促进资源回收，有效的控制 IT 成本。

* 部门管理员对资源使用情况监控，合理规划部门资源。
