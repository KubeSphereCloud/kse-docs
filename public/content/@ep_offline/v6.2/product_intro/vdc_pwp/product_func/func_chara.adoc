== 企业成员，角色分工

企业空间满足从集团到分公司再到部门视角的角色分配，BOSS 运维管理平台的运维管理人员（即集团管理员），可以创建集团的租户信息，每一个租户相当于一个分公司，租户登录至 Console 工作台后，根据分公司组织架构及资源管理情况，创建部门及部门下的成员，并为成员分配角色权限。

image::/images/ep_offline/vdc_pwp/func_chara_role.png[,70%]


== 基于组织架构的多级资源管理

企业空间提供的组织管理功能，在该功能下支持租户最多创建 5 级组织（部门）。在指定的组织（部门）下，创建成员并关联角色，则相应成员将拥有指定角色下的权限策略。根据权限策略规则，相应成员可对指定资源进行创建、管理等操作。组织下创建的资源将消耗该组织的相应资源配额。

image::/images/ep_offline/vdc_pwp/func_chara_res.png[]

== 上级部门的 VPC 资源可共享给其子部门，实现网络互通

上级部门创建的 VPC 网络资源，可以共享给其所管理的下级部门使用，从而实现上下级部门网络互通。示例如下：

在**资源管理**页面可查看``资源 ID``为 ``rtr-cpehenfm`` 的 VPC 网络``所属部门``为``上海分公司``。

image::/images/ep_offline/vdc_pwp/func_chara_vpc_1.png[]


``资源 ID``为 ``i-jfsd9icz`` 的云服务器``所属部门``为``市场开发部``，其中市场开发部是上海分公司的下级部门。


image::/images/ep_offline/vdc_pwp/func_chara_vpc_2.png[]


``上海分公司``的部门管理员可手动将云服务器（i-jfsd9icz）绑定到 VPC 网络（rtr-cpehenfm）上，实现上下级部门的网络互通。

image::/images/ep_offline/vdc_pwp/func_chara_vpc_3.png[]

== 通过配额管理，限制资源发放，控制 IT 成本

企业空间提供组织配额管理功能，默认情况下各个成员可共享租户的资源配额。主账户也可以为指定组织限制资源的分配额度，相应组织下的成员在创建资源时，将会消耗该组织独享的资源。同时，针对组织企业空间也支持回收其独立资源配额，对回收的资源做重新分配。通过资源配额限制，可有效的控制企业 IT 成本。

在企业空间内资源管理主要分为三个模式：共享模式、分配模式以及混合模式。以 20 台云服务器作为总资源量为例，三种资源管理模式，可做以下描述：

* 共享模式：假设分公司有 20 台云服务器，采用共享模式的情况下，分公司及下级所有部门将一起消耗这 20 台云服务器资源，直至用完为止。
* 分配模式：假设分公司有 20 台云服务器，自留 2 台使用，另外 18 台根据情况分别分配给下级各部门使用，每个部门均有其专属的云服务器资源。此种模式下，分公司以及各下级部门均有专属的独立资源配额。
* 混合模式：假设分公司有 20 台云服务器，将其中 10 台分配给下级各部门供其独立使用（即各部门均有专属的独立配额），剩余资源供分公司和下级各部门共享使用。

== 通过资源费用和用量统计，合理规划 IT 成本

企业空间提供基于组织的资源用量和费用的监测，部门管理员可快速查看部门资源使用情况，及时发现资源使用瓶颈或闲置资源，合理规划部门资源用量。

* 资源用量监测
+
支持查看当前组织下的资源概览、告警趋势、事件监控、资源使用排行 Top5 等信息。
+
image::/images/ep_offline/vdc_pwp/func_chara_bill_1.png[,70%]

* 资源费用监测
+
支持查看当前组织下的总消费、按资源类型统计或按区域统计的消费，同时，提供相应的消费趋势图等信息。
+
image::/images/ep_offline/vdc_pwp/func_chara_bill_2.png[,70%]

== 流程管理，规范 IT 资源申请流程

企业空间管理功能在为云平台用户解决了复杂的组织结构的资源管理问题的同时，为满足和匹配大型企业的资源流程审批管理体系，企业空间支持在自定义流程模板和流程审批。通过流程管理，规范企业 IT 资源申请和发放。

针对资源管理者角色，其可实现：

* 管理流程：定义整个公司的流程体系，包括自定义流程类型和模板、基本信息以及审批步骤等。
* 审批流程：从管理者视角审批各项资源申请用途，避免资源浪费。
* 监管流程：监管流程审批情况，处理流程审批过程中的异常问题等。

针对资源申请者角色，在需要资源时可提出申请流程，已创建所需资源。


image::/images/ep_offline/vdc_pwp/func_chara_process.png[]

== 通过身份提供商对接客户账户系统

针对组织结构复杂、子公司众多的客户，使用企业空间功能时，需要批量为其公司员工创建对应的成员账户，此时，可采用 LDAP（Lightweight Directory Access Protocol）方式，将客户已有的其他系统上的账户与云平台打通，公司成员可直接用业务系统的账户登录云平台。

image::/images/ep_offline/vdc_pwp/func_chara_ldap.png[,70%]
