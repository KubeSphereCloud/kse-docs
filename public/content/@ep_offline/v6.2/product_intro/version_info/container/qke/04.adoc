优化 CCM 组件，完善集群 LB、EIP 资源新增与释放能力。

== 功能介绍

Cloud Controller Manager 提供 Kubernetes 集群与云平台基础产品的对接能力，例如负载均衡器的管理，qingcloud-cloud-controller-manager 是云平台实现的 CCM 组件，在集群创建时会作为默认组件运行在集群中，用于 Kubernetes 集群和云平台之间的联动，目前涉及资源主要是云平台负载均衡器和公网 IP。

CCM 负责为 LoadBalancer 类型的 service 创建或者配置负载均衡器，包括创建负载均衡器、配置监听器的属性、配置监听器 backend 数量等；持续关注集群节点和 LoadBalancer 类型的 service 的变化，如果服务后端节点发生变化，及时更新负载均衡器监听器的 backend。

== 价值说明

金融行业客户对于网络资源管理相对其他行业用户有更严格的要求，涉及网段的分配、负载均衡器的管理、公网 IP 的使用等。金融行业用户通常使用 QKE 集群作为业务的底层平台使用，对运维要求比较高，优化 CCM 组件实现业务部署、组件安装引起的 LB、EIP 自动创建、绑定及删除后的自动释放等，有助于客户对于网络资源的更灵活的管理与运维。
