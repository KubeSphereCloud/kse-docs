对接 IAM，可以实现集群内 IAM 用户权限管理。

== 功能介绍

访问鉴权管理（Identity and Access Management，IAM）是云平台上提供的身份识别和访问控制服务。使用者可以统一管理和控制接入实体的认证和授权，更安全地自主管控账户下所有资源访问权限。通过和 IAM 的对接，QKE 能够提供租户管理能力，满足多个子账号访问并管理同一个集群的使用场景，并通过权限管理实现不同子账号对集群的操作/访问粒度。

== 价值说明

权限管理与控制是企业用户非常关注的一个模块，Kubernetes 集群拥有 RBAC 鉴权机制，允许用户通过 Kubernetes API 动态配置策略，实现基于组织中用户的角色调节控制对计算机或网络资源的访问。QKE 和 IAM 对接后完成了云平台账号管理和集群 RBAC 的打通，能够通过云平台的账号管理实现对集群的细粒度管控。
