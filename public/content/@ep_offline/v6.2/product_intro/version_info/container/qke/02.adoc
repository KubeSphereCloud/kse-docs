对接 U10000 文件系统，可作为持久卷存储后端。

== 功能介绍

U10000 是一款面向海量非结构化数据的统一数据存储平台，同时支持多种数据访问协议及多协议间数据无损互通，满足数据全生命周期管理的需求，为上层业务提供高可靠、易扩展、高性价比的海量数据存储。

用户可在 QKE 集群创建/使用过程中选择将 U10000 文件系统作为集群默认 pvc，完成容器挂载存储多读多写的使用，在 QKE 控制台可以看到挂载的 U10000 文件系统的使用信息。

== 价值说明

QKE 当前使用的存储默认是云平台上的 NeonSAN 存储，能满足通常情况下的数据存储场景（OLTP 和 OLAP 场景），但是 NeonSAN 不支持多读多写的存储要求，所以如果有用户存在共享存储的场景（如运行 Nginx），QKE 需要对接其他存储后端进行实现，最初 QKE 通过对接 vNAS 提供服务，但是 vNAS 的稳定性和性能都存在不足，在 U10000 推出后，QKE 完成了和 U10000 文件系统的对接，可以更好地支持 Kubernetes 集群使用共享存储的场景需求。
