weight: 1
---


BOSS 主要以管理、监控和告警三个方向对云平台资源进行管理和运维。

== 管理

* 支持以 `上帝视角` 查看{product_name}，以超级管理员的身份对云资源进行操作，可对管理节点、计算节点、SDN 等终端用户无感知的设施进行运维操作。

* 提供资源管理、服务管理、用户管理、计量&计费管理、用户操作日志，以及 API 接口管理等功能，以帮助{product_name}的运营方完成日常运营工作，面向最终用户提供自助服务流程，实现按需自助服务的目的。
* 支持接入更多的青云管理工具，如 BI2.0、V2V 迁移工具等。

== 监控

BOSS 从物理资源和虚拟资源两个角度对{product_name}的运行数据进行收集，列表展示{product_name}的资源使用情况，帮助运维人员了解云资源的增长趋势，从而协助决策集群扩容的时机。

== 告警

* BOSS 通过 API 获取集群中所有告警的信息，运维人员可简单方便的查看物理机、虚拟机的运行情况，管理物理机、虚拟机等物理设施以及虚拟设施的状态以及查看告警日志等。

* 通过对告警信息分类、存储、聚合以判断告警是否需要发起通知，支持对通知对象和通知频率进行定制，且支持邮件、短信、Webhook 推送等通知方式。
