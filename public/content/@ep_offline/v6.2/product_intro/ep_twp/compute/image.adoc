镜像（Image）是带有操作系统并包含软件及必要配置的云服务器模板。用户可以基于已有镜像创建云服务器，也可以将自有云服务器捕获为新镜像，以备后用；用户可以根据自己的需求将任何应用软件（比如，数据库、中间件等）放入镜像中。

== 镜像类型

在企业云平台，镜像分为**系统镜像**、**自有镜像**、**共享镜像**、**市场镜像**。

[cols="1,4"]
|===
|镜像类型 |说明

|系统镜像
a|云平台官方提供的镜像，为常见的标准操作系统（ Linux、Windows、Unix）镜像，所有用户可见，包括操作系统以及预装的公共应用。系统镜像具有高度稳定性，皆为正版授权。

|自有镜像
|用户通过捕获一个云服务器来自行创建的镜像，包含操作系统或业务数据、预装的公共应用以及用户的私有应用的镜像，仅用户个人可见。 +
自有镜像可以共享给项目，共享后项目范围内成员可见。

|市场镜像
|提供预装操作系统、应用环境和各类软件的优质第三方镜像。

|共享镜像
|由云平台上其他用户共享而来的自有镜像。
|===

== 镜像格式
[cols="1,4"]
|===
|镜像类型 |说明

|QCOW2
a|QCOW2 格式镜像是 QEMU 模拟器支持的一种磁盘镜像，是用一个文件的形式来表示一块固定大小的块设备磁盘。与普通的 RAW 格式镜像相比，QCOW2 格式有如下几个特性：

* 支持更小的磁盘占用。
* 支持写时拷贝，镜像文件只反映底层磁盘变化。
* 支持快照，可以包含多个历史快照。
* 支持压缩和加密，可以选择 ZLIB 压缩和 AES 加密。

|RAW
|RAW 格式是直接给云服务器进行读写的文件。RAW 不支持动态增长空间，是镜像中 I/O 性能最好的一种格式。

|===

== 镜像注册及使用流程

系统镜像和自有镜像可由云计算运维管理平台通过镜像注册生成镜像模板和租赁信息，为云平台用户在 Console 提供 Linux、Windows 和 Unix 模板，并能更具上游厂商的更新版本及时制作新模板。用户基于镜像模板创建云服务器，并可基于已有云服务器制作称新镜像（自有镜像），以及共享自有镜像。整个业务流程如下：
