== 计算虚拟化

计算虚拟化是企业云平台的服务器虚拟化组件，是整个云平台架构的核心组件。

计算虚拟化通过在硬件之上增加一个 Hypervisor，将 CPU、内存、I/O 等服务器物理资源转化为一组可统一管理、调度和分配的逻辑资源，实现软硬件的解耦。基于这些逻辑资源，可在单个物理服务器上构建多个可同时运行又相互隔离的虚拟主机执行环境，实现更高的资源利用率，同时满足应用更加灵活的资源动态分配需求。

简单来说，计算虚拟化使得在一台物理的服务器上可以运行多台虚拟机，虚拟机共享物理机的 CPU、内存、IO 硬件资源，但逻辑上虚拟机之间是相互隔离的。

== KVM 虚拟化

青云企业云平台的计算虚拟化采用 *KVM* 及 *QEMU* 等 Hypervisor 组件及技术，将通用的裸金属架构下的 x86/ARM 服务器资源进行抽象，以虚拟机的形式呈现给用户。

KVM 虚拟机包括虚拟内存、虚拟 CPU 和虚拟机 I/O 设备，其中 KVM 用于 CPU 及内存虚拟机，QEMU 用于I/O 虚拟化，通过 Linux 进程调度器实现虚拟机管理。

* KVM 属于 Linux Kernel 的一个模块，KVM 会复用部分 Linux 内核的能力，如进程管理调度、设备驱动，内存管理等。虚拟化平台可通过加载内核模块的方式启动 KVM，管理虚拟硬件的设备驱动，实现 CPU 与内存虚拟化，包括虚拟机创建，内存分配与管理、vCPU 执行模式切换等。
* QEMU 通过 ioctl 系统调用与 KVM 内核交互，KVM 需要加载 QEMU，实现 IO 虚拟化与各设备模拟（如磁盘、网卡、显卡、声卡等），然后由 KVM 内核与 QEMU 相互配合实现虚拟机的管理。
