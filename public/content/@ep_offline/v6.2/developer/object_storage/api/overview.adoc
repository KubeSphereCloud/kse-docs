对象存储服务提供的 API 符合 REST (Representational State Transfer) 风格所定义的语义，即：通过 URL 指定想要访问的资源，并使用标准的 HTTP 方法定义对资源的操作方式。

== 使用说明

对象存储 API 文档主要介绍接口的请求语法、相关参数含义以及请求和返回示例。如果您要进行二次开发，建议您使用 SDK 开发包。关于 SDK 的安装和使用，请参考link:../../sdk/[ SDK 文档]。

* 分段上传时，对象存储对 Object 的一个分段大小做出了限制。最小为 4M，最大为 5G。
* 单个 Object 上传时，对象存储对其上限做出了限制，最大为 5G。
* 自定义元数据是指所有前缀为 `x-qs-meta-` 的键值对。计算自定义元数据大小时，不含前缀 `x-qs-meta-`。

== 术语解释

[cols="1,1,3"]
|===
| 英文 | 中文 | 说明 

| Bucket | 存储桶 |用于存储对象的存储空间，所有的对象都必须隶属于某个存储桶。
| Object | 对象 | 存储数据的基本单元，也称为文件。
| Endpoint | 访问域名 |对象存储的访问域名。
| AccessKey | API 密钥 |对象存储通过使用对称加密的方法来验证某个请求的发送者身份。

|===
