WebInstaller 流程正常执行完成后，实施人员需对环境进行检查，确认当前的部署流程均正常执行完成，方能进行后续的初始化操作。详细内容如下。


== 后台程序确认

实施人员可以使用 `check.py` 脚本确认{product_name}的后台程序是否正常运行。

. 执行如下命令行，登录 FirstBox。其中 `<firstbox ip address>` 是安装 FirstBox 时配置的管理 IP，需根据实际情况进行修改。
+
[source,shell]
----
ssh root@<firstbox ip address>
----

. 执行以下命令，查看环境简略信息。`check.py` 更多使用方式，请执行 `check.py -h` 命令查看。
+
[source,shell]
----
/pitrix/check/check.py -n all -v
----
+
命令回显示例
+
image::/images/ep_offline/installation/deploy_check.png[,40%]
+
回显字段说明
+
[cols='1,2a']
|===
| 字段值 | 说明

| Hostname | 主机名称。
| Service | 服务名称。
| Result | 检查结果。

* `Normal` 或 `ok` 表示环境正常。
* `Ignored` 表示忽略检查的配置项。
* `Abnormal`、`Unknown` 以及 `error`  表示环境不正常，需要逐一排查并进行修复。
|===


. 执行以下命令，查看不正常项的详细信息。
+
[source,shell]
----
cat /pitrix/check/formatted_result | egrep 'Abnormal|Unknown|error'
----
+
* 若没有输出内容，表示环境正常运行。
* 若有输出内容，说明存在异常项，此时可执行如下命令行，重启不正常的服务，若仍然未恢复到正常状态，请逐一排查并进行修复。命令行中的 `<Hostname>` 与 `<Service>` 根据上一步查询结果进行填写。
+
[source,shell]
----
fb exec_nodes <Hostname> 'supervisorctl restart <Service>'
----
+
** 命令行示例一
+
重启所有 Webservice 节点上的 `alarm_server` 服务，则执行如下命令行。
+
[source,shell]
----
fb exec_nodes webservice 'supervisorctl status alarm_server'
----

** 命令行示例二
+
重启 `uat1a-webservice0` 节点上的 `alarm_server` 服务，则执行如下命令行。
+
[source,shell]
----
fb exec_nodes uat1a-webservice0 'supervisorctl status alarm_server'
----

== 删除测试域名

若当前部署的私有云环境不能访问互联网，此时，需人为删除 `server.yaml` 文件中的 `test_domains` 配置，否则 BOSS 会有 DNS 相关告警。操作步骤如下。

. 执行如下命令行，登录 FirstBox。其中 `<firstbox ip address>` 是安装 FirstBox 时配置的管理 IP，需根据实际情况进行修改。
+
[source,shell]
----
ssh root@<firstbox ip address>
----

. 执行以下命令行，修改 `server.yaml.<zone_id>` 文件。
+
[source,shell]
----
vi /pitrix/conf/variables/global/server.yaml.<zone_id>
----
+
删除如下内容。
+
[source,shell]
----
  test_domains:
    - 'www.baidu.com'
    - 'www.taobao.com'
----

. 执行以下命令，刷新 `server.yaml.<zone_id>` 到各节点。
+
[source,shell]
----
/pitrix/build/build_pkgs.py -p pitrix-global-conf
/pitrix/upgrade/update.sh all pitrix-global-conf
----

== 登录{product_name}

安装部署完成后，实施人员可登录{product_name}的 Console、BOSS 与配置中心进行相关检查与设置操作。


=== 配置 Hosts 文件

通过 Web 访问{product_name}的环境，需将{product_name}的 Host 信息添加至本地主机的 Hosts 文件中。可参考link:../standard/install/#_开始安装[前文内容]获取 Host 信息。


打开本地主机 的 hosts 文件，写入获取到的 Hosts 信息。

[source,shell]
----
// hosts 示例：
10.10.0.231 console.democloud.com appcenter.democloud.com api.democloud.com docs.democloud.com
10.10.0.211 cb0zcsf1.democloud.com
10.10.0.212 cb1zcsf1.democloud.com
10.10.0.231 boss.democloud.com
----

include::../../../../../_components/note.adoc[]

* Linux 系统 hosts 文件路径为 `/etc/hosts`。
* Windows 系统 hosts 文件路径为 `C:\Windows\System32\drivers\etc\hosts`。

include::../../../../../_components/table_footer.adoc[]
