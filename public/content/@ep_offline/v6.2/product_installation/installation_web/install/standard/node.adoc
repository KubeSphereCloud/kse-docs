节点规划阶段分两步，先收集各物理服务器信息，再对收集物理服务器进行校验与配置。确认操作步骤无误后，点击**下一步**。

image::/images/ep_offline/installation/webinstaller_08.png[webinstaller_08,100%]


== 收集节点

. 点击页面的**开始收集**，弹出**收集节点**对话框。
+
image::/images/ep_offline/installation/webinstaller_09.png[webinstaller_09,100%]

. 用户根据页面提示信息，填写相关参数。页面参数说明如下。
+
[cols='1,3a']
|===
| 参数 | 说明

| 登录节点 | 填写登录各物理服务器的用户名与密码。用于登录各物理服务器。

* 若安装操作系统时，使用的默认用户名密码，这里需选择 `默认用户名密码`，后台将使用 `yop/zhu1241jie` 来访问各物理服务器。
* 若安装操作系统时，实施人员自定义了用户名密码，这里需选择 `自定义用户名密码`，并根据前期安装操作系统时设置的用户名密码进行填写。
| IP 列表 | 填写待部署环境中，所有物理服务器的 IP 地址。

* 一行一个 IP 地址的格式进行填写。最少 5 个 IP 地址，最多 254 个 IP 地址。
* 物理服务器信息收集成功后，系统默认会将该列表中的前 3 个物理服务器作为 KS 节点。实施人员可以在收集完成后进行修改。
|===


. 点击**开始收集**。后台程序便开始收集各物理服务器信息。

. 节点信息收集完成后，弹出如下对话框，仔细查看对话框上的统计信息，符合预期后，可点击**校验并配置节点**。
+
image::/images/ep_offline/installation/webinstaller_10.png[webinstaller_10,100%]

== 调整节点配置

查看节点详情时，发现与硬件的实际情况不相符，或需调整管理节点与计算节点的分配时，实施人员可参考以下步骤。

include::../../../../../../_components/note.adoc[]

做完相应的调整后，需点击**保存**或**确认**，以免页面跳转，丢失修好后的信息。

include::../../../../../../_components/table_footer.adoc[]

=== 调整节点数量

. 可将 `管理节点` 按需调整为 `计算节点`。
+
image::/images/ep_offline/installation/webinstaller_14.png[webinstaller_14,100%]

. 可将 `计算节点` 按需调整为 `管理节点`。
+
image::/images/ep_offline/installation/webinstaller_15.png[webinstaller_15,100%]

. 不建议执行节点的**删除**操作，执行**删除**操作后的节点，如需重新加载，需要点击**计算节点**模块的**+收集节点**再次收集。**删除**操作通常是当该物理服务器不在预期内才需执行。


. 如需新增节点，可点击**计算节点**模块的**+收集节点**，弹出**收集节点**对话框，用户可参考link:#_收集节点[前文内容]操作。
+
include::../../../../../../_components/note.adoc[]

**收集节点**默认是将收集后的节点作为 `计算节点`，若不符合预期，实施人员可参考前文，将相应的 `计算节点` 调整为 `管理节点`。

include::../../../../../../_components/table_footer.adoc[]

=== 调整计算节点配置

计算节点可调整 `存储模式`，还可增加 `附加功能`。点击计算节点的**配置**，弹出**配置节点**对话框。

image::/images/ep_offline/installation/webinstaller_36.png[webinstaller_36,100%]

如上图所示，目前系统中支持的 `存储模式` 有以下几种。用户根据实际情况进行选择。默认为 `sanc` 模式。

* `sanc` 云盘存储，是指计算节点将使用 NeonSAN 作为底层存储。该存储模式具有低时延、高性能、持久性、高可靠等特性，可以随时创建或释放，也可以随时扩容。
* `local` 本地盘存储，是指计算节点将直接使用本地硬盘。该存储模式具有低时延、高随机 IOPS、高吞吐量、高性价比等优势，但存在单点故障风险，需要{product_name}用户自行做好数据冗余以保障可用性。
* `tpsc` 第三方存储，需在第一步**基本信息**中开启并配置 `远端存储`。
* `pair` 本地存储 SDS1.0，是指使用 DRBD 的存储方案，该存储模式要求计算节点必须成对出现，且为过时的存储方案，不建议选用。
* `repl` 本地存储 SDS2.0，是指使用 ZFS 文件存储方案。该存储模式为青云过时的存储方案，不建议选用。


如上图所示，目前系统中支持的 `附加功能` 有 `GPU 虚拟化` 与 `USB 透传`，用户根据实际情况进行选择。默认无附加功能。

* 选择开启 `GPU 虚拟化` 功能的物理服务器，必须配置有独立的 GPU 显卡。
* 选择开启 `USB 透传` 功能的物理服务器，其性能会受到一定影响。如需开启该功能，建议选择少量、性能较好的物理服务器。
