该部分参数用于配置各服务的部署方式，如使用物理机或虚拟机部署，以及各服务的 IP 地址、VIP 地址等信息。配置主页面如下图所示。

== 默认服务

默认服务均已内置相关配置。用户可根据需求，点击服务列表的**编辑**按钮，逐一确认各服务的安装信息是否符合预期，并进行相应的调整。

include::../../../../../../_components/note.adoc[]

* 当前页面上，各服务组件的部署方式，均有默认的推荐方案，若无特殊需求，建议实施人员无需修改。 
* 需部署多台的服务组件，无论虚拟机部署还是物理机部署，均建议选择不同的物理服务器，以达满足高可用。

include::../../../../../../_components/table_footer.adoc[]

[cols='1,3a']
|===
| 服务 | 说明

| 镜像服务 | 仅支持物理机部署。

* 需任选至少两台管理节点服务器，用于部署镜像服务。
* 部署完成后，需要导入镜像文件后才能在 Console 使用。
| 备份服务 | 包括 VBR 与 Snapshot 服务，仅支持物理机部署。

* 需任选至少两台管理节点服务器，用于部署备份服务。
* Snapshot 节点，用于储存快照及备份数据；VBR 节点用于创建云硬盘与云服务器的快照、备份以及回滚服务。

| 公用 DNS 服务 |仅支持虚拟机部署。

* 需任选两台管理节点服务器，用于启动部署 DNS 服务的虚拟机。
* 需设置两台虚拟机的 IP 地址。
** 虚拟机的 IP 地址支持 `自动获取` 与 `手动设置` 两种方式。
** 选择 `自动获取` 时，系统会自动从前期设置的的 `管理虚机 IP 范围` 中任选空闲 IP，作为该虚拟机的 IP 地址。
** 选择 `手动设置` 时，需人为从前期设置的 `管理虚机 IP 范围` 中，任选空闲 IP 地址输入。

* 需给两台虚拟机设置 VIP 地址，
** DNS 服务的 VIP 地址支持 `自动获取` 与 `手动设置` 两种方式。
** 选择 `自动获取` 时，系统会自动从前期设置的的 `组件浮动 IP 范围` 中任选空闲 IP，作为 DNS 服务的 VIP 地址。
** 选择 `手动设置` 时，需人为从前期设置的 `组件浮动 IP 范围` 中，任选空闲浮动 IP 地址输入。


| 路由推送服务 |仅支持虚拟机部署。简称 Rsserver 服务，使用 SDN 3.0 网络模式时，用于同步基础网络路由。

* Rsserver 服务需 4 台虚拟机部署。建议 4 台 Rsserver 虚拟机中，0 和 1 分别部署于不同的管理节点上，2 和 3 也分别部署于不同的管理节点上。
* 每台 Rsserver 虚拟机都需要设置 AS 号。AS 号支持 `自动获取` 与 `手动设置` 两种方式。
* 若 Rsserver 虚拟机的 AS 号选择 `手动设置` 的方式，在{product_name}部署完成后，还需要将 Rsserver 虚拟机的 AS 号设置到相应的交换机上。可参考link:../../check/#_交换机配置更新[ SDN 3.0 交换机配置]。

| 数据库服务 |支持物理机或虚拟机部署，是 Pgpool 与 Pgserver 服务的总称。

* 物理机部署时，Pgpool 与 Pgserver 服务需部署于相同的两台管理节点服务器上。虚拟机部署时，启动虚拟机的物理机也应保持相同。
* 虚拟机的 IP 地址支持 `自动获取` 与 `手动设置` 两种方式。
** 选择 `自动获取` 时，系统会自动从前期设置的的 `管理虚机 IP 范围` 中任选空闲 IP，作为该虚拟机的 IP 地址。
** 选择 `手动设置` 时，需人为从前期设置的 `管理虚机 IP 范围` 中，任选空闲 IP 地址输入。
* Pgpool 节点的 VIP 地址支持 `自动获取` 与 `手动设置` 两种方式。
** 选择 `自动获取` 时，系统会自动从前期设置的的 `组件浮动 IP 范围` 中任选空闲 IP，作为 Pgpool 服务的 VIP 地址。
** 选择 `手动设置` 时，需人为从前期设置的 `组件浮动 IP 范围` 中，任选空闲浮动 IP 地址输入。

| 中间件服务 |支持物理机或虚拟机部署，是 Zookeeper、Redis、Memcache、Cassandra 服务的总称。

* 物理机部署时，需选择三台管理节点用于部署以上中间件服务。
* 选择虚拟机部署时，可将三台虚拟机分布于不同的管理节点上。
* 虚拟机的 IP 地址支持 `自动获取` 与 `手动设置` 两种方式。
** 选择 `自动获取` 时，系统会自动从前期设置的的 `管理虚机 IP 范围` 中任选空闲 IP，作为该虚拟机的 IP 地址。
** 选择 `手动设置` 时，需人为从前期设置的 `管理虚机 IP 范围` 中，任选空闲 IP 地址输入。
* 虚拟机部署时，需配置节点的浮动 IP。浮动 IP 地址支持 `自动获取` 与 `手动设置` 两种方式。
** 选择 `自动获取` 时，系统会自动从前期设置的的 `组件浮动 IP 范围` 中任选空闲 IP，作为中间件服务服务的 VIP 地址。
** 选择 `手动设置` 时，需人为从前期设置的 `组件浮动 IP 范围` 中，任选空闲浮动 IP 地址输入。

| 全局门户服务 | 仅支持虚拟机部署。包含 Proxy 与 Webservice 服务。

* Proxy 与 Webservice 服务均需部署于两台虚拟机上。
* 虚拟机的 IP 地址支持 `自动获取` 与 `手动设置` 两种方式。
** 选择 `自动获取` 时，系统会自动从前期设置的的 `管理虚机 IP 范围` 中任选空闲 IP，作为该虚拟机的 IP 地址。
** 选择 `手动设置` 时，需人为从前期设置的 `管理虚机 IP 范围` 中，任选空闲 IP 地址输入。

* Proxy 与 Webservice 节点的 VIP 地址支持 `自动获取` 与 `手动设置` 两种方式。
** 选择 `自动获取` 时，系统会自动从前期设置的的 `组件浮动 IP 范围` 中任选空闲 IP，作为 Proxy 或 Webservice 服务的 VIP 地址。
** 选择 `手动设置` 时，需人为从前期设置的 `组件浮动 IP 范围` 中，任选空闲浮动 IP 地址输入。


| 统一运维门户 |简称 BOSS，仅支持虚拟机部署。

* 任选两台管理节点作为启动虚拟机的物理节点。
* 虚拟机的 IP 地址支持 `自动获取` 与 `手动设置` 两种方式。
** 选择 `自动获取` 时，系统会自动从前期设置的的 `管理虚机 IP 范围` 中任选空闲 IP，作为该虚拟机的 IP 地址。
** 选择 `手动设置` 时，需人为从前期设置的 `管理虚机 IP 范围` 中，任选空闲 IP 地址输入。

* VIP 地址支持 `自动获取` 与 `手动设置` 两种方式。
** 选择 `自动获取` 时，系统会自动从前期设置的的 `组件浮动 IP 范围` 中任选空闲 IP，作为 BOSS 服务的 VIP 地址。
** 选择 `手动设置` 时，需人为从前期设置的 `组件浮动 IP 范围` 中，任选空闲浮动 IP 地址输入。

| 监控服务 | Cloudsat、Cronus、Grafana 服务的总称。仅支持虚拟机部署。

* 虚拟机的 IP 地址支持 `自动获取` 与 `手动设置` 两种方式。
** 选择 `自动获取` 时，系统会自动从前期设置的的 `管理虚机 IP 范围` 中任选空闲 IP，作为该虚拟机的 IP 地址。
** 选择 `手动设置` 时，需人为从前期设置的 `管理虚机 IP 范围` 中，任选空闲 IP 地址输入。

* 若当前可用区与计划对接的全局可用区网络不通时，Grafana 服务需启用转发代理。若开启转发代理时，默认端口号为 `3000`，实施人员可根据需求进行修改。


| 安装信息管理 |  

* 简称 Installer，用于在 BOSS 平台显示和管理云平台的安装信息。
* 若当前可用区与计划对接的全局可用区网络不通时，该服务需启用转发代理。若开启转发代理时，默认端口号为 `9999`，实施人员可根据需求进行修改。

|===

== 可选服务

可选服务列表中的内容，需根据前期规划勾选是否部署。如需部署，可点击编辑，在弹出的对话框内填写相应参数。各服务说明信息如下。

[cols='1,3a']
|===
| 服务 | 说明

| 弹性 IP 控制器 | 简称 Eipctl，用于管理 H3C-VG 硬件交换机及其公网 IP。部署于两台虚拟机服务器上。

* 配置其虚拟机节点 IP 时，若选择 `手动设置` 时，需人为从前期设置的 `管理虚机 IP 范围` 中，任选空闲 IP 地址输入。
* 安装完成后需调试对接，对接时可按需调整修改配置。建议安装前就准备好正确的交换机回环地址。

| 访问鉴权管理 | 简称 IAM，为 Console 用户提供更安全地管控本账户下任意资源访问权限的服务，开启后仅安装其核心组件。

* 若当前环境为 Global Zone，则安装完成后还需进一步配置才能在 Console 使用。
* 若当前环境不为 Global Zone，则安装完成后需要先完成与 Global Zone 的对接后，再进一步配置才能在 Console 使用。

| 企业管理中心 |简称 VDC ，为 Console 用户提供基于企业组织结构的云上多级资源管理服务，依赖 IAM 服务。

* 若当前环境为 Global Zone 时，可选择安装。
* 启用后将同时安装 IAM 和 VDC 核心组件，安装完成后还需进一步配置才能在 Console 使用。

| 消费数据仓库 |简称 Warehouse，用于对消费信息进行统计、分析、展示。部署于 Global Zone 的两台虚拟机服务器上。

* 配置其虚拟机节点 IP 时，若选择 `手动设置` 时，需人为从前期设置的 `管理虚机 IP 范围` 中，任选空闲 IP 地址输入。

* 若当前环境为 Global Zone 时，Warehouse 的代理 IP 指向前文 `全局门户服务` 中的 Proxy 服务的浮动 IP，并指定代理端口。
* 若当前环境不为 Global Zone，只需要将 Warehouse 代理 IP 指向待对接的 Global Zone 的 `全局门户服务` 中的 Proxy 服务的浮动 IP，并指定代理端口。


| API 网关服务 |为 Console 用户提供 API 的创建、维护、发布、运行、下线等完整生命周期管理，开启后仅安装其管理服务。

* 若当前环境为 Global Zone 时，可选择安装。
* 安装完成后还需要导入相应 AppCenter 应用镜像并进一步配置才能在 Console 使用。

| 容器引擎 QKE |QKE 为 Console 用户提供快速构建 Kubernetes 集群并进行智能可视化管理的能力，开启后仅安装其 Apiserver 服务。

* 若当前环境为 Global Zone 时，可选择安装。
* 安装完成后还需要导入相应 AppCenter 应用镜像并进一步配置才能在 Console 使用。

| 高性能计算 | 简称 HPC，为 Console 用户提供快速构建超算集群以进行云上仿真测试，开启后仅安装其管理服务。部署于两台虚拟机服务器上。

* 配置其虚拟机节点 IP 时，若选择 `手动设置` 时，需人为从前期设置的 `管理虚机 IP 范围` 中，任选空闲 IP 地址输入。
* Global Zone 或非 Global Zone 均可选择安装。
* 安装完成后还需要导入相应 AppCenter 应用镜像并进一步配置才能在 Console 使用。

| 服务迁移中心 |简称 SMC，为 Console 用户提供迁移服务，将其它虚拟化平台的云主机系统及数据完整迁移至当前{product_name}。

* 部署于两台虚拟机服务器上。

* 配置其虚拟机节点 IP 时，若选择 `手动设置` 时，需人为从前期设置的 `管理虚机 IP 范围` 中，任选空闲 IP 地址输入。

| VMware 纳管 |  为 Console 用户提供统一管理 VMware vSphere 虚拟化平台的功能。部署于两台虚拟机服务器上。

* 配置其虚拟机节点 IP 时，若选择 `手动设置` 时，需人为从前期设置的 `管理虚机 IP 范围` 中，任选空闲 IP 地址输入。
* 若当前环境为 Global Zone 时，可选择安装。

|===
