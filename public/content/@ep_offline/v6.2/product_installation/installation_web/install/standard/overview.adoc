进入 WebInstaller 界面后，如下图所示，需选择部署模式，实施人员才能开启{product_name}安装部署之旅。后续操作，若无特殊说明，均在 WebInstaller 界面完成。

image::/images/ep_offline/installation/webinstaller_02.png[webinstaller_02,100%]

为简化部署流程，提升安装部署效率，WebInstaller 的安装部署流程中，界面程序做了相应的限制。详细内容如下。

== 部署模式

{product_name}推荐使用 `标准部署` 模式，`融合部署` 模式仅供测试或 POC 场景使用。

* `标准部署` 的可用区类型支持 `全局与计算`、`仅计算`，暂不支持 `仅全局`。
* `融合部署` 可用区类型仅支持 `全局与计算`，暂不支持 `仅计算`。可选服务仅支持 `warehouse`、`iam`、`apigateway`、`vdc` 以及 `qkeapi`。
* 若使用 SDN 网络模式时，`标准部署` 模式下仅支持 SDN3.0 版本，`融合模式` 模式下仅支持 SDN2.0 版本。



include::../../../../../../_components/note.adoc[]

* 可用区类型中提及到的 `全局` 是指全局服务 Global，部署有 Global 服务的 Zone，简称为 Global Zone。

* `全局与计算` 是指同时部署 Global 与 Sub 服务的 Zone，`仅计算` 是指部署 Sub 服务的 Zone，我们称之为 Sub Zone。

== 节点规划

* `标准部署` 一次最多支持 254 个物理节点服务器，`融合部署` 一次最多支持 10 个物理节点服务器。
* 收集节点信息时，系统默认选取列表前三个物理服务器为管理节点，其余服务器为计算节点，进入配置页面后支持调整。若为 `融合部署` 模式下，管理节点也会作为计算节点。
* 收集节点信息时，系统默认将所有计算节点的存储模式设置为 `sanc`，进入配置页面后，支持修改和调整。

== 服务组件

在link:../service[第四步：服务组件]配置的过程中，组件使用虚拟机部署时，所有用于部署各服务组件的虚机 IP 均取自于网络规划步骤中的管理虚机 IP 范围，所有浮动 IP 均取自于网络规划步骤中的浮动 IP 范围，这些 IP 均不得重复出现。

同时，用于部署同一服务组件的两台虚拟机同属于一台物理服务器时，也能成功部署与正常运行，但在后期使用过程中，可能会出现高可用的问题。
