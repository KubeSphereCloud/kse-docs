FirstBox 是平台安装节点，用于下发配置和安装包至各节点服务器。这里选择上传有安装包的节点服务器作为 FirstBox 的宿主机。

== 创建 FirstBox


. 登录上传有安装包的节点服务器。

. 执行如下命令行，开启 `tmux`。
+
[source,shell]
----
tmux
----
+
include::../../../../../_components/note.adoc[]

* 因为安装时间较长，若中途网络断连可能会导致操作终止。开启 `tmux` 后，在命令执行操作过程中，若网络断连，命令会在后台继续执行。恢复连接后，执行 `tmux attach` 即可恢复到之前的操作界面。
* 实施人员可根据需求，选择是否开启 tmux。

include::../../../../../_components/table_footer.adoc[]


. 执行以下命令行，安装 FirstBox。
+
[source,shell]
----
/root/qingcloud-installer/bootstrap/bootstrap_allinone.sh -m <mgmt_interface > -a <firstbox ip address> -p <pitrix_disk> -z <zone_id>
----
+
include::../../../../../_components/attention.adoc[]

* 该命令行执行过程较长，大约需要 30 分钟，请耐心等待。
* 命令行执行过程中，请勿强制中断安装进程，否则可能导致未知错误。

include::../../../../../_components/table_footer.adoc[]

+
**参数说明**
+
[cols="1,3,1"]
|===
| 参数 | 说明 | 是否必须

| -a a| * 指定 FirstBox 的管理 IP。
* 该 IP 需要和各节点服务器的管理网络处于同一个网段。
* 可以从前期规划的管理网络地址池中选择一个未被使用的 IP 地址分配给 FirstBox。 | 是

| -m a| 指定当前 KS 节点的管理网卡。请根据环境规划进行配置。

* 若网卡做了 Bond，此处应填写 Bond 名，如 `bond0`。
* 若网络使用 VLAN 模式 ，此处应填写 VLAN 名，如 `vlan100`。| 否
| -p a| * `/pitrix` 目录挂载盘。需填写系统盘盘符，不能填写数据盘盘符。
* 仅融合部署场景，需要该参数。
| 否
| -z a| 指定 Zone ID。根据前期规划进行填写。
| 是

|===
+
**命令行示例**
+
[source,shell]
----
/root/qingcloud-installer/bootstrap/bootstrap_allinone.sh -m ens3f1 -a 172.31.60.190
----


. 命令行执行完成，回显如下信息，表示 FirstBox 创建成功。
+
image::/images/ep_offline/installation/deploy_create_firstbox_02.png[deploy_create_firstbox_02,60%]
+

include::../../../../../_components/note.adoc[]
若命令行执行失败，请查看如下日志文件，检查报错原因并修复。若仍不能解决，请联系技术支持。

* KS 节点日志文件：`/root/enable_vt.log`、`/root/launch_fb.log`。
* FirstBox 节点日志文件：`/root/deploy.log`。

include::../../../../../_components/table_footer.adoc[]



== 登录 FirstBox

//FirstBox 成功创建后，各节点服务器的用户名与密码登录方式均被关闭。由于前期准备工作中，已经配置了本地主机与待部署环境的节点服务器之间的免密访问，因此，实施人员仍然能通过本地主机 SSH 至已创建的 FirstBox 节点。


具体操作步骤如下。


. 执行以下命令行，登录 FirstBox。其中，`<firstbox ip address>` 是link:#_创建_firstbox[创建 FirstBox] 时命令行中配置的管理 IP，需根据实际情况进行修改。
+
[source,shell]
----
ssh root@<firstbox ip address>
----

. 在 FirstBox 节点上，执行以下命令，检查各服务状态。
+
[source,shell]
----
supervisorctl status
----
+
回显信息如下，各服务状态均为 `RUNNING`，表示服务正常运行。
+
image::/images/ep_offline/installation/deploy_create_firstbox_03.png[]

. 在 FirstBox 节点上，执行以下命令，检查 WebInstaller 服务状态。
+
[source,shell]
----
service nginx status
----
+
回显信息如下，显示 `Active: active (running)`，表示 WebInstaller 服务正常运行，可继续后续操作。
+
image::/images/ep_offline/installation/webinstaller_01.png[webinsatller_01,100%]
