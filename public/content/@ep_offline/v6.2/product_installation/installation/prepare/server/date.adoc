---
title: 配置系统时间
weight: 6
---


为确保操作系统能正常运行，我们需要对各物理服务器进行时间同步的操作。

== 注意事项

* 若待部署环境已经规划了单独的 NTP 服务器，则无需执行此步骤，只需要在配置文件中填写 NTP 服务器地址即可。
* 若待部署环境没有规划单独的 NTP 服务器，在 Installer 部署过程中，系统会自动选择 VG 节点或 KS 节点作为时钟同步的服务器，并对其他的物理服务器和管理虚拟机做时间同步。因此，为确保环境时间的准确性，用户需手动配置 VG 节点或 KS 节点的时间。

== 操作步骤

. 根据前期规划内容，逐个登录 VG 或 KS 节点。

. 执行以下命令，查看服务器当前系统时间。
+
[source,shell]
----
date
----
. 若系统时间不正确，与实际时间有偏差，执行以下命令，修改服务器时间。
+
[source,shell]
----
date -s “yyyymmdd hh:mm:ss”
----
+
如修改系统时间为 `20220831 12:00:00`，则命令行如下。
+
[source,shell]
----
date -s “20220831 12:00:00”
----
. 执行以下命令，保存修改。
+
[source,shell]
----
hwclock --systohc
----

. 逐个登录 VG 或 KS 节点执行以上步骤，确保 VG 或 KS 节点均完成修改。
