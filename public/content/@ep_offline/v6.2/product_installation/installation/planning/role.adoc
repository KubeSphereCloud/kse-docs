根据各节点服务器的用途，{product_name}将其划分为多个不同的角色。包括 KS、Hyper、VBR、SNAPSHOT、以及 VG 节点。

//如下图所示：

//image::/images/ep_offline/installation/intro_architecture_01.jpg[,60%]

各角色节点服务器部署的服务以及承担的作用分别说明如下。

== KS 节点

KS（Kernel Service）节点，又称为管理节点，为{product_name}提供基础管理功能。该节点上部署有内部数据库、Web 界面、API 接口、租户信息管理、计费管理、BOSS 系统、VM 镜像库、VM 备份服务等。

KS 节点上运行的服务包括但不限于下表。

[cols="1,3a"]
|===
| 服务 | 描述 


| zoocassa	a| zoocassa 为一组服务的统称，包含如下 4 个服务：

* Zookeeper 服务：用于节点的集群管理。
* Cassandra 数据库：用于存放监控数据。
* Redis 数据库：用于计费服务缓存。
* Memcache 数据库：用于前端数据缓存。


| pgserver	| PostgreSQL 数据库，用于存放云平台管理信息。

| webservice	| WebConsole 服务。
| seed	| 镜像模板服务。
| pgpool	| 数据库连接池。
| boss	| BOSS 服务。
| cronus（influxdb）	| 监控组件数据库。
| warehouse	| 计费功能相关服务。如需使用计费功能，需部署该服务。
| hpc	| 超性能计算服务。
| smc	| 迁移中心。比如将虚拟机从 VMware 平台迁移到{product_name}。
| cloudsat	| 云监控服务。

|===

== VBR 节点

VBR 节点，又称为虚拟备份恢复节点，主要用于数据备份。可根据{product_name}的规模和客户备份数据量的大小来决定是否需要独立部署。



若无独立的节点服务器用于部署 VBR 与 Snapshot 服务，则可将其部署于 KS 节点上。




// [cols="1,3",options="header"]
// |===
// | 服务 | 描述 

// | snapshot	| 数据备份服务。
// | vbr	| 用于数据压缩。

// |===

== Hyper 节点

又称之为计算节点，为{product_name}提供计算、网络、存储等云资源。

== VG 节点

又称为虚拟网关节点，提供内部网络的基本安全保护，以及负载均衡器的流量转发控制。若无独立的 VG 节点，则以下服务可部署于 KS 节点上。


VG 节点上运行的服务包括但不限于下表。

[cols="1,3",options="header"]
|===
| 服务 | 描述 

| vgateway | 虚拟网关服务。
| proxy | Web Console 接入代理。
| dnsmaster | DNS 服务，为云平台提供内网域名服务。

|===



== 存储节点

可根据需求，在{product_name}中增加专门的存储节点，用于海量冷数据的低价存储。
