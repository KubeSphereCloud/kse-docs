QKE 对传统应用的支持很友好，可以在最大化减少用户的学习成本的前提下，提供完善的应用上云服务。

* 对于已有的应用，如果日志写到标准输出，可以直接被平台获取；如果写到本地的日志文件，则可以通过在平台上指定日志文件的目录来被平台获取。所有被平台获取的日志，用户均可以在日志搜索界面通过关键字、项目、应用名等方式搜索到。

* 对于习惯将 java 应用打包成 jar 或者 war 包的形式，平台提供 binary to image 的功能。用户只要上传 jar 包或者 war 包，平台会自动构造容器并部署。
