青云企业云平台不仅可以运行在 x86 架构的服务器上，同时还可以支持国产架构，鲲鹏、海光等，以及国产的操作系统，如麒麟、UOS 等。企业云用户可以根据自身业务特点灵活选择不同类型的云端资源，构建灵活、敏捷、高效的全模式业务系统，并实现统一管理和运维。

从物理硬件组成角度来看，企业云由多种角色的物理服务器节点构成，不同角色上部署不同的服务。节点之间通过交换机等网络设备互联，交换机均为 2 台堆叠在一起，形成高可用的网络架构。

image::/images/ep_offline/ep_twp/platform_physical_struc.png[]

根据服务器节点的用途及功能，服务器被划分为管理节点、计算节点、存储节点等角色。

== 管理节点

管理节点又称为 KS 节点，用于部署云平台的核心管理服务，如 ws、dns、ntp、pgsql 等，以及部署一些全局的基础功能或服务，如 BOSS 服务、监控服务、告警通知、账户管理和计费系统等。

为保证云平台的高可用，管理节点需部署至少 3 个。单节点故障时，不会影响平台的使用。由于管理与计算节点的分离架构，管理节点故障，不会影响计算节点上正在运行的业务。

管理节点上部署的管理服务大多为两两配对，即同一服务，分别部署于两个不同的物理节点上，因此扩容管理节点时，也需按照两两配对的方式进行。但 zoocassa 服务需部署于三个不同的物理节点上。

| 服务角色 | 说明

|proxy

|webservice
|提供 Webconsole 服务。包括云平台网关、账户、计费、消息中心、自动伸缩、定时器、全局资源锁等子服务。

| boss
| 承载 BOSS 平台相关服务。

|pgserver
| 云平台后台数据库服务，采用 PostgreSQL 数据库，用于存放云平台管理信息。

| zoocassa
a| zoocassa 为一组服务的统称，包含如下 4 个服务：

* Zookeeper 服务：用于节点的集群管理。
* Cassandra 数据库：用于存放监控数据。
* Redis 数据库：用于计费服务缓存。
* Memcache 数据库：用于前端数据缓存。

|dns
|为虚拟机提供 DNS 服务，负责云平台虚拟机的 IP 地址解析。

|eipctl
|提供公网地址控制器服务，用于向 VG 交换机同步配置信息及路由。

|swctl
|提供 Border 控制器服务，用于向 VPC Border 或物理 Border 交换机同步配置信息及路由。

|rs-server
|提供基础网络路由同步服务，用于将云平台内基础网络 IP 路由通过 BGP IPv4 传递给云平台外部物理交换机。

| cloudsat
| 提供云监控 CloudSat 服务。

|===

== 计算节点

计算节点又称为 Hyper 节点，为企业云平台提供 CPU、内存、网络、存储等云资源，主要用于运行虚拟机。

为了保证云平台的高可用，计算节点至少需部署 2 个，一个节点为另外一个节点的镜像节点（Mirror），

////

计算节点上运行的关键服务如下表所示。

[cols="1,3"]
|===
|服务名称 |说明

|compute_agent
|通过对端节点发送心跳，用以测试检查当前 Hyper 节点与其 Mirror 节点是否正常。需部署于所有 Hyper 节点上。


|compute_server
|用于处理运行于当前 Hyper 节点之上的虚拟机的相关操作，如虚拟机开关机操作、备份操作、虚机迁移、删除以及销毁等。需部署于所有 Hyper 节点上。

|compute_monitor
|对云平台虚拟资源进行监控，并将监控数据汇总存储到 Cassandra 数据库。需部署于所有 Hyper 节点上。

|repl_server
|用于处理 Hyper 节点与其 Mirror 节点之间的的数据同步，包括发送副本、接受副本等。需部署于所有 Hyper 节点上。

|===

== 存储节点

存储节点为整个企业云平台提供块存储服务。存储节点采用 NeonSAN 分布式块存储系统构建存储集群，支持计算与存储融合部署以及计算与存储独立部署，为保证更高的性能，建议采用独立部署的方式来部署 NeonSAN 的存储节点。

根据节点的存储介质的不同，可区分不同类型的云硬盘，去满足海量数据场景以及重 IO 应用等不同场景。

对象存储节点则是为企业云平台提供面向海量非结构化数据存储的对象存储服务，需要独立的物理设备进行部署。


== 备份节点

备份节点通常又称为 VBR 节点，主要用于处理数据备份。云租户在使用云平台提供的备份功能时，vbr 服务会拷贝云资源的快照数据，并进行压缩，最后备份归档至 snapshot 存放。

备份节点通常与管理节点融合部署，在备份容量明确且较大的情况下，需要单独部署。

备份角色节点上的主要服务组件如下表所示。为保证其高可用，该服务采用多活架构，需将其分别部署于两台节点上。

[cols="1,3"]
|===
|服务名称 |说明

|storage_vbr
|处理数据备份与恢复。 +
备份时，vbr 拷贝云资源的快照数据，并进行压缩，然后推送至 snapshot 进行存档；数据恢复时，vbr 从 snapshot 读取相关处理。

|snapshot
|用云存储云平台的数据备份。可以与 vbr 部署于同一节点，也可以单独设立节点。

|===

== 镜像节点

镜像节点又称为 Seed 节点，该角色提供云平台的虚拟机系统、网络组件（负载均衡器、VPC、NAT 网关）、虚拟存储（vSAN、vNAS）、Appcenter 等的镜像资源。

镜像节点通常与管理节点融合部署，在规模较大的情况下，需要单独部署。

////
运行于镜像节点的主要服务为 storage_server，提供镜像仓库服务。为保证其高可用，该服务采用多活架构，需将其分别部署于两台不同的节点上。
