云平台提供的网卡是基于虚拟机所在的物理设备，通过虚拟化技术模拟出的网络设备，服务于主机的网络连接能力，虚拟网络、公网 IP 地址、内网 IP 地址的配置都需要基于网卡资源。

== 网卡类型

网卡分为主网卡和从网卡：

* *主网卡*：在创建云服务器时，随云服务器默认创建的网卡称作主网卡。主网卡与云服务器机的⽣命周期⼀致，无法与云服务器进行解绑，云服务器被销毁时，主网卡即被销毁。一个云服务器只能绑定 1 张主网卡。
* *从网卡*：从网卡可手动申请，并分配给云服务器。从网卡独立于云服务器的生命周期，可以被连接到任意运行中的云服务器上。一个云服务器可以最多绑定 63 张从网卡。

== 网卡特性

* 一张网卡只能同时绑定到一台云服务器上，但一台云服务器可以同时绑定多张网卡。
* 主网卡与从网卡的网络类型互不影响，即当主网卡的网络类型是私有网络时，从网卡可以为相同或不同私有网络或基础网络。
* 主网卡不支持修改 IP 地址，从网卡可修改 IP 地址。
* ⽹卡具有地域属性，仅⽀持绑定⾄相同地域的云服务器。
* 每块⽹卡⽀持绑定⼀个安全组，提供⽹卡级别安全控制。

=== 网卡 API 支持

网卡提供以下 API 接口。

[cols="1,3"]
|===
|接口名称 |说明

|AttachNics | 将一块或多块“可用”（ available ）状态的网卡加载到某台“运行”（ running ） 状态的云服务器。

|CreateNics | 创建网卡。

|DeleteNics | 删除一个或多个网卡。 +
删除网卡前，需要先与云服务器解绑。


|DescribeNics | 获取一个或多个网卡的配置。

可根据 MAC 地址、名称、云服务器、网络、网络类型、状态来作为过滤条件获取网卡列表。 +
如果不指定任何过滤条件，默认返回你的所有网卡。

|DetachNics | 将一块或多块挂载状态的网卡从某台云服务器中卸载。

|ModifyNicAttributes | 修改网卡属性， 包括 IP 地址，vxnet_id 和网卡名称。
|===
