Web 应用防火墙（WAF），用于防护 SQL 注入、XSS、命令注入等各类 Web 攻击和威胁。通过结合机器学习、智能语义分析等优势技术，为用户的网站系统构建一道安全的屏障。

WAF 通过作为代理的角色来实现安全防护。

== 场景一：业务 IP 绑定在负载均衡器

image::/images/ep_offline/ep_twp/srp_waf_01.png[]


* *数据流向*：
+
这种场景中，外部访问的流量经过 EIP 到达负载均衡器，再由负载均衡器将流量送到 WAF 的前端网卡（nic.base）中，然后由 WAF 的后端网卡（nic.vxnet）送到云服务器。WAF 在此过程中充当代理和流量牵引的作用，出入的流量都需要经过 WAF，以达到防护作用。

+

include::../../../../../../../_components/note.adoc[]
WAF 至少需要两张网卡：一张在基础网络，用于接收负载均衡器送过来的流量；一张加入到需要防护的私有网络中，便于与受防护的 vm 通信。

include::../../../../../../../_components/table_footer.adoc[]

+
该场景下，WAF 的代理转发将会改变原有的网络拓扑，即增加了 WAF 的代理转发。代理转发的好处是有利于横向扩展，后续可以将流量转发到不同的私有网络中。

* *操作方法*：
+
. 将 WAF 实例加入到受防护 vm（云服务器）所在的 VPC 网络，让 WAF 与 vm 网络互通。
. 将负载均衡器中原有转发策略进行修改，将流量转发到 WAF 的前端网卡上。
. 在 WAF 控制台配置站点防护和代理模式，将流量通过后端网卡转发到 vm，实现南北向引流。

== 场景二：业务 IP 绑定在云服务器

image::/images/ep_offline/ep_twp/srp_waf_04.png[]


* *数据流向*：
+
外部访问的流量经过 EIP 直接到 WAF 中，再由 WAF 转发到 vm 中。

+
该场景下将 WAF 作为代理转发实现防护，会改变原有的网络拓扑。

* *操作方法*：
+
. 将 WAF 实例加入到 vm 所在的私有网络中，让 WAF 与 vm 网络互通。
. 将 EIP 从 vm 上解绑，并绑定到 WAF 的前端网卡上。
. 在 WAF 控制台配置站点防护，实现南北向引流。
