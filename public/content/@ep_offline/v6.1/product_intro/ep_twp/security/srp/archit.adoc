安全资源池作为一种类 PaaS 服务，云平台自动纳管安全实例，安全实例属于用户，能够在资源侧直接提供安全服务。

在安全资源池的实际应用场景中，整体系统架构由安全组件底座（对应一台云主机）、用户安全实例以及用户生产环境云主机组成，三者资源关系如下图所示。

image::/images/ep_offline/ep_twp/srp_archit.png[]


* *安全组件底座*
+
使用一个云账户（例如管理员账户admin@yunify.com#scs）来管理三方安全厂商的“安全组件底座”（如安恒云底座）。这台云主机需要接入云平台的基础网络。

* *用户安全实例*
+
用户会创建安全实例，默认加入基础网络，从而实现和“安全组件底座的云主机”二层或三层直接互通。

* *用户生产环境的云主机*
+
用户生产环境的云主机可以通过基础网络和安全实例通讯，也可以通过给安全实例加入私有网络实现私网二层直接通讯。
