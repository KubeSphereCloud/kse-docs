== 网络设备要求

* 要求最低使用万兆网络，接入交换机或汇聚交换机均为 2 台，用于配置高可用的网络架构。
* 所有网络链路均要求有冗余。各服务器均需双口网卡，且需将两个网口做 Bond，Bond 配置为 `mode1` 或 `mode4`，交换机端需支持 `mode1` 或 `mode4`。
* 网卡使用 Intel 82599 芯片系列、Mellanox CX4 /CX5 双口 10G 或 25G 网卡。





== 服务器要求



* KS-BM 节点必须配置双网卡，并配置两路网络，一路 MGMT 网络，一路 PXE 网络，默认网关在 MGMT 网络上。

* VG 节点必须配置双网卡，并配置两路网络，一路 MGMT 网络，一路 EIP 网络，默认网关在 EIP 网络上。
* 对于网络隔离模式为 VLAN 的场景，一定要确保所有 Hyper 节点的网卡名都一致且对应。
* 如果需要在 KS 节点上配置一路网络用于与个人笔记本互通，务必使用 `192.168.254.0/24` 这个网段。


== 磁盘要求

除 VG 节点外，所有节点服务器均需配有数据盘，其中 Seed、VBR 以及 Snapshot 这类冷存储节点上的数据盘空间要充足。

{product_name}支持使用机械盘，机械盘的性能影响着业务性能。为尽可能提升性能，特对机械盘做出如下要求。



* 硬件 RAID 是否自带缓存对机械盘的性能影响较大，要求使用带有缓存的硬件 RAID。

//因此，我们对 `RAID50` 与 `RAID10` 做如下要求。

// ** 使用 `RAID50` 时，必须使用带缓存的 `硬件 RAID`。
// ** 使用 `RAID10` 时，如果使用 `硬件 RAID`，则不要求带缓存。

* iCAS 对顺序读写使用直通模式，即 `seq-cutoff`，避免顺序读写占用缓存。

* 为提高缓存的利用率，需要缓存空间尽可能的大。所以在做 CAS 设备的时候，要避免将一块 SSD 磁盘分成多个区使用。正确的做法是将整块 SSD 磁盘做成共享的缓存设备，再分给两个 DRBD 使用。

* 最佳实践方案可参考：iCAS 缓存和 Core 存储之间的容量配比在 `1:10`。

// ** iCAS 缓存和 Core 存储之间的容量配比在 `1:10`。
// ** iCAS + SDS 1.0 + 带至少 `1G 缓存的硬件 RAID` + `RAID50`。
// ** iCAS + SDS 1.0 + `不带缓存的硬件 RAID` + `RAID10`。
