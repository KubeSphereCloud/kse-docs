待 FirstBox 成功创建后，需收集已准备好的各物理服务器的硬件环境、操作系统等各项信息，确认当前环境信息与实际情况一致。



执行如下命令行登录 FirstBox，其中 `<firstbox ip address>` 是安装 FirstBox 时配置的管理 IP，需根据实际情况进行修改。

[source,shell]
----
ssh root@<firstbox ip address>
----

include::../../../../../_components/note.adoc[]

若无特殊说明，后续操作均在 FirstBox 节点执行。

include::../../../../../_components/table_footer.adoc[]



== 创建 ip_list 文件

. 执行以下命令，在 FirstBox 的 `/root` 目录下创建 `ip_list` 文件。
+
[source,shell]
----
touch /root/ip_list
vim /root/ip_list
----

. 将待部署中所有物理服务器的 IP 地址，以每行一个 IP 地址的方式，写入 `ip_list` 文件。
+
[source,shell]
----
172.31.60.195
172.31.60.196
172.31.60.197
172.31.60.198
172.31.60.199
----


== 添加节点

. 执行以下命令，添加新节点并自动收集硬件信息。
+
[source,shell]
----
/pitrix/cli/add-nodes.py -I /root/ip_list
----

. 查看命令回显内容，若回显信息类似如下，表示添加成功。
+
image::/images/ep_offline/installation/deploy_collect_server_info_01.png[,60%]

. 执行以下命令，查看节点信息。
+
[source,shell]
----
/pitrix/cli/describe-nodes.py -s new
----
+
若回显信息中 `recv` 字段包含了所有已添加的节点，且返回类似如下信息，表示节点添加成功。
+
[source,shell]
----
  "ret_code": 0,
  "total_count": 5
----

== 配置免密访问



若安装操作系统时，物理服务器设置的用户名密码不是默认的 `yop/zhu1241jie`，则需执行如下命令行，手动建立各物理服务器之间的免密访问。

. 执行以下命令，通过脚本将密钥下发到所有物理服务器。
+
[source,shell]
----
/pitrix/node/collect/establish_ssh.sh /root/ip_list [SSH_PORT] [USER] [PASSWD]
----
+
**参数说明**
+
[cols='1,3']
|===
| 参数 | 说明

| SSH_PORT | 端口号，默认为 `22`。

| USER & PASSWD | 安装操作系统时设置的用户名和密码。
|===

+
**示例**
+
[source,shell]
----
/pitrix/node/collect/establish_ssh.sh /root/ip_list 22 yop zhu1241jietest
----

. 查看命令行回显信息，若回显内容如下，表示操作成功。
+
image::/images/ep_offline/installation/deploy_collect_server_info_02.png[,60%]
+

include::../../../../../_components/note.adoc[]
* 若命令行执行失败，请查看 FirstBox 节点上的日志文件 `/pitrix/log/node/establish_ssh.log`，检查报错原因并修复。

* 若仍不能解决，请联系技术支持。

include::../../../../../_components/table_footer.adoc[]

== 格式化数据盘

若物理服务器的数据盘有脏数据，可能会导致收集硬件信息的过程失败。因此，我们建议在收集硬件信息前对各物理服务器的数据盘进行格式化操作。

include::../../../../../_components/note.adoc[]

* 若确定当前环境中的磁盘数据均已清理，可以跳过此步骤。
* 若格式化磁盘的命令执行失败，请查看 FirstBox 节点上的日志文件 `/pitrix/log/node/erase_data_disks.log`，检查报错原因并修复。若仍不能解决，请联系技术支持。

include::../../../../../_components/table_footer.adoc[]


=== 单磁盘格式化

如需格式化指定 Hyper 节点上的数据盘，可执行以下命令行。

[source,shell]
----
/pitrix/bin/erase_data_disks.sh -i <hyper_ip> -d <disk_name>
----

**参数说明**

[cols='1,3']
|===
| 参数 | 说明

| -i | 待格式化的磁盘所在的物理服务器 IP。
| -d | 待格式化的磁盘盘符。
|===

**命令行示例**

[source,shell]
----
/pitrix/bin/erase_data_disks.sh -i 172.31.60.191 -d nvme0n1
----


=== 批量格式化

如需批量格式化若干 Hyper 节点上的数据盘，可执行如下操作。

include::../../../../../_components/note.adoc[]
批量格式化磁盘的操作，要求各物理服务器上的盘符一致。若各物理服务器上的盘符不一致，则只能使用link:#_单磁盘格式化[单磁盘格式化]的方式逐个物理服务器进行磁盘格式化操作。

include::../../../../../_components/table_footer.adoc[]

. 创建并编辑 `hyper_ip_list` 文件，填写待格式化数据盘的 Hyper 节点 IP，以每行一个 IP 地址的方式，写入 `hyper_ip_list` 文件。
+
[source,shell]
----
vim /root/hyper_ip_list
----
+
填写示例
+
[source,shell]
----
172.31.60.191
172.31.60.192
172.31.60.193
172.31.60.194
172.31.60.195
----


. 执行以下命令行，批量格式化数据盘。其中 `<disk_name>` 需根据实际情况进行填写。
+
[source,shell]
----
/pitrix/bin/erase_data_disks.sh -i /root/hyper_ip_list -d <disk_name>
----

. 若需查询该命令其他各参数及说明，可以执行如下命令进行查看。
+
[source,shell]
----
/pitrix/bin/erase_data_disks.sh -h
----





== 收集硬件信息

在安装{product_name}之前，该收集命令可以重复执行以修正硬件信息。安装{product_name}后，请勿再次执行该收集命令。

. 执行以下命令，下发收集硬件信息任务。
+
[source,shell]
----
/pitrix/cli/collect-nodes.py -I /root/ip_list
----
+
命令回显信息如下，表示收集任务下发成功。
+
image::/images/ep_offline/installation/deploy_collect_server_info_03.png[,60%]

. 执行以下命令，查看收集硬件信息任务当前的执行状态。
+
[source,shell]
----
/pitrix/cli/get-collect-nodes-status.py
----
+
回显信息如下，状态为 `successful` 时，表示收集任务执行成功。
+
image::/images/ep_offline/installation/deploy_collect_server_info_04.png[,60%]
+

include::../../../../../_components/note.adoc[]
* 若状态为 `step0`，表示任务正在执行中，请耐心等待，稍后重新执行该命令查看执行状态。

* 若状态异常，请查看相关日志，检查原因并修复。若仍不能解决，请联系技术支持。
** 节点详细日志文件路径：`/pitrix/log/node/collect.log`。
** 节点简略日志可执行命令 `/pitrix/cli/get-collect-nodes-log.py` 进行查看。


include::../../../../../_components/table_footer.adoc[]

== 校验硬件信息

待硬件信息收集完毕后，需查看确认各物理服务器信息是否正确。

. 可执行以下命令，获取所有物理服务器的详细信息。
+
[source,shell]
----
/pitrix/cli/describe-nodes.py
----
+
查看指定物理服务器的详细信息，可执行以下命令，其中 `<node_ip>` 为待查看的物理服务器 IP，需根据实际情况进行修改。
+
[source,shell]
----
/pitrix/cli/describe-nodes.py -i <node_ip>
----

. 确认收集到的物理服务器的信息是否正确。
+
* 是否包含所有待部署物理服务器的相关信息。
* 确认所有物理服务器的硬件信息，包括服务器名称、磁盘数量、磁盘大小、磁盘类型、CPU、内存大小是否与物理配置一致。

. 若硬件配置不符合预期，可以对硬件进行调整，调整后请重新执行以下命令行，再次收集信息。
+
[source,shell]
----
/pitrix/cli/collect-nodes.py -I /root/ip_list
----


. 若收集到了多余的磁盘或其他原因需要调整硬件信息，则可执行如下命令。详细操作请参见link:../../faq/other/command[常用命令 > 修改硬件信息]。
+
[source,shell]
----
/pitrix/cli/modify-node-attributes.py
----
+
include::../../../../../_components/note.adoc[]
使用该命令行修改硬件信息后，不能再次执行命令行 `/pitrix/cli/collect-nodes.py -I /root/ip_list`。因为执行 `/pitrix/cli/collect-nodes.py -I /root/ip_list` 命令行，会覆盖掉通过命令修改的内容。

include::../../../../../_components/table_footer.adoc[]


. 执行以下命令，查看各物理服务器的主机名，确认各主机名称满足link:../../prepare/server/hostname/#_命名要求[命名要求]。
+
[source,shell]
----
/pitrix/cli/describe-nodes.py | grep hostname
----
+
命令行回显内容如下，则说明主机名称正确配置。
+
image::/images/ep_offline/installation/deploy_collect_server_info_05.png[]
