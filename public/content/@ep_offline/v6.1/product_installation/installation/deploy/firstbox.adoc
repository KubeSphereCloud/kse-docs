== 创建 FirstBox

. 登录上传有安装包的物理服务器。

. 执行如下命令行，开启 `tmux`。
+
[source,shell]
----
tmux
----
+
include::../../../../../_components/note.adoc[]

* 因为安装时间较长，若中途网络断连可能会导致操作终止。开启 `tmux` 后，在命令执行操作过程中，若网络断连，命令会在后台继续执行。恢复连接后，执行 `tmux attach` 即可恢复到之前的操作界面。
* 实施人员可根据需求，选择是否开启 tmux。

include::../../../../../_components/table_footer.adoc[]


. 执行以下命令行，安装 FirstBox。
+
[source,shell]
----
/root/qingcloud-installer/bootstrap/bootstrap_allinone.sh -m <mgmt_interface > -a <firstbox ip address> -p <pitrix_disk>
----
+
include::../../../../../_components/attention.adoc[]

* 该命令行执行过程较长，大约需要 30 分钟，请耐心等待。
* 命令行执行过程中，请勿强制中断安装进程，否则可能导致未知错误。

include::../../../../../_components/table_footer.adoc[]

+
**参数说明**
+
[cols="1,3,1"]
|===
| 参数 | 说明 | 是否必须

| -a a| * 指定 FirstBox 的管理 IP。
* 该 IP 需要和各节点服务器的管理网络处于同一个网段。
* 可以从前期规划的管理网络地址池中选择一个未被使用的 IP 地址分配给 FirstBox。 | 是

| -m a| 指定当前 KS 节点的管理网卡。请根据环境规划进行配置。| 否
| -p a| * `/pitrix` 目录挂载盘。需填写系统盘盘符，不能填写数据盘盘符。
* 仅融合部署场景，需要该参数。
| 否

|===
+
**命令行示例**
+
[source,shell]
----
/root/qingcloud-installer/bootstrap/bootstrap_allinone.sh -m ens3f1 -a 172.31.60.190
----


. 命令行执行完成，回显如下信息，表示 FirstBox 创建成功。
+
image::/images/ep_offline/installation/deploy_create_firstbox_02.png[deploy_create_firstbox_02,100%]
+

include::../../../../../_components/note.adoc[]
若命令行执行失败，请查看如下日志文件，检查报错原因并修复。若仍不能解决，请联系技术支持。

* KS 节点日志文件：`/root/enable_vt.log`、`/root/launch_fb.log`。
* FirstBox 节点日志文件：`/root/deploy.log`。

include::../../../../../_components/table_footer.adoc[]



== 登录 FirstBox

. 执行以下命令行，登录 FirstBox。其中，`<firstbox ip address>` 是link:#_创建_firstbox[创建 FirstBox] 时命令行中配置的管理 IP，需根据实际情况进行修改。
+
[source,shell]
----
ssh root@<firstbox ip address>
----

. 在 FirstBox 节点上，执行以下命令，检查各服务状态。
+
[source,shell]
----
supervisorctl status
----
+
回显信息如下，各服务状态均为 `RUNNING`，表示服务正常运行，可继续执行后续操作。
+
image::/images/ep_offline/installation/deploy_create_firstbox_03.png[]
