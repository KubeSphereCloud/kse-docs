`qke_apiserver` 服务正常运行的前提下，可执行以下步骤对其进行配置。



== 步骤一：配置数据库信息

添加 QKE 数据库的相关配置信息。详细操作如下。

. 登录{product_name} Global Zone 的 FirstBox 节点。
+
include::../../../../../_components/note.adoc[]

登录方式可咨询{product_name}安装部署人员。

include::../../../../../_components/table_footer.adoc[]

. 在 Global Zone 的 FirstBox 节点上，执行如下命令行，查看配置。命令行中的 `<zone_id>` 为 Global Zone 的 ID，需根据实际情况进行修改。
+
[source,shell]
----
vim /pitrix/conf/variables/global/postgresql.yaml.<zone_id>
----

. 追加如下图所示的 `qke_plus` 配置内容。
+
image::/images/ep_offline/qke_install/qke_install_16.png[]
+
include::../../../../../_components/note.adoc[]

* 配置参数的值，可参考本文件中其他模块的配置内容填写。

* 若当前配置文件中，已有上述内容，则继续link:#_步骤二创建数据库[步骤二]。

include::../../../../../_components/table_footer.adoc[]


. 执行以下命令行，生成配置文件内容。
+
[source,shell]
----
fb build_pkgs -p pitrix-global-conf
----

. 执行以下命令行，下发配置文件内容。
+
[source,shell]
----
fb update all pitrix-global-conf
----


== 步骤二：插入初始化数据

在数据库中插入 QKE 的初始化数据，包括 Zone 信息，主机规格信息以及 QKE 的版本列表。详细操作如下。


. 登录{product_name} Global Zone 的 FirstBox 节点。
+
include::../../../../../_components/note.adoc[]

登录方式可咨询{product_name}安装部署人员。

include::../../../../../_components/table_footer.adoc[]


. 执行如下命令行，查看{product_name}的 Webservice 节点信息。
+
[source,shell]
----
cat /pitrix/conf/nodes/webservice
----
+
命令回显示例
+
[source,shell]
----
nodes=( testbm1a-webservice0 testbm1a-webservice1 testbm2a-webservice0 testbm2a-webservice1 );
----

. 根据查询结果，登录 Webservice 节点。这里以登录 `testbm1a-webservice0` 节点为例。
+
[source,shell]
----
ssh testbm1a-webservice0
----

. 在 Global Zone 的 Webservice 节点上，执行如下命令行，进入脚本所在目录。
+
[source,shell]
----
cd /pitrix/lib/pitrix-scripts/qke
----

. 执行如下命令行，修改配置文件 `config.yaml`，详细内容可参考link:#_config_yaml_示例[附录内容]。
+
[source,shell]
----
vi conf/config.yaml
----
+
参数填写说明
+
[cols='1,1,3a']
|===
| 参数 | 详细参数 | 说明 

| regions | -
| 按照格式填写当前 Global 下所有的 Region、Zone 信息。


.5+^.^| roles

| node_bm | QKE v3.0 新增参数，配置用户通过 Console 创建 QKE 集群时，其 `工作节点-BM` 的主机规格和磁盘规格。

. 正确修改配置文件 `config.yaml` 后，执行以下命令行，确认上述配置生成的 SQL 是否符合预期。
+
[source,shell]
----
./qke -e ./conf/config.yaml
----
+
include::../../../../../_components/note.adoc[]

若配置内容不符合预期，可重复修改配置文件 `config.yaml`。

+
[source,shell]
----
./qke -e ./conf/config.yaml -d 0
----
+

[source,shell]

== 步骤三：启动 qke_apiserver

配置 `qke_apiserver` 服务并启动。


. 登录{product_name} Global Zone 的 FirstBox 节点。
+
include::../../../../../_components/note.adoc[]

登录方式可咨询{product_name}安装部署人员。

include::../../../../../_components/table_footer.adoc[]


. 执行如下命令行，查看{product_name}的 Webservice 节点信息。
+
[source,shell]
----
cat /pitrix/conf/nodes/webservice
----
+
命令回显示例
+
[source,shell]
----
nodes=( testbm1a-webservice0 testbm1a-webservice1 testbm2a-webservice0 testbm2a-webservice1 );
----

. 根据查询结果，登录 Webservice 节点。这里以登录 `testbm1a-webservice0` 节点为例。
+
[source,shell]
----
ssh testbm1a-webservice0
----

. 在 Global Zone 的 Webservice 节点上，新增配置文件 `/pitrix/conf/qke_apiserver.yaml`。配置文件示例参考link:#_qke_apiserver_yaml_示例[附录]。
+
[source,shell]
----
vim /pitrix/conf/qke_apiserver.yaml
----
+
**参数填写说明**
+
* 示例文件中的 `<zone_id>` 为当前{product_name}的 Global Zone ID，需根据实际情况填写。
* 参数 `csi_qingcloud.volume_type` 用于设置 QKE 集群内 Kubernetes PVC 的磁盘类型，需根据当前{product_name}的环境进行配置。
+
include::../../../../../_components/attention.adoc[]

* 若当前{product_name}使用全闪的 NeonSAN 存储，则将 `csi_qingcloud.volume_type` 配置为 `5`。
* 若当前{product_name}使用混插的 NeonSAN 存储，则将 `csi_qingcloud.volume_type` 配置为 `6`。
* 更多磁盘类型请参见link:../../faq/faq_1[支持的主机类型和硬盘类型]。

//本地存储只能挂载到同类型的主机上。

include::../../../../../_components/table_footer.adoc[]


. 执行如下命令行，启动 `qke_apiserver` 服务。
+
[source,shell]
----
supervisorctl restart qke_apiserver
----


== 附录

=== `./qke` 命令行

* 查看生成的 SQL 语句是否符合预期
+
[source,shell]
----
./qke -e ./conf/config.yaml
----
+
仅打印配置文件生成的 SQL 语句，并不执行 SQL 语句。

* 插入数据库
+
[source,shell]
----
./qke -e ./conf/config.yaml -d 0
----
+
该命令行会执行 SQL 语句，并最终插入数据库。

* ARM 版本的资源包插入数据库
+
[source,shell]
----
/qke -e ./conf/config.yaml -d 0 -a 1
----

* 只插入数据库版本
+
[source,shell]
----
./qke -e ./conf/config.yaml --upgrade=true -d 0
----

* 清空初始化数据
+
[source,shell]
----
./qke -e ./conf/config.yaml -c logic
----
+
已执行过插入数据的操作。如需修改配置文件内容，可清空数据，再次重新导入。`-c` 支持两个参数。

** `logic` 表示逻辑删除，将数据库中相应数据的 `delete` 字段更新为 `true`。
** `physical` 表示物理删除，直接删除数据。

* 清空初始化数据
+
[source,shell]
----
./qke -e ./conf/config.yaml -c logic -d 0
----
+
`-d 0` 表示删除数据操作，不以 `dryrun` 模式运行。




=== qke_apiserver.yaml 示例


[source,yaml]
----
database:
  migrate: false
  name: "qke_plus"
  port: 5432

server:
  addr: "0.0.0.0:9102"
  workers: 3

log:
  level: "debug"
  path: "/pitrix/log"

appcenter:
  url: "tcp://<zone_id>-proxy:8565"
  ws_url: "tcp://<zone_id>-proxy:9365"
  ws_enable: true
  console_id: "admin"
  role: "partner"
  privilege: 10
  channel: "session"

#开启鉴权之后需要写
iaasapi:
  ak: "<access_key_id>"
  sk: "<access_secret_key>"
  endpoint: "<site_protocol>://api.<domain>/iaas"

# 用于多副本选主。
zk:
  url: "<zone_id>-zoocassa:2181"
  root: "/qke_plus"
  master: "/master"
  enable: true

# 托管版参数，表示是否需要根据 worker 节点的数量自动调整 master 节点的规格，默认为 false。
auto_resize:
  enable: false

# 是否开启 U10000 存储插件支持，具体的参数(confd_addrs,qfs_gateway,global_domain)咨询 U10000
csi_hydrofs:
  enable: false
  confd_addrs: "172.22.55.2:9296,172.22.55.4:9296,172.22.55.3:9296"
  qfs_gateway: "172.22.55.2:8180,172.22.55.3:8180,172.22.55.4:8180"
  hydrofs_csi_image: "docker.io/qingcloud/hydrofs-csi-driver:1.0"
  hydro_fuse_image: "docker.io/qingcloud/hydro_fuse:1.1"

# 控制 PVC 磁盘类型
csi_qingcloud:
  volume_type: "6"
----
