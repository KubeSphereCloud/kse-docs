在创建 QKE 集群时，可以自定义节点配置，系统默认提供如下节点规格供用户选择。

image::/images/ep_offline/qke_install/faq_04.png[,70%]

若默认的规格不满足需求，用户也可以执行以下步骤，新增节点规格。

== 操作步骤

. 登录{product_name}任一 Pgserver 节点。
. 执行以下命令，连接 qke_plus 数据库。
+
[source,shell]
----
psql -d qke_plus
----

. 插入一条新的节点规格数据。
+
如下示例，表示增加一个 24 核 96G 的企业型节点规格。
+
[source,sql]
----
INSERT INTO public.qke_db_spec (name, instance_class, instance_type, cpu, cpu_model, memory, volume_class, volume_type, volume_size, gpu, gpu_class, zone, role, os_volume_class, os_volume_size, deleted) VALUES
('ec3.6xlarge.r4', 203, 'ec3.6xlarge.r4', 24, NULL, 98304, 200, 200, 80, 0, 0, 'pek3', 'node_super_perf', 200, 80, false);
----
+
详细参数说明如下：
+
[cols='1,3'] 
|===
| 参数 | 说明 

| name a| 表示节点的规格名称。

下面以 ec3.6xlarge.r4 为例，介绍命名规则：

* ec3 表示企业型。
+ 
请根据实际情况进行修改，例如，s1 表示基础型。
* 6xlarge 表示 CPU 是 24 核（6*4=24）。
+
如果是 8 核 CPU，那么此处就是 2xlarge（8/4=2）。
* r4 表示内存是 96G（4*cpu=96）。
+
r4 中的 4 为 memory/cpu 的值，例如内存是 8G，CPU 是 2 核，那么此处也是 r4（memory/cpu=4）。

| instance_class | 表示节点的主机类型。
| instance_type | 与 name 保持一致。
| cpu | CPU 的核数。
| cpu_model | 默认为空。
| memory | 内存大小。
| volume_class 和 volume_type | 表示磁盘类型，例如 6。
| volume_size | 表示主机默认数据盘大小。
| gpu | 大于 0 表示该规格为 GPU 规格。
| gpu_class | GPU 主机规格。
| zone | 此操作作用的区域。
| role a| 节点角色。

* master：表示为 Master 节点（主节点）增加一个规格。
* node_pref：表示为基础型工作节点增加一个规格。
* node_super_pref：表示为企业型工作节点增加一个规格。
|===
