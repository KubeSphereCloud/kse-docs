== 网络性能

=== Iperf 工具

{product_name}的网络性能测试，使用 Iperf 工具。

Iperf 是一个网络性能测试工具，它可以测试 TCP 和 UDP 的带宽性能，可以报告带宽，延迟抖动和数据包丢失。并且具有多种参数和 UDP 特性，用户可以根据需求进行调整。

Iperf 的使用，可参考link:https://iperf.fr/iperf-doc.php#doc[文档]。

image::/images/ep_offline/delivery_guidelines/iperf_server.png[iperf_server,40%]

稳态下的带宽为 500M。

== 存储性能

=== 概述

存储性能有三个指标，如：IOPS、吞吐量以及访问时延。

* IOPS
+
IOPS：I/Os Per Second，即每秒输入输出次数，是指单位时间内设备能处理的 I/O 请求数量，一般以每秒处理的 I/O 请求数量为单位，是衡量磁盘性能的主要指标之一。
+
常用的 IOPS 指标包括顺序操作和随机操作，如下所示。
+
[cols="1,3", options="header"]
|===
| 指标 | 描述

|总 IOPS
|每秒执行的 I/O 操作总次数

|随机读 IOPS
|每秒执行的随机读 I/O 操作的平均次数

|随机写 IOPS
|每秒执行的随机写 I/O 操作的平均次数

|顺序读 IOPS
|每秒执行的顺序读 I/O 操作的平均次数

|顺序写 IOPS
|每秒执行的顺序写 I/O 操作的平均次数

|===


* 吞吐量
+
吞吐量 Throughput，是指单位时间内可以成功传输的数据数量，单位为 MB/s。如果知道每个 I/O 的平均大小，以及每秒钟的 I/O 数量，那么就可以算出这些 I/O 总共产生了多大的数据量，即带宽：Bandwidth。
+
因此，IOPS * I/O size = Bandwidth。

* 访问时延
+
访问时延是处理一个 I/O 需要的时间，单位为 s、ms 或者 μs。过高的时延会导致应用性能下降或报错。

* 总结
+
在测试过程中，IOPS 数值会受到很多因素的影响，包括 I/O 负载特征，如：读写比例，顺序和随机，工作线程数，队列深度，数据记录大小等，以及硬件配置，如：系统配置、操作系统、磁盘驱动等等。因此对比 IOPS 时，必须在同样的测试基准下进行，即便如此也会产生一定的随机不确定性。
