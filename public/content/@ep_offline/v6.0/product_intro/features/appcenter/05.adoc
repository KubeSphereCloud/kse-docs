== 集群管理列表重构

AppCenter 集群管理子 portal 重构，提升用户交互体验，前端架构及设计组件更新，以便于更好地进行功能优化及扩展。

本次集群列表功能重构主要在设计与前端代码重构，重构后集群菜单整合分类，便于用户集群管理。对集群功能也进行了样式重构，采用云平台最新交互方式重构了集群设置、集群运维、集群费用管理、集群标签与项目管理等。

== 集群前后端配置解耦

集群部署的时候，主机类型、硬盘类型、匹配关系等在前后端都有配置，新增或修改需要同时修改，修改成本高、维护易出错。解耦后部分配置由后端接口处理，例如：

* 使用 appcenter_config 存储配置信息
* 前端通过 describe_appcenter_config 接口查询配置信息（传入 keys）
* 平台里面不支持的配置，返回默认类型，如数据：[]，对象：{}
* 已经支持的配置 SUPPORTED_APP_VOLUME_INSTANCE_PAIR（硬盘匹配主机关系）

== 集群监控重构

本次集群管理重构不包括集群详情页重构，但集群监控对新老监控进行了兼容，新监控以自定义监控数据为主，应用如果对接了自定义监控项，则集群监控数据既有新监控同时可以切换到老监控，保障开发阶段新老监控数据完整。
