== 场景一：主机维度引流

主机维度引流可以在不改变现有网络拓扑结构的情况下，将流经云服务器（图中 vm）的所有流量牵引到防火墙（图中 ngfw）中，达到南北向和东西向安全防护作用。

image::/images/ep_offline/ep_twp/srp_fw_03.png[]

*数据流向*：

Internet 或者其他网络（不限制内网外网）的访问流量通过云服务器的通信网卡，再由引流网卡牵引进入防火墙，流经防火墙再沿引流网卡返回给云服务器处理，处理完成后通过通信网卡返回数据达到 Internet 或者其他网络。

include::../../../../../../../../_components/note.adoc[]
* 云服务器至少需要两张网卡，一张通信网卡，一张引流网卡。
* 通信网卡可以是绑定公网 IP 与外部通信的网卡，也可以是与 VPC 连通的外部网卡，取决于需要保护的云服务器所在位置；引流网卡的作用是将通往通信网卡的所有流量牵引到防火墙中。

== 场景二：EIP 维度引流

EIP 维度引流也是不改变现有的网络拓扑结构的前提下，将流经 EIP 的所有流量通过 MCN 牵引到防火墙（图中 ngfw）中，以达到南北向安全防护作用。

image::/images/ep_offline/ep_twp/srp_fw_04.png[]

*数据流向*：

** 未引流前，云服务器绑定公网 IP，直接与 Internet 外网达成联通。
** 引入 MCN（EIP 流量牵引服务）后，可以将 EIP 的流量全部引流到防火墙，流经防火墙再返回给 MCN，最后引流到 EIP 绑定的云服务器中。
