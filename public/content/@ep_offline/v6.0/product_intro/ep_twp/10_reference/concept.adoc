本文对文档中涉及的云平台业务名词进行解释。业务名词是青云为云平台相关的硬件、软件、服务、组件或业务概念等自定义命名的词汇，通常不区分大小写，均表示同一含义，本文按照常用写法进行罗列。

////
== A

* *ASW*
+
Access Switch，接入交换机，一般指与服务器直接连接的交换机。

* *安置组*
+
place_group

== B

* *BOSS*
+
Business & Operation Support System，简称 BOSS，是青云企业云平台的运维管理系统，提供了统一运维门户和统一的基础运维功能。

== C

* *Console*
+
控制台，用于指代企业云管理控制台，为云租户提供云资源、云产品或服务。

* *CSW*
+
Core Switch，核心交换机，一般指云平台出口交换机。

== D

* *DVR*
+
分布式虚拟路由器，


== F

* *fg*
+
FrontGate，前置网关服务，Zone 的消息入口，zone 和 global 消息交互的唯一通道，主要功能为用户界面和 API 服务的提供接口。

== G

* *Global*

== H

* *Hyper*
+
从 Hypervisor 演变而来，一般用于指代计算节点。

== K

* *KS*
+
Kernel Service，内核服务，云平台的核心服务进程。是一组服务的统称。KS 又包含 Global KS 与 Zone KS，分别为全局核心管理服务和可用区级别的核心管理服务。
+
若是表示服务器角色，KS 节点用于指代管理节点。

== N

* *NeonSAN*

== R

* *rs-server*
+
基础网络路由同步节点。

* *Region*
+
区域，指数据中心或云平台资源所在的地理区域，通常按照数据中心所在的城市划分。例如，“北京3区”表示数据中心所在城市是北京。

== S

* *seed*
+
指代镜像模板服务。用于存放和管理镜像模板。

* *SG*
+
Security Group，安全组。


== V

* *VBR*
+
Veeam Backup & Replication，虚拟机备份，通常使用 VBR 节点来指代备份节点。

* *VG*
+
Virtual Gateway，虚拟网关，是虚拟主机通过 EIP 与公网交互数据的网关，实现云平台内网与外网（企业网络）的连接。VG 可以是一个物理服务器，也可以是一个虚拟机，也可以是网络设备。
+
企业云当前采用 NOVG 方案，即复用已有的网络交换机设备，通过 BGP IPv4/IPv6 来进行 EIP 流量牵引，不需要对 EIP 流量进行 vxlan 封装，而是直接通过三层网络进行转发。


* *VM*
+
Virtual Machine，虚拟机，通常用于指代云服务器。

== W

* *ws*
+
webservice 或 ws_server 的简称，是云平台的 API 网关服务，Global 服务入口，提供参数检查、访问控制、请求转换和转发，并提供工单、全局配额、通知列表、消息中心等全局资源的业务处理。

== Z

* *Zone*
+
可用区，是指在同一区域内，电力和网络互相独立的物理区域。在同一区域（Region）内，可用区与可用区之间内网互通。各可用区之间可以实现故障隔离，即如果一个可用区出现故障，则不会影响其他可用区的正常运行。

|===
| 名词/缩略语 | 说明 

|ASW
|Access Switch，接入交换机，一般指与服务器直接连接的交换机。

|AppCenter
|青云自主研发的应用交付与运营平台，为用户提供了在云平台中开发、部署、购买、管理各类应用的一站式服务。

|BOSS
|Business & Operation Support System，简称 BOSS，是青云企业云平台的运维管理系统，提供了统一运维门户和统一的基础运维功能。

|EIP
|Elastic IP，通常用于指代“弹性公网 EIP” 服务或弹性公网 IP 地址。

|bot
|机器人程序，部署在每个计算节点，提供数据采集、计算和分析功能，可以 7x24 小时监控着云中所有资源，为调度决策提供数据依据。

|Console
|控制台，用于指代企业云的用户管理控制台，为云租户提供云资源、云产品或服务。

|CFW
|Cloud Firewall，云防火墙，是云平台提供的一款网络安全产品，用于防护进出公网 IP 的网络攻击。

|CloudSat
|全称为“云监控 CloudSat”，是云平台提供的一项针对云服务产品资源进行实时监控及告警的服务。


|CSW
|Core Switch，核心交换机，一般指云平台出口交换机。

|DVR
|Distributed Virtual Router，分布式虚拟路由器，是企业云平台 SDN 数据平面的核心组件。 +
每个 DVR 管理着所在计算节点包含的网络中的所有 VM，为 vxnet 提供了数据包二层交换和三层路由能力。

|fg
|FrontGate，前置网关服务，Zone 的消息入口，zone 和 global 消息交互的唯一通道，主要功能为用户界面和 API 服务的提供接口。

|Global
|全局区域，是逻辑上划分的区，非真实存在，部署于 Global 上的服务主要负责云平台的全局管理以及处理全局区域级别的请求。

| Hyper
| 由 Hypervisor 演变而来，一般用于指代计算节点。

| KS
| Kernel Service，内核服务，云平台的核心服务进程，是一组服务的统称。 +
KS 包含 Global KS 与 Zone KS，分别指全局核心管理服务和可用区级别的核心管理服务。 +

若是表示服务器角色，KS 节点用于指代管理节点。

|LB
|Load Balancer，负载均衡，通常用来表示负载均衡服务或负载均衡器。

|MCN
|青云自研的多云网络管理平台，是为了构建混合云，让不同云厂商的网络能够通过 VLAN/VXLAN/BGP 模式接入 MCN，达到不同云网络之间互通的目的。

|NeonSAN
|QingStor NeonSAN，青云科技自主研发的企业级分布式块存储系统。

|Region
|区域，指数据中心或云平台资源所在的地理区域，通常按照数据中心所在的城市划分。例如，“北京3区”表示数据中心所在城市是北京。

|Seed
|指代镜像模板服务。用于存放和管理镜像模板。

|SG
|Security Group，安全组。


|VBC

|Virtual Basic Cloud，虚拟基础网络，通常简称“基础网络”，是指云平台统一规划和管理的网络，云租户只能使用，无法自主管理。

|VBR
|Veeam Backup & Replication，虚拟机备份，通常使用 VBR 节点来指代备份节点。

|VG
|Virtual Gateway，虚拟网关，是虚拟主机通过 EIP 与公网交互数据的网关，实现云平台内网与外网（企业网络）的连接。VG 可以是一个物理服务器，也可以是一个虚拟机，也可以是网络设备。

企业云当前采用 NOVG 方案，即复用已有的网络交换机设备，通过 BGP IPv4/IPv6 来进行 EIP 流量牵引，不需要对 EIP 流量进行 vxlan 封装，而是直接通过三层网络进行转发。

|VM
|Virtual Machine，虚拟机，通常用于指代云服务器。

|VPC
|Virtual Private Cloud，虚拟私有网络，又称 “VPC 网络”，是云平台为云租户提供的专属私有网络，不同 VPC 之间逻辑隔离。在 VPC 网络内，用户可以自定义 IP 地址段、路由表、安全策略等，灵活管理属于自己的云上网络。

|vxnet
|私有网络，一个 vxnet 表示 VPC 网络下的一个子网。

|ws
| webservice 或 ws_server 的简称，是云平台的 API 网关服务，Global 服务入口，提供参数检查、访问控制、请求转换和转发，并提供工单、全局配额、通知列表、消息中心等全局资源的业务处理。

|WAF
|Web Application Firewall，Web 应用防火墙，是云平台提供的 Web 安全防护产品，用于降低网站资产和数据泄露的风险，保障 Web 应用的可用性。

|Zone
|可用区，是指在同一区域内，电力和网络互相独立的物理区域。在同一区域（Region）内，可用区与可用区之间内网互通。 +
各可用区之间可以实现故障隔离，即如果一个可用区出现故障，则不会影响其他可用区的正常运行。

|===
