QKE 基于 QingCloud 云平台部署，可以充分利用云平台网络和存储的服务能力，为用户提供整体解决方案。

QKE 整体物理架构如下图所示。

image::/images/ep_offline/qke_twp/02_qke_physical_architecture.png[]

各节点说明如下：

* *控制节点（Master）*：提供 Kubernetes 所需的控制面板组件，提供资源调度、控制等能力，为达到高可用，一般建议为三个节点。
* *工作节点（Worker）*：控制节点会动态调度 Pod 到工作节点，工作节点上运行着承载服务能力的容器。
* *etcd 节点*：保存 Kubernetes 集群信息，为达到高可用，一般建议三个节点。etcd 服务可以独立部署为专门用于运行 etcd 的集群，也可部署在 QKE 集群所用的节点或者以 Pod 方式部署在 Kubernetes 集群中。
* *镜像仓库节点*：集中保存平台所需的 Docker 镜像，为达到高可用，需要两台主节、一台数据库节点、台缓存节点、一台日志节点。镜像仓库也可以 Pod的方式部署在 Kubernetes 集群中，建议使用青云提供的 Harbor 镜像仓库，和 QKE 对接更方便。
* *监控/日志/告警*：可将监控/日志/告警服务部署于集群中，也可对接外部的企业已有的监控、日志告警系统，同时可使用 QingCloud 云平台提供的CloudSat 监控&告警服务对集群底层资源进行监控和告警。
