== Nginx 配置

. 登录{product_name} FirstBox 节点。
. 执行以下命令，查看 Webservice 节点信息。
+
[source]
----
cat /etc/hosts | grep webservice
----
. 分别登录{product_name} Global Zone 的两个 Webservice 节点。
. 执行以下命令，修改 `webconsole.conf` 文件。
+
[source]
----
vim /etc/nginx/sites-available/webconsole.conf
----
+
. 按如下说明新增相关参数。
+
[source]
----
  # 新增：
   location /paas/ {
       alias /pitrix/lib/pitrix-webconsole-paas/;
       try_files $uri /paas/index.html;
   }

  # 新增一个 paas 路径
   location ~ ^/(notify|iaas|hpc|epfs|cfw|iam|resources|accounts|nat|finance|paas|apig|smc)/static/(.*)$ {
       alias /pitrix/lib/pitrix-webconsole-$1/static/$2;
       expires 30d;
   }
----

== Python 配置

include::../../../../../_components/attention.adoc[]
该操作会刷新 Webservice 节点数据，请提前备份好 Webservice 节点 /pitrix/lib/pitrix-webconsole 数据，避免定制化内容被覆盖而无法找回。
include::../../../../../_components/table_footer.adoc[]

. 登录{product_name} FirstBox 节点。
. 执行以下命令，修改 `settings.py` 文件。
+
[source]
----
vim /pitrix/conf/variables/webs/webconsole/settings.py
----
+
. 在 `PITRIX_SETTINGS` 字段下新增如下 `qke_config` 配置。其中，**endpoint** 参数中 `<zone_id>` 为环境 Golbal Zone 的 ID，其他参数保持默认值即可。
+
[source,py]
----
    'qke_config': {
        'endpoint': '<zone_id>-webservice', # 代理的后端服务节点有 proxy，配置 proxy 节点，可以在配置之前使用 nc -zv <zone_id>-webservice 9102 检查一下
        'port': 9102,
        'verify_ssl': False,
        'protocol': 'http',
        'prefix': 'api/v1/qke'
    },
----

. 执行以下命令，刷新配置到 Global Zone 的 Webservice 节点。其中， `<global_zone_id>` 为环境 Golbal Zone 的 ID。
+
[source,py]
----
fb build_pkgs -p pitrix-ks-webservice-webconsole
fb update <global_zone_id>-webservice pitrix-ks-webservice-webconsole
----

== 前端配置

. 登录{product_name} FirstBox 节点。
. 执行以下命令，查看 Webservice 节点信息。
+
[source]
----
cat /etc/hosts | grep webservice
----
. 分别登录 Global Zone 的两个 Webservice 节点。以下所有操作均需在两个 Webservice 节点执行。
. 执行以下命令，进入 configs 目录。
+
[source]
----
cd /pitrix/lib/pitrix-webconsole-paas/configs
----
. 执行以下命令，创建并编辑 `local_config.yaml` 文件。
+
[source]
----
vim local_config.yaml
----
