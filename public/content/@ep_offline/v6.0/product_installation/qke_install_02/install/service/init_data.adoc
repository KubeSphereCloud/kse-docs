---
title: "导入初始化数据"
weight: 30
---

1. 登录一个 Webservice 节点。
2. 执行以下命令，进入 /pitrix/lib/pitrix-scripts/qke 目录。
+
[source]
----
cd /pitrix/lib/pitrix-scripts/qke
----
3. 执行以下命令，修改 /conf/config.yaml 文件。
+
[source]
----
vi conf/config.yaml
----
+
根据实际环境修改 regions 和 roles 里面的字段。
+
配置示例：
+
[source,yaml]
----
regions:
  pek3: # region 名称
    - pek3b # zone 名称
    - pek3c
    - pek3d
  sh1:
    - sh1a
    - sh1b

roles: 
  - master:
      - prefix: s1
        instance_class: 101
        volume_type: 100
      - prefix: ec3
        instance_class: 203
        volume_type: 200
  - node_perf:
      - prefix: s1
        instance_class: 101
        volume_type: 100
  - node_super_perf:
      - prefix: ec3
        instance_class: 203
        volume_type: 200
----
+
参数说明：
+
[cols='1,4'] 
|===
| 参数 | 说明 

| regions | 配置 region 名称和 zone 名称。
| roles a| 节点主机规格。依据实际环境，可以选择两种不同规格的主机，一个性能型，一个超高性能，然后填写对应的 instance_class 和 volume_type。

* 基础型主机，请配置 instance_class 101 ，volume_type 100。
* 企业型主机，请配置 instance_class 203 ，volume_type 200。

如果只有一种主机类型，master 中可只保留一个，node_perf、node_super_perf 中可以填写一样的。
|===
4. 执行以下命令，打印初始化数据。
+
[source]
----
./qke -e ./conf/config.yaml
----
5. 执行以下命令，插入初始化数据。

** 不插入arm版本
+
[source]
----
./qke -e ./conf/config.yaml -d 0
----
** 插入arm版本
+
[source]
----
./qke -e ./conf/config.yaml -d 0 -a 1
----
+
