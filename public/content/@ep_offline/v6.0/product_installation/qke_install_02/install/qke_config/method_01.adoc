== 配置 webconsole

在 webconsole 的 production_services 中启用 containers 和其下的 qke 服务，即可在 Console 中显示 QKE 服务。

1. 登录配置中心。
2. 在左侧导航栏，选择**应用管理**，进入应用管理页面。
3. 选择 **webconsole** 应用，进入 webconsole 配置管理页面。
4. 在 production_services 所在行，点击右侧的**管理配置**。
+
image::/images/ep_offline/qke_install/install_qke_config_01.png[]
5. 进入 production_services 配置页面。
+
启用 containers 和其下的 qke 服务。
+
示例如下：
+
[source,yaml]
----
- name: containers
  title: Container Service
  enable: true # 这里改成true
  items:
    - name: qke
      title: KubeSphere®(QKE)
      enable: true # 这里改成true
      icon: qke
      link: /paas/qke
      zh-cn: 容器引擎 QKE
----
6. 配置完成后，点击**保存**，在弹出的对话框中输入备注信息，点击**提交**。
7. 在右侧点击启用，大约一分钟后生效，即可登录 Console 查看 QKE 服务。

== 配置 webconsole-pass

在 webconsole-pass 的 global_config 中配置支持 qke 服务的 region 和可用区。

=== 配置 region

1. 登录配置中心。
2. 在左侧导航栏，选择**应用管理**，进入应用管理页面。
3. 选择 **webconsole-pass** 应用，进入 webconsole-pass 配置管理页面。
4. 在 global_config 所在行，点击右侧的**管理配置**。
+
image::/images/ep_offline/qke_install/install_qke_config_02.png[]
5. 进入 global_config 配置页面。
+
修改 zone_groups 参数。
+
示例如下：
+
[source]
----
zone_groups:
  abroad:
  - {{ region_id }} # 根据实际环境修改
  domestic:
  - {{ region_id }} # 根据实际环境修改
----
+
参数说明：
+
[cols='1,3'] 
|===
| 参数 | 说明 

| abroad | 海外 region。若没有海外 region，可以删除此参数。
| domestic | 国内 region。
|===

=== 配置 volume_class

1. 登录配置中心。
2. 在左侧导航栏，选择**应用管理**，进入应用管理页面。
3. 选择 **webconsole-pass** 应用，进入 webconsole-pass 配置管理页面。
4. 在 zone_config 所在行，点击右侧的**管理配置**。
+
image::/images/ep_offline/qke_install/install_qke_config_03.png[]
5. 在 zone_config 配置页面，修改 volume_class 相关参数。
+
示例如下：
+
[source]
----
default_volume_class: 6  # 根据实际环境修改
default_volume_class_config:
  default: 6  # 根据实际环境修改
  key: volume_class
  range: # 根据实际情况修改
  - 6
default_volume_size_config:
  max: 2000
  min: 100
  step: 10
----
+
参数说明：
+
* default_volume_class：默认的磁盘类型。
** 如果 NeonSAN 环境是纯 SSD 环境，则配置 volume_class 5。
** 如果 NeonSAN 环境为混插型硬盘，则配置 volume_class 6。
6. 配置完成后，点击**保存**，在弹出的对话框中输入备注信息，点击**提交**。
7. 在右侧点击**启用**，大约一分钟后生效。
