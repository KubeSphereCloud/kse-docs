title: SSH 服务器无法连接
weight: 02
---

== 问题现象

本地电脑和某一个管理节点做了互信后，通过 SSH 方式，使用 yop/zhu1241jie 登录上了该管理节点。然后在该管理节点上 SSH 连接另外一个节点时提示“SSH 服务器拒绝了密码” 。

物理节点的操作系统为 Ubuntu 20.04 版本、网络已经配好，telnet 22 端口也是通的。

== 问题原因

只做了本地电脑和管理节点之间的互信，管理节点与其他节点之间并没做互信，所以提示 SSH 服务器拒绝。

== 解决方法

可以在本地电脑使用 yop/zhu1241jie 分别登录其他节点。
