---
title: 安装操作系统时自动安装到了非 sda 盘
weight: 10
---

== 问题现象

为服务器安装操作系统时，总是自动安装到非 sda 盘（例如 sdb 盘）。

== 可能原因

若 sda 盘符被其他设备/磁盘占用，将会安装到其他盘上。

场景示例：

* 服务器上架后连接有 KVM 线缆，KVM 线带 USB 口，会占用系统里的 sda 盘符。
* SDS2.0（现在不推荐此方式）混插场景，且机械盘为直通模式，会占用系统里的 sda 盘符。
* 混插戴尔 740， sda 盘符总是会被 U 盘或者 ISO 占用。

== 解决方案

* **方案一**：检查服务器上的其他设备，拔掉占用 sda 盘符的设备。

* **方案二**：使用非 sda 盘符安装操作系统。
+
--

* **方法一**：遇到安装不下去时，按 Ctrl+Alt+F2 切换至 Terminal，执行命令 ``ls /dev/sd*`` 查看盘符分配情况，并记录系统盘的盘符。打开 U 盘安装盘中的 preseed 目录，编辑 qingcloud.seed（或者 autoinstall.seed）文件，把 sda 替换成实际的系统盘盘符。
* **方法二**：在方法一基础上，如果要通过“带外管理”使用 ISO 文件安装操作系统，需要使用 Utraliso 等工具重新制作成 ISO，然后使用新的 ISO 挂载安装。
* **方法三**：仅对系统盘做 RAID，数据盘先不做 RAID，等到系统安装完成之后再对数据盘做 RAID。
* **方法四**：拔掉所有数据盘后，再进行操作系统的安装。待操作系统安装完成后，再插上数据盘。
