== 云平台部署架构

企业云平台根据管理服务与计算服务是否分离，可以分为标准部署与融合部署两种方式。

* **标准部署模式**
+
我们又称之为分离部署。该部署模式中，云平台的管理服务与计算服务分别部署于不同的服务器节点。

* **融合部署模式**
+
融合部署模式下，云平台的管理服务与计算服务共用服务器节点，部署于相同的服务器节点。


== 计算存储部署架构

云平台的存储，除使用 NeonSAN 外，支持对接第三方存储。

=== Hyper-Sanc 架构

当云平台使用 NeonSAN 作存储时，我们称之为 Hyper-Sanc 架构。Hyper-Sanc 部署架构也支持融合部署与分离部署。

* **Hyper-Sanc 独立部署模式**
+
是指 NeonSAN 的存储服务，不使用云平台的服务器节点，而是使用独立的服务器节点进行部署。使用该部署架构时，建议先部署 NeonSAN 集群，再部署企业云平台。

* **Hyper-Sanc 融合部署模式**
+
是指 NeonSAN 的存储服务与云平台的计算服务，共用节点服务器，即 NeonSAN 的相关服务，部署于云平台的 Hyper 节点上。使用该部署架构时，需要先部署企业云平台，再部署 NeonSAN。

=== Hyper-Tpsc 架构

当云平台使用第三方存储时，我们称之为 Hyper-Tpsc 架构。对于 Hyper-Tpsc 的部署架构，云平台目前仅支持分离部署的方式，即第三方存储服务使用专用的集中式存储控制器，不与云平台服务器共用节点。
