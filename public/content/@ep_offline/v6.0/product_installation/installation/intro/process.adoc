企业云平台的安装部署，使用自研的 Installer 工具。在安装部署前期，相关人员需对整个待部署环境进行系统的规划，确定部署架构，再对整个待部署环境进行一系列的准备工作后，使其达到一个待部署的状态。安装部署人员再使用 Installer 工具进行部署。

== 环境规划

在该阶段，实施人员需根据实际情况，对待部署环境进行清晰的需求分析和范围界定，并输出解决方案。详细内容参考link:../../planning/hardware[环境规划]。

== 环境准备

完成环境规划，明确部署架构后，实施人员对待部署环境进行前期的软、硬件的准备工作，使其达到一个 Ready 的状态。详细内容参考link:../../prepare[环境准备]。


== 安装部署

企业云平台部署完成后，即可提供相应的云资源以及云服务，默认提供的服务，可参考link:../features[功能说明]。同时，企业云平台也开放增值服务，待企业云平台部署完成后，用户可根据需求，选择安装相应的增值服务。详细内容参考link:../../deploy[安装部署]。

== 环境验证

Installer 工具运行完成后，用户需依次登录 Console、BOSS 以及配置中心，确认各服务运行正常以及进行企业云平台的初始化后，才能将环境投入生产使用。详细内容参考link:../../deploy/check_env[环境验证]。
