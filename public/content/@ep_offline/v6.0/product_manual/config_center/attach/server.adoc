== 文件规范

* 大小写敏感
* 使用缩进表示层级关系
* 缩进时不允许使用 Tab 键，只允许使用空格。
* 缩进的空格数目不重要，只要相同层级的元素左侧对齐即可

=== 区域可使用资源范围设置

resource_limits 配置分为三个层级：

* default：该层级配置在各个区域均生效。
* region：该层级配置在所在区域优先生效，即会覆盖 default 层级下的配置，default 层级下的配置可以添加到 region 下，之前版本实施的环境该层级为zone_id，当有多个区域时按需求填写每个区域的配置。
* zone：此配置定义 resource_limits 内配置生效的区域填写各个区域的 region_id，installer4.x 之前版本实施的环境填写该区域的 zone_id。

如下为 server.yaml 中 vol 参数与 volumetype 对应关系：

[source]
----
VOLUME_TYPE_HIGH_PERFORMANCE = 0
VOLUME_TYPE_HIGH_CAPACITY = 1
VOLUME_TYPE_HIGH_CAPACITY_PLUS = 2
VOLUME_TYPE_HIGH_PERFORMANCE_PLUS = 3
VOLUME_TYPE_SAN = 4
VOLUME_TYPE_HIGH_PERFORMANCE_SHARED = 5  # high performance shared volume, neonsan on "label SSD"
VOLUME_TYPE_HIGH_CAPACITY_SAN = 6 # neonsan on "label HDD"
VOLUME_TYPE_BM_LOCAL = 10  # local volume on bm instance
VOLUME_TYPE_ST = 100
VOLUME_TYPE_ET = 200
----
