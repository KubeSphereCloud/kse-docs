== 背景信息

丰富的防火墙配置策略可以将恶意攻击阻挡在门外，企业云平台为每个用户提供了一个缺省防火墙，用户也可以自建更多的防火墙。不同的云服务器可以被设置不同的访问强策略。

安全组（Security Group）：安全组则是一种虚拟防火墙，能极大地提高一个内部网络或主机的安全性，通过策略配置，过滤不安全的服务，从而降低内部网络或主机的风险。为了加强位于基础网络 vxnet-0 中云服务器或路由器的安全性，可以在云服务器或路由器之前放置一个安全组。云平台安全组是位于内部网和外部网之间的屏障，它按照系统管理员预先定义好的规则来控制数据包的进出。

== 场景描述

该任务指导管理员进入安全组页面，查看安全组列表信息。

== 功能描述

安全组列表集中展示当前云环境中所有防火墙的名称及 ID、用户邮箱及 ID、应用修改状态以及创建时间信息。

== 前提条件

已获取 *云计算运维管理平台* 的账号和密码。

== 操作步骤

. 登录 *云计算运维管理平台*。
. 选择 *资源管理* > *安全* > *安全组*，进入 *安全：安全组* 页面，可查看安全组列表和详情。
+
image::/images/boss/manual/resource_mgt/security_groups_operation.png[]

. 点击安全组名称，进入安全组详情页面，可查看安全组详细信息以及操作日志。
+
在安全组详情页下方可查看该安全组所有的规则信息：
+
上行规则：从云资源访问外部。
+
下行规则：从外部访问云资源。
+
在页面右上角点击 *导出全部规则*，点击报表管理中心，在报表管理中心点击安全组规则的文件名称，即可将所有安全组规则保存到本地。
+
image::/images/boss/manual/resource_mgt/security_groups_info.png[]

=== 导出安全组列表

. 在安全组页面右上角点击 *导出*。
. 勾选 *包含规则* 或 *包含资源*。
+
image::/images/boss/manual/resource_mgt/security_groups_export.png[,60%]

. 点击 *确定*。
. 点击 *报表管理中心*。
. 在 *报表管理中心* 页面，点击 *安全组（包含资源）* 或 *安全组（包含规则）* 模块所在行的文件名称。
+
安全组列表下载到本地。
