{product_name} 支持融合部署与分离部署。下面对两种方式分别加以说明。


== 融合部署

融合部署是指网关服务与存储服务部署于同一节点服务器上，即集群中仅存在全能节点，不存在独立的存储节点与网关节点。

**部署要求：**

初次部署成本较低，最小规模为：3 个全能节点。

**扩容要求：**

* 可扩容全能节点，无法扩容独立的网关节点与存储节点。扩容时，需同时扩容接入能力和存储能力。
* 扩容全能节点时，应以存储池为单位进行扩容。若存储模式为副本模式，则按照 3 节点每存储池的方式进行扩容；若存储模式为 EC 集群模式，则按照 EC 配比来决定扩容存储池节点数。
  如：EC 配比为 4+2，则按照 3 节点每存储池的方式进行扩容。
* 新扩容的节点，需在存储硬盘个数、容量、型号等方面保持一致。



== 分离部署

分离部署是指网关服务与存储服务分别部署于不同的节点服务器上，即集群中不存在全能节点。初次部署成本较融合部署高。

**部署要求：**

初次部署时，最小规模为：3 个网关节点 + 3 个存储节点。

**扩容要求：**

* 可单独扩容存储节点与网关节点，不支持扩容全能节点。
* 扩容存储节点时，与link:#_融合部署[融合部署]环境里，扩容节点的需求相同。
* 新扩容的存储节点，需在存储硬盘个数、容量、型号等方面保持一致。
* 网关节点支持单节点扩容，每扩容一个节点，相应接入能力会获得提升，累计扩容三个网关节点后，可以将一部分索引 Rebalance 到新节点。


== 部署建议

* 分离部署初次部署成本较高，适用于大规模部署的场景。这种部署方式使得在业务需求较大的并发访问能力或更大的带宽时可单独扩容网关节点，在需求较大存储空间时可单独扩容存储节点。
* 大规模部署的场景中，{product_name} 建议单个存储池的节点数控制在 10～12 台以内。
* 融合部署初次部署成本较低，适用于访问量变化不大，存储量持续增长的场景。

用户可根据实际情况选择合适的部署方式。
