dd 工具是把指定的输入文件拷贝到指定的输出文件中，并且在拷贝过程中可以进行格式转换。dd 也是我们经常使用到的磁盘测试工具，用来测试硬盘、块设备的读写性能，不建议用来测试集群的性能。

== 安装配置

Linux 操作系统安装完成后，可直接使用 dd 工具进行测试。


== 测试步骤

**准备工作**

. 使用正确的用户名和密码登录 Portal。
. 点击**创建文件系统**，输入合法参数后点击**确定**，创建文件系统。
+

include::../../../../../../../_components/note.adoc[]

用户可通过设置文件系统数据块大小以达到最优效果。数据块大小与预读数据块大小均设置为 `自动`。

include::../../../../../../../_components/table_footer.adoc[]

. 进入文件系统详情页面，点击**创建共享**，输入合法参数创建共享。
. 在客户端上挂载文件系统下的共享点，进行性能测试。

**性能测试**

在客户端使用 dd 工具测试性能时，用户需根据实际情况对参数进行配置，参数说明可参考link:../../../../additional/dd_para/[附录]相关内容。

* 测试写入速度
+
[source,shell]
----
# dd if=4k_1g_test of=/dev/null bs=4k
----

* 测试读取速度
+
[source,shell]
----
# dd if=4k_1g_test of=/dev/null bs=4k
----

include::../../../../../../../_components/note.adoc[]

* `if=/dev/zero` 不产生 IO，因此可以用来测试纯写速度。
* `of=/dev/null` 不产生 IO，可以用来测试纯读速度。
* 将 `if` 拷贝到 `of` 则同时测试了读写速度。

include::../../../../../../../_components/table_footer.adoc[]

== 结果分析

由于服务器的配置不同，网络等各种原因，历史测试的结果以及得出的最优性能对应的配置参数等，只能作为参考。本章内容主要介绍如何查看测试结果。
