== 更新 global_config

. 使用浏览器登录 IaaS 的配置中心。
+
include::../../../../../../_components/note.adoc[]

* 默认登录方式为：地址 `cadmin.<domain>`，用户名 `admin@<domain>`，密码 `zhu88jie`。
* 其中 `<domain>` 为 IaaS 的域名信息，需根据实际情况修改。
* 登录示例为：地址 `cadmin.democloud.com`，用户名 `admin@democloud.com`，密码 `zhu88jie`。

include::../../../../../../_components/table_footer.adoc[]


. 在配置中心，点击**配置管理**，进入**配置管理**主页面，在搜索框中输入 `webconsole`，点击**搜索**后，找到配置文件 `global_config`，点击**管理配置**。操作流程如下图所示。
+
image::/images/u10000_v5.0/install_webconsole_1.png[install_webconsole_1,100%]


. 在** global_config **的配置页面，修改 `qingstor` 相关配置参数。
+
image::/images/u10000_v5.0/install_webconsole_2.png[install_webconsole_2,100%]
+
**待修改及新增参数示例**
+
[source,yaml]
----

  qingstor_disable_virtual_host: true
  qingstor_isolated_cname: true
  qingstor_charged: false

  # 配置此选项之后 console 就会给 qs-global-gateway 发送 zone_id 而非 region_id.
  qs_region_zone_mapping:  //对象用
     pek3: pek3b
  qfs_region_zone_mapping:  //文件用
     pek3: pek3b

  qingstor_region_az:  // QingStor 对象存储 v3.3 新增参数
     pek3:
       - pek3b
     sh1:
       - sh1a
     ap3:
       - ap3  

  service_unsupported_zones_map: // QingStor 对象存储 v3.3 新增参数
     qingstor: [ap2a, pekt3]


----
+
**填写说明**
+
[cols="3a,5a", options="header"]
|===
| 参数 | 说明 

|qingstor_disable_virtual_host | 是否使用 Virtual Host 格式的 URL 地址。需配置为 `true`，不使用 Virtual Host 格式的 URL 地址。
| qingstor_isolated_cname |

* 是否支持配置自定义域名。默认情况下，Console 需要检查域名是否备案。
* 私有云环境，可跳过该检查，设置为 `false`。

| qingstor_charged a| 是否显示计费功能。

* 私有云配置为 `false`。
* 若有完整的计费服务，则可开启该功能，即设置为 `true`。


|  * qs_region_zone_mapping
* qfs_region_zone_mapping

| * 用于解决当 region_id 与 zone_id 不一致时，导致的在 Console 上传文件出错的问题。
* 填写格式参考示例。`pek3: pek3b` 中的 `pek3b` 为当前部署环境的 Zone ID，`pek3` 为当前部署环境所属的 Region ID。

|  qingstor_region_az | 用于配置 Region 与其 Zone 列表，表示哪些 Region 与其下面的 Zone 能创建 Bucket。
| service_unsupported_zones_map | IaaS 环境中有多个 Zone，没有对接 {product_name} 的 Zone，需添加在此。
|===

. 点击**启用**即可。


== 更新 settings

. 登录 IaaS 的配置中心。

. 在配置中心，点击**配置管理**，进入**配置管理**主页面，在搜索框中输入 `webconsole`，点击**搜索**后，找到配置文件 `settings`，点击**管理配置**。
+
image::/images/u10000_v5.0/install_webconsole_4.png[install_webconsole_4,100%]

. 在** settings **的配置页面，修改 `qs_config` 相关配置参数。
+
image::/images/u10000_v5.0/install_webconsole_5.png[install_webconsole_5,100%]
+
**填写说明**
+
[cols="3a,5a", options="header"]
|===
| 参数 | 说明 

|qy_access_key_id | {product_name} 中查询到的 `qs_secret_access_key_id` 并加密。可参考link:../u1w/#_密钥信息[前文]查询内容进行填写。
| qy_secret_access_key |{product_name} 中查询到的 `qs_secret_access_key` 并加密。可参考link:../u1w/#_密钥信息[前文]查询内容进行填写。
| * endpoint 
* port
* protocol| 对象存储的 Global 域名相关参数。可与 IaaS 是不同的域名。

* `https` 协议对应 `443` 端口号。
* `http` 协议对应 `80` 端口号。

|  proxy | * 若启用了 `proxy` 字段，那么 `endpoint`、`protocol` 与 `port` 这几个字段的配置无效。
* {product_name} 建议不开启 `proxy`。
| verify_ssl | 维持默认即可。
| maxFileNum | 表示能够在 Console 上同时上传的最大文件个数。私有云项目中，可将该值调大，比如修改为 `200`。
|===

. 点击**启用**即可。
