== 查看总览

成功登录 Portal 后，系统默认显示**总览**页面。该页面默认展示近一个月的集群容量历史数据和未来一个月的预测数据。用户可在页面选择**时间范围**，调整显示数据。

* 图例
+
image::/images/u10000/portal_main.png[portal_main,100%]

* 说明
+
[cols="1,3a", options="header"]
|===
| 模块 |说明

|集群容量 | 显示当前环境中所有集群的容量信息，监控图上的指标，可以根据需求进行选择。

* **已用容量**表示当前环境中所有存储池已使用的容量总和。
* **可用容量**包含了 `Readonly` 存储池的剩余容量，并在页面上给予提示信息。
* **时间范围**参数，可以只选择历史时间，也可以选择历史 + 未来，或选择未来的时间范围，粒度是 1天。
* **设置阈值**后，容量折线图上会根据当前容量的百分比画出一条直线。
* `3天`、`7天`、`30天` 的移动平均线，是以 `3天`、`7天`、`30天` 为周期计算容量增长情况，并据此预测后续容量的变化趋势，计算对应的剩余容量可用时间。
* 使用数据导出功能时，集群的历史容量导出粒度为一天一个数据。

|集群容量吞吐量 | 
* 用户可根据需求调整折线图数据取值的时间范围。
* 实时数据，其更新频率为每 5分钟更新一次。

|集群 IOPS | 

* 用户可根据需求调整折线图数据取值的时间范围。

* 实时数据，其更新频率为每 5分钟更新一次。

|待处理告警 |* 展示 `紧急`、`重要`、`次要` 三个状态下待处理的告警统计数字，点击可跳转至待处理告警列表页，同时会按照指定状态进行过滤。

* 列举当前环境中，最新的 5 条告警记录，点击某个告警记录可跳转至该告警的详情页面。
* 点击**更多**可跳转至待处理告警列表页。

|物理资源 | 展示 `服务器`、`索引盘`、`数据盘` 的统计信息，点击可跳转至对应资源界面。
|===

== 阈值设置

集群使用容量的默认阈值为 80%，超过该阈值，用户需进行扩容。用户可根据需要自定义该阈值，用以提前准备扩容或做出其他相应决策。设置的阈值会在**集群容量**模块展示。

如需调整，用户可点击**集群容量**右上角的**设置阈值**，在弹出的对话框内，根据提示内容进行设置。
