浏览器默认不允许调用不同域名下的资源，当用户使用 JavaScript AJAX 向 {product_name} 对象存储发起的请求属于跨源请求时，浏览器为了安全考虑，会拒绝该请求。此时需要为 Bucket 配置 CORS 规则。

== 添加 CORS

//. 成功link:../../../../../login[登录 Portal]。

. 进入link:../../sub_resource/[子资源详情]页面，在 **CORS** 卡片，点击**添加请求源**。

. 弹出**添加请求源**对话框，用户可根据页面提示信息，填写相关参数后，点击**提交**。
+
image::/images/u10000/stor_bucket_cors.png[stor_bucket_cors,50%]
+
**页面信息说明**

* `允许请求源`：跨域请求来源。
* `允许 HTTP 方法`：设置源所允许的 HTTP 方法。
* `允许 HTTP 请求头`：设置源所允许的 HTTP 请求头。
* `允许访问 HTTP 响应头`：设置用户能够从其应用程序进行访问的 HTTP 响应头。
* `最大响应缓存时间`：设置浏览器为预检请求 HEAD 缓存响应的时间。其他类型请求不适用。

. 成功设置的 CORS，在link:../../sub_resource/[子资源详情]页面的 **CORS** 卡片列表显示。

== 修改 CORS

//. 成功link:../../../../../login[登录 Portal]。

. 进入link:../../sub_resource/[子资源详情]页面，在 **CORS** 卡片，选择待修改的 CORS 条目后，点击**修改**。

. 在弹出的对话框内，修改相应的参数后，点击**提交**，即可完成修改。

. 更新后的 CORS，在link:../../sub_resource/[子资源详情]页面的 **CORS** 卡片列表显示。

== 删除 CORS

//. 成功link:../../../../../login[登录 Portal]。

. 进入link:../../sub_resource/[子资源详情]页面，在 **CORS** 卡片，选择待删除的 CORS 条目，点击**删除**。

. 在弹出的对话框内，确认操作无误后，点击**确定**。

. 成功删除的 CORS，不在link:../../sub_resource/[子资源详情]页面的 **CORS** 卡片列表显示。
