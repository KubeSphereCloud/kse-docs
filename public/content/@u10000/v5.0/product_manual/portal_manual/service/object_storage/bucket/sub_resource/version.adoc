---
title: 版本管理
weight: 2
---



版本管理（BucketVersioning）是在同一 Bucket 中保留 Object 的多个版本的方式。对于 Bucket 中存储的每个 Object，均可使用版本管理功能来保存、检索和还原它们的各个版本。

通过版本管理，用户能够轻松从意外操作和应用程序故障中恢复历史数据。

Bucket 的版本管理功能包含三种状态，分别为：`未开启`、`开启`、`暂停`。一旦 Bucket 开启版本管理功能，将无法返回至 `未开启` 状态，只能在 `开启` 与 `暂停` 两种状态间切换。详细操作如下。

== 开启 Bucket 版本管理

//. 成功link:../../../../../login[登录 Portal]。

. 在link:../../sub_resource/[子资源详情]页面，点击**Bucket 版本管理 > 启用版本管理**。
 
. 弹出确认对话框，用户确认操作无误后，点击**确定**以启用 Bucket 版本管理功能。Bucket 一旦开启版本管理功能，将无法返回至 `未开启` 状态，只能在 `开启` 与 `暂停` 两种状态间切换。

== 关闭 Bucket 版本管理

用户如需停止版本管理功能，可在 Bucket 子资源页面点击**Bucket 版本管理 > 暂停 > 保存修改**，以完成暂停版本管理功能的设置。暂停版本管理功能的 Bucket，当用户上传或删除同名的 Object 时，系统不再为其产生新的历史版本。该操作对已存在的历史版本无影响。

== 启用 Bucket 版本管理

用户如需启用已停止的版本管理功能，可在 Bucket 子资源页面点击**Bucket 版本管理 > 启用 > 保存修改**，以完成启用版本管理功能的设置。

== 查看文件列表

用户可在 link:../../file_opt/file_version_list[Bucket 文件列表]页面，查看 Object 的所有版本。
