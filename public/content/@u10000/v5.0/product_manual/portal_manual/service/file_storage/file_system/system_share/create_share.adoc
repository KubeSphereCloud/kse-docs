待用户成功创建一个文件系统后，需针对该文件系统创建挂载点。



== 操作步骤

. 成功登录 Portal。

. 点击**存储服务 > 文件存储 > 文件系统**，进入文件系统列表页面。
+
image::/images/u10000/stor_file_system_main.png[stor_file_system_main,100%]

. 点击文件系统名，进入文件系统详情页面后，点击**挂载点**标签，进入文件系统挂载点列表页面。
+
image::/images/u10000/stor_file_system_share.png[stor_file_system_share,100%]

. 在文件系统挂载点列表页面点击**创建挂载点**。

. 弹出**创建挂载点**对话框，用户根据页面提示信息填写相关参数后，点击**确定**。
+
image::/images/u10000/stor_file_system_create_share.png[stor_file_system_create_share,40%]



== 页面参数

[cols="1,3a", options="header"]
|===
|参数 |说明 

|协议类型 |客户端访问文件系统的协议，目前支持 SMB、NFS 以及 FTP 协议。填写建议如下。

* 若客户端为 Windows 操作系统，{product_name} 建议使用 `SMB` 协议。
* 若客户端为 Linux 操作系统，{product_name} 建议使用 `NFS` 协议。
* 若客户端为 MAC，{product_name} 建议使用 `NFS` 协议。
* 选择使用 `FTP` 协议，客户端无需做额外操作，可直接在本地进行访问。

|共享目录 |当前文件系统待共享的目录。目录格式为绝对路径。若用户指定的共享路径不存在，则系统将为用户自动创建该目录。

* 填写 `/` 表示将整个文件系统作为一个大目录进行共享。挂载至客户端后，用户可以看到这个文件系统下所有的子目录和文件。
* 填写 `/test` 等目录，表示将文件系统下的 `/test` 目录进行共享，挂载至客户端后，用户只能看到 `/test` 目录下的所有子目录和文件。
* 若同一个目录通过不同的协议挂载到不同的客户端，则用户可以同时通过 NFS 和 SMB 协议访问到该目录的数据。

|权限组 | * 系统提供 NFS、SMB、FTP 默认权限组，如无特殊权限要求，用户可直接使用该默认权限组。

* 用户可参考本文其他章节内容，了解link:../../../auth_mgm[权限组]相关信息。


|===
