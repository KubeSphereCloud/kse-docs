== 查看用户列表

此页面可查看当前系统中的 Console 用户。

. 成功登录 Portal。

. 点击**运维管理 > 用户管理**，进入用户列表。
+
image::/images/u10000/maintain_user_list.png[maintain_user_list,100%]
+
**页面信息说明**

* `用户 ID`：系统自动生成，用于唯一标识该用户。
* `空间大小`：显示的是当前用户所有 Bucket 已使用的存储空间。
* `Bucket 配额`：显示已使用的额度和整体额度。如：`1/2` 表示已创建 1个 Bucket，可创建 2个 Bucket。
* 如页面所示，当前页面支持针对用户状态与关键字的筛选，便于快速查找目标用户。


== 创建用户



此处创建的用户可用于登录 Console，创建 Bucket，上传下载文件等操作，使得用户可以正常使用 {product_name} 的对象存储服务。

. 在link:#_查看用户列表[用户列表页面]，点击**创建用户**。

. 弹出**创建用户**对话框，用户根据页面提示信息，填写相关参数后，点击**保存**。
+
image::/images/u10000/maintain_user_create.png[maintain_user_create,50%]

. 新创建的用户，在link:#_查看用户列表[用户列表页面]，状态显示为 `活跃`。

== 禁用用户

禁用的用户，不能成功登录 Console 与 Portal，对该用户之前的操作无影响。

. 在link:#_查看用户列表[用户列表页面]，选择待操作的用户，点击**更多 > 禁用用户**。
+
image::/images/u10000/maintain_user_m.png[maintain_user_m,100%]

. 弹出确认对话框，用户确认操作无误后，点击**确定禁用**。

. 被禁用的用户，在link:#_查看用户列表[用户列表页面]，状态显示为 `禁用`。 在用户状态恢复之前，该用户不能进行任何操作。

== 恢复用户

被禁用的用户，可以恢复其状态，使其能正常使用。

. 在link:#_查看用户列表[用户列表页面]，选择待操作的用户，点击**更多 > 恢复用户**。
+
image::/images/u10000/maintain_user_m_restore.png[maintain_user_m_restore,100%]

. 弹出确认对话框，用户确认操作无误后，点击**确认恢复**。

. 恢复的用户，在link:#_查看用户列表[用户列表页面]，状态显示为 `活跃`。

== 修改用户密码

已创建的用户，可在 Portal 上对其密码进行修改。修改后的密码，立马生效。

. 在link:#_查看用户列表[用户列表页面]，选择待操作的用户，点击**更多 > 重置密码**。
+
image::/images/u10000/maintain_user_m.png[maintain_user_m,100%]

. 弹出**重置密码**对话框，用户根据页面提示信息，输入新密码后，点击**提交**。


== 查看用户详情

在link:#_查看用户列表[用户列表页面]，选择待查看的用户，点击用户 ID，进入用户详情页面。

image::/images/u10000/maintain_user_info_main.png[maintain_user_info_main,100%]

**页面信息说明**

* 如图所示，用户详情页面分两个卡片，基本信息与配置信息。
* `基本信息`：显示了用户 ID、用户名、邮箱、当前状态、创建时间以及所在区域。
* 配置信息包括监控信息、配额信息、API 密钥以及 Bucket 信息。

== 用户监控信息

Portal 提供基于用户的 API 请求、流量以及存储容量相关信息查询。

在link:#_查看用户详情[用户详情页面]，点击**监控**，进入用户监控标签页面。

image::/images/u10000/maintain_user_monitor.png[maintain_user_monitor,80%]

**页面信息说明**

* 如图所示，用户可以根据需求，选择查看指定时间段内监控数据。鼠标移动至页面，可查看指定时间的详细数据。
* API 请求相关信息，包括外网读请求、外网写请求、内网读请求以及内网写请求。
* 流量支持实时数据查询，用户根据需求，点击**实时**开关。流量绘制单位为 MB，包括外网出流量、外网进流量、内网出流量、内网进流量。
* 存储模块的监控项为已使用的存储空间，单位 MB。


== 调整用户配额

系统默认用户的 Bucket 配额与 API 密钥配额均为 2。当有需求调整时，Portal 支持修改。

. 在link:#_查看用户详情[用户详情页面]，点击**配额**，进入用户配额标签页面。
+
image::/images/u10000/maintain_user_limit.png[maintain_user_limit,100%]

. 点击 Bucket 配额卡片的**修改**，弹出**修改 Bucket 配额**对话框。用户根据需求，调整配额后，点击**提交**。
+
image::/images/u10000/maintain_user_limit_bucket.png[maintain_user_limit_bucket,40%]

. 点击 API 密钥配额卡片的**修改**，弹出**修改 API 密钥配额**对话框。用户根据需求，调整配额后，点击**提交**。
+
image::/images/u10000/maintain_user_limit_api.png[maintain_user_limit_api,40%]

. 调整后的配额，用户配额标签页面显示。
