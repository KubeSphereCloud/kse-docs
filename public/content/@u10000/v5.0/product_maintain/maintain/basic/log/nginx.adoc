Nginx 服务是所有请求的入口。

//== 操作步骤

. 执行如下命令行，登录 {product_name} 的 FirstBox 节点。命令行中的 `<firstbox ip>` 为 {product_name} 的 FirstBox 节点 IP。
+
[source,shell]
----
ssh -p 6023 root@<firstbox ip>
----

. 执行如下命令行，跳转至 {product_name} 任一网关节点。
+
[source,shell]
----
ssh core0
----

. 执行如下命令行，查看 Nginx 日志。
+
[source,shell]
----
less /qingstor/log/nginx/access.log
less /qingstor/log/nginx/error.log
----
+

运维人员严禁使用 `vi` 查看日志文件。`less` 用法参考link:../less[前文内容]

// == 批量搜索

// 使用 {product_name} 的运维工具，可批量搜索相关日志。

// . 执行如下命令行，登录 {product_name} 的 FirstBox 节点。命令行中的 `<firstbox ip>` 为 {product_name} 的 FirstBox 节点 IP。
// +
// [source,shell]
// ----
// ssh -p 6023 root@<firstbox ip>
// ----

// . 执行如下命令行，搜索 Nginx 日志。
// +
// [source,shell]
// ----
// /qingstor/bin/search_nginx_log <keyword>
// ----
// +
// include::../../../../../../_components/note.adoc[]

// * `<keyword>` 是搜索关键字，可以是 HTTP 的错误码，如 `400`、`500`、`502` 等，也可以是 `request-id` 的值，也可以是 Bucket 名，对象名等。 
// * 该命令行回显内容会包括当天所有符合条件的请求，如果请求量大，则会大量刷屏。因此，可在执行该命令行时，将结果重定向到一个文件后，再进行查看看。

// include::../../../../../../_components/table_footer.adoc[]
