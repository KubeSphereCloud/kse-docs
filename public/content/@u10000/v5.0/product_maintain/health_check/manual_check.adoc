当自动巡检报错时，用户可参考以下步骤，对相应集群状态进行手动检查。

== 检查容器状态

. 登录当前环境的 FirstBox 节点。
+
[source,shell]
----
ssh -p 6023 root@<firstbox ip>
----
+
include::../../../../_components/note.adoc[]

* 命令行中的 `<firstbox ip>` 为当前环境的 FirstBox 节点 IP，需根据实际情况进行填写。
* 登录密码默认为 `zhu88jie`。

include::../../../../_components/table_footer.adoc[]


. 在 FirstBox 节点中执行如下命令行，查看所有容器的状态。当 `status` 字段为 `running`，说明正常，其它状态均为异常。
+
[source,shell]
----
./qs_installer shell -a "/qingstor/bin/public/serviceTool.sh status" -l all_node
----
+
命令回显示例
+
[source,shell]
----
projectName                     status          pid     uptime
hydro_stor                      running         1               07:35:33
cosbench                        running         1               8-04:58:14
qs_gateway3                     running         2238            08:37:10
hydro_confd                     running         1               07:57:00
......
tipd_server_0                   running         1               8-17:00:41
tidb_server                     running         1               8-17:00:07
tikv_server                     exited          NONE    NONE
......
----
+
如上所示，`tikv_server` 容器状态异常，需查明原因并解决。

== 检查 Hydro 集群状态

使用 Hydro 的命令行工具，检查当前环境中 Hydro 集群信息。

. 登录当前环境的 FirstBox 节点。
+
[source,shell]
----
# ssh -p 6023 root@<firstbox ip>
----
+
include::../../../../_components/note.adoc[]

* 命令行中的 `<firstbox ip>` 为当前环境的 FirstBox 节点 IP，需根据实际情况进行填写。
* 登录密码默认为 `zhu88jie`。

include::../../../../_components/table_footer.adoc[]


. 从 FirstBox 节点跳转至 core0 节点。
+
[source,shell]
----
ssh core0
----

. 执行如下命令行，查看当前 Hydro 集群列表。
+
[source,shell]
----
docker exec -it hydro_confd /qingstor/bin/hydro cluster list
----

. 命令行回显内容如下所示，确认磁盘是否都正常。
+
[source,shell]
----
Cluster   Type       Disks(Normal)  Total   Used     Use%
Cluster1  Replica 3  36(36)         13.1TB  995.0GB  7%  
----
+
返回字段说明
+
[cols="2,8", options="header"]
|===
|字段 |说明

|Cluster |集群名称，以集群 ID 结尾。
|Type |集群的数据冗余模式，这里的 `Replica 3` 即表示当前集群使用 3 副本的数据冗余模式。
|Disks(Normal) |集群的磁盘总数量与当前正常磁盘数量，这里的 `36(36)` 即表示当前集群中总磁盘数量为 36，其中正常磁盘数量为 36。两个数字相等，说明当前集群中磁盘均正常。
|Total |集群总容量。
|Used |集群当前已使用容量。
|Use% |集群当前已使用容量占比。
|===



. 查看指定集群的详细信息。命令行中 `<cluster_id>` 为集群 ID，需根据上一步查询结果进行填写。
+
[source,shell]
----
docker exec -it hydro_confd /qingstor/bin/hydro cluster <cluster_id> info
----
+
如上输出所示，当前集群名为 `Cluster1`，因此查询命令如下。
+
[source,shell]
----
docker exec -it hydro_confd /qingstor/bin/hydro cluster 1 info
----

. 命令行回显内容如下所示，确认所有磁盘的 `Status` 字段均为 `Normal` 状态，且磁盘容量获取正常即可。
+
[source,shell]
----

Devices
Dev_ID UUID(SPA_ID)                                             Role        Use%               Status

== 检查数据一致性

若当前环境中，某些节点有过断网或者重启，用户需要查看一下等待修复的文件列表。

. 登录当前环境的 FirstBox 节点。
+
[source,shell]
----
ssh -p 6023 root@<firstbox ip>
----
+
include::../../../../_components/note.adoc[]

* 命令行中的 `<firstbox ip>` 为当前环境的 FirstBox 节点 IP，需根据实际情况进行填写。
* 登录密码默认为 `zhu88jie`。

include::../../../../_components/table_footer.adoc[]


. 从 FirstBox 节点跳转至 core0 节点。
+
[source,shell]
----
ssh core0
----

. 执行如下命令行，查看修复队列中的文件列表。命令行中 `<cluster_id>` 为待检查数据一致性的集群 ID，需根据实际情况进行填写。
+
[source,shell]
----
/qingstor/bin/hydro_confd/hydro cluster <cluster_id> heal-info
----

.. 若输出结果如下所示，则说明当前环境中，无等待修复的文件列表。
+
[source,shell]
----
There are no repairing or pending processes
----
