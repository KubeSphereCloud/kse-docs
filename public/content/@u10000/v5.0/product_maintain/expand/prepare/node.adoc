待扩容的物理节点服务器，在扩容前，需根据要求完成操作系统的安装、网络的配置以及磁盘的初始化等。具体操作步骤视节点类型的不同，稍有区别。

== 硬件准备


根据《{product_name} 环境规划指南》中**服务器配置**相关章节内容，准备待扩容节点。


== 网络规划

参考《{product_name} 环境规划指南》网络规划章节内容，为各待扩容节点规划 IP。

include::../../../../../_components/note.adoc[]

* 扩容存储集群时，各存储节点 IP 需为同一网段。
* 扩容网关节点时，待扩容节点 IP 需与原环境中网关节点 IP 保持同一网段。除此之外，待扩容节点还需规划一个 VIP。若待扩容节点需配置公网 IP，则同时需为其额外规划一个 VIP 用于向外提供公网访问。
* 扩容全能节点时，待扩容节点 IP 需为同一网段。除此之外，待扩容节点还需规划一个 VIP。若待扩容节点需配置公网 IP，则同时需为其额外规划一个 VIP 用于向外提供公网访问。

== 操作系统

待扩容节点的操作系统版本，需与原环境中各节点的操作系统版本保持一致。


== 准备硬盘

参考《{product_name} 安装部署指南》**硬盘准备**章节内容，将各待扩容节点上的硬盘进行初始化。
