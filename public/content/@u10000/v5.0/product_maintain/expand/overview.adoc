== 部署方式

{product_name} 环境支持分离部署与混合部署。

* 分离部署
+
分离部署是指将 {product_name} 的网关服务与存储服务分别部署于不同的物理节点服务器上。分离部署的环境中，存在网关节点与存储节点，不存在全能节点，也称为标准部署。

* 混合部署
+
混合部署是指将 {product_name} 的网关服务与存储服务部署于相同的物理节点服务器上。混合部署的环境中，仅存在全能节点。


// == 节点类型

// {product_name} 环境中的节点类型可分为网关节点、存储节点与全能节点。

// * 网关节点
// +
// 网关节点是指部署有 {product_name} 网关服务的节点。

// * 存储节点
// +
// 存储节点是指部署有 {product_name} 存储服务的节点。

// * 全能节点
// +
// 全能节点是指同时部署有 {product_name} 网关服务与存储服务的节点。

// 根据部署方式的不同，{product_name} 环境中的节点类型也不相同。分离部署时，


// 网关节点支持单节点扩容，每扩容一个节点，相应接入能力会获得提 升，累计扩容三个网关节点后，可以将一部分索引 Rebalance 到新节 点。

// {product_name} 环境扩容存储节点时，应以存储集群为单位进行扩容。若存储模式为副本模式，则按照 3 节点每集群的方式进行扩容。 若存储模式为 EC 集群模式，则按照 EC 配比来决定扩容集群节点数。 如:EC 配比为 4+2，则按照 6 节点每集群的方式进行扩容。

== 扩容要求

//=== 节点要求

不同部署模式下，扩容要求稍有不同。

* 分离部署
+
可单独扩容存储节点与网关节点，不支持扩容全能节点。
+
扩容存储节点时，应以存储集群为单位进行扩容。扩容存储节点时，有如下要求。

** 若存储模式为副本模式，则按照 3 节点每集群的方式进行扩容。
** 若存储模式为 EC 集群模式，则按照 EC 配比来决定扩容集群节点数。如，EC 配比为 `4+2`，则按照 6 节点每集群的方式进行扩容。



* 混合部署
+
可扩容独立的存储节点与全能节点，无法扩容独立的网关节点。
+
扩容全能节点或单独扩容存储节点时，其要求与分离部署模式下，扩容存储节点的要求相同。
