---
title: 网关节点配置
weight: 2
---

. 登录原环境的 FirstBox 节点，命令行中的 `<firstbox ip>` 是原环境的 FirstBox 节点 IP。
+
[source,shell]
----
ssh -p 6023 root@<firstbox ip>
----


. 在配置文件 `/qingstor/etc/ip.ini` 中，`[role]`、`[keepalive_int]`、`[nas_vip]` 字段下，按照示例格式，添加待扩容节点信息。
+
[source,yaml]
----
[role]
192.168.10.6 = oss nas cluster1 global
192.168.10.7 = oss nas cluster1 global
192.168.10.8 = oss nas cluster1 global
# 新增节点
192.168.10.2 = oss nas

[keepalive_int]
192.168.10.6: vip=192.168.10.100 inf=eth0
192.168.10.7: vip=192.168.10.101 inf=eth0
192.168.10.8: vip=192.168.10.102 inf=eth0
# 新增节点配置的 vip
192.168.10.2: vip=192.168.10.106 inf=eth0

[nas_vip]
......
192.168.10.6: vip=192.168.10.103 inf=eth0
192.168.10.7: vip=192.168.10.104 inf=eth0
192.168.10.8: vip=192.168.10.105 inf=eth0
# 新增节点配置的 nas vip
192.168.10.2: vip=192.168.10.107 inf=eth0
----
+
**填写说明**
+
[cols="1,5a", options="header"]
|===
| 参数 | 说明 

| role a| 用于配置新扩容网关节点的角色。

* 新扩容的网关节点无需部署 `global` 服务。
* `192.168.10.2` 为待扩容节点的 IP 地址，根据前期规划依次填写。
* `oss` 为对象存储网关服务，可参考原配置内容进行填写。
* `nas` 为文件存储网关服务，可参考原配置内容进行填写。若原配置中没有部署 `nas` 角色，新扩容的全能节点，可根据需求部署 `nas` 角色，从而对外提供文件存储服务。

| keepalive_int a| 用于配置节点虚 IP。示例格式说明。

* `192.168.10.2` 待扩容节点的 IP 地址，根据前期规划依次填写。
* `192.168.10.106` 待扩容节点的 VIP 地址，根据前期规划依次填写。
* `eth0` 网卡名，各节点网卡名必须一致。

| nas_vip a| 填写用于访问文件存储服务的 VIP。示例格式说明。

* `192.168.10.2` 待扩容节点的 IP 地址，根据前期规划依次填写。
* `192.168.10.107` 待扩容节点的 VIP 地址，根据前期规划依次填写。
* `eth0` 网卡名，各节点网卡名必须一致。

|===
