随着容器技术的不断更新和成熟，越来越多的企业在 AI、大数据、边缘计算、高性能计算等新兴业务场景，不断地采用容器技术来构建解决方案。基于容器的实际用户应用正在从无状态应用程序扩展到需要持久化存储支持的有状态应用程序。

自 2013 年 Docker 技术出现以来，容器的持久存储需求逐渐显现，但各存储厂商所提供的直连容器和外部存储系统的存储插件各有不同，直到引入 CSI 接口。

CSI 接口标准化了外部存储系统与许多容器编排系统的集成，容器化的应用基于 CSI 对接底层存储，不需要知道任何底层存储的信息，只需要将应用数据写入文件系统或块设备的容器存储卷，而容器编排系统将透明地处理存储编排和调度的事情。调用逻辑如下图所示：

image::/images/u10000/product_4.png[,100%]

当前支持容器化部署的有状态应用可大致分为以下几类：

* SQL 数据库、NoSQL 数据库。
* 分析应用程序、内容管理。
* 持续集成/持续交付（CI/CD）工具、数据处理、高性能计算、机器学习。

尽管不同应用对容器存储的要求不同，但容器本身的特性和应用数据的同步需求从根本上要求底层存储具备以下能力：

* 数据资源共享
* 易扩展
* 使用、运维简单
* 适应高频率的容器启停


U10000 通过标准的 CSI 接口，无缝对接 K8s 等主流容器平台；支持多种非结构化数据访问协议的互通，并对外提供数据的持久共享访问权限，从而实现容器在集群内任意迁移；同时，U10000 还给用户提供友好的图形化运维管理平台，降低运维成本，充分满足容器场景的数据存储需求。
