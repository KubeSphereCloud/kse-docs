---
title: 创建端口转发
weight: 5
---

由于创建的云服务器过多，而公网 IP 分配过少，避免资源浪费，可给一个公网 IP 配置多个端口转发。本地链接云服务器，可使用 `公网 IP + 端口号` 
的方式，逐个登录云服务器。


. 登录 link:https://console.qingcloud.com/[Console]，输入正确的用户名密码。

. 选择**产品与服务** > **网络服务** > **VPC 网络**，进入 VPC 网络主页面。
+
image::/images/u10000/vpc_list.png[vpc_list,100%]

. 点击已创建的云服务器节点所在的 VPC 网络名称，进入其详情页面。
+
image::/images/u10000/vpc_detail_info.png[vpc_detail_info,100%]

. 点击**管理配置**，进入管理配置标签页，点击**端口转发** > **添加规则**：
+
image::/images/u10000/vpc_config_info.png[vpc_config_info,100%]

. 弹出**添加端口转发规则**对话框：
+
image::/images/u10000/vpc_port_transfer_create.png[vpc_config_info, pdfwidth=60%]
+
**参数说明**
+
[cols="2,5", options="header"]
|===
| 参数 |说明

| 名称	| 用户自定义即可。这里填写 `rul1`。
| 协议	| 选择 `TCP`。
| 源端口	| 可自行定义。
| 内网 IP	a| 填写各云服务器节点的主网卡 IP 地址。


| 内网端口	a| 填写需要映射的内网端口号。

|===


. 参考以上步骤，创建若干条转发规则。分别如下：
** 节点 1 的端口映射规则：源端口号为 `1002`，内网 IP 为 `192.168.100.2`，内网端口号 `22`。
** 节点 2 的端口映射规则：源端口号为 `1003`，内网 IP 为 `192.168.100.3`，内网端口号 `22`。
** 节点 3 的端口映射规则：源端口号为 `1004`，内网 IP 为 `192.168.100.4`，内网端口号 `22`。
** FirstBox 节点的端口映射规则：源端口号为 `1006`，内网 IP 为 `192.168.100.2`，内网端口号 `6023`。
** Portal 与 Console 服务的端口映射规则：源端口号为 `80`，内网 IP 为 `192.168.100.2`，内网端口号 `80`。
** Jenkins 服务的端口映射规则：源端口号为 `1007`，内网 IP 为 `192.168.100.2`，内网端口号 `8888`。

. 完成端口转发规则创建后，点击**应用修改**。

. 查看端口转发的规则列表，显示内容如下：
+
image::/images/u10000/vpc_port_transfer_list.png[vpc_port_transfer_list,100%]
