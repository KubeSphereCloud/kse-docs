---
title: 创建云服务器
weight: 4
---

本文介绍融合部署的场景，因此，这里仅需创建 3 台云服务器，分别绑定一张网卡即可。

. 登录 link:https://console.qingcloud.com/[Console]，输入正确的用户名密码。

. 选择**产品与服务** > **计算** > **云服务器**，进入计算基础服务主页面。
+
image::/images/u10000/vm_main.png[vm_main,100%]

. 在云服务器标签页，点击**创建**。

. 进入**购买云服务器**页面。需依次填写相关参数后，点击**立即购买**。
+
* 填写购买信息
+
image::/images/u10000/vm_basic_info.png[vm_basic_info,100%]
+
**参数说明**
+
[cols="2,5", options="header"]
|===
| 参数 |说明

| 计费方式	|选择 `按需计费`。
| 区域及可用区	| 选择 `北京3区-A`。与前文创建的link:../vpc[私有网络]保持一致即可。

|===


+
* 填写配置信息
+
image::/images/u10000/vm_config_info.png[vm_config_info,100%]
+
**参数说明**
+
[cols="2,5", options="header"]
|===
| 参数 |说明

| 规格	| vCPU 选择 `8 核`，内存选择 `24 G`。根据以上要求，选择相应的实例即可。
| 镜像	| 选择 `标准镜像`。操作系统选择 `Ubuntu`，版本选择 `20.04` 系列即可。
| 存储	a| 

* 系统盘选择 `150 G`。
* 还需额外添加 3 块数据盘分别用作索引盘与数据盘。均选择 `容量型云盘`，大小为 `300 G`，不勾选 `自动格式化`。

| 网络	a| 选择 `VPC 私有网络`。并在下拉框内选择前期已创建的 link:../vpc[VPC 私有网络]。选择 `自动分配 IP 地址`。

| 网卡| 维持默认即可。

| 弹性公网 IP| 选择 `暂不绑定`。

|===

+
* 填写云服务器设置
+
image::/images/u10000/vm_machine_info.png[vm_machine_info,100%]
+
**参数说明**
+
[cols="2,5", options="header"]
|===
| 参数 |说明

| 云服务器名称	| 待创建的云服务器名称。
| 登录方式	| 选择 `SSH密钥`。并在下拉框内选择前期已创建的 link:../add_local_ssh[SSH 密钥]。

| 安全组| 选择 `已有安全组`。并在下拉框内选择 `默认安全组`。
| 高级配置| 维持默认，无需勾选 `现在配置`。

|===


+
* 特惠加购信息，维持默认，无需填写。
+
image::/images/u10000/vm_extra_info.png[vm_extra_info,100%]
+


. 重复以上步骤，创建 3 台云服务器。

. 在云服务器主页面，查看已创建的云服务器列表：
+
image::/images/u10000/vm_list.png[vm_list,100%]

. 记录下各云服务器 IP，这里分别为：`192.168.100.2` `192.168.100.3` `192.168.100.4`。

. 在云服务器主页面，点击云服务器 ID，进入云服务器详情页面，需确认如下信息：
+
* 硬盘成功绑定 3 块。
* 内网 IP 成功绑定，且分别为：`192.168.100.2` `192.168.100.3` `192.168.100.4`。
* SSH 密钥绑定的为前期创建的 link:../add_local_ssh[SSH 密钥]。这里为 `local_ssh`。
* 若以上信息不准确，可根据link:https://docsv3.qingcloud.com/compute/vm/manual/disk/10_mount_disk/[公有云用户手册]进行修改。
