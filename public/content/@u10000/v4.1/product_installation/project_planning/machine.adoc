为达到更好的性能，以及更好的用户体验，在搭建私有 {product_name} 时，{product_name} 建议用户根据要求选用相应的节点服务器。

== 硬件要求

{product_name} 的节点服务器根据用途，可分为全能节点，网关节点与存储节点。从综合性能以及性价比等角度考虑，现对 {product_name} 的节点服务器给出如下配置建议：

=== 全能节点

* CPU 线程数：32~56。
* 内存配置：128G~256G。
* 系统盘：两块系统盘做 RAID 1。单块容量建议 480GB 以上。
* 索引盘：一块 1.92T 以上的 NVME SSD 用于存储索引。也可根据文件数量选择更大容量的 NVME 硬盘作为索引盘，或增加全能节点。索引盘容量估算方式：Object 数量 * 1KB。
* 存储盘：配备 12~36 块 4T~14T 的硬盘用做存储盘，支持 SATA 与 SAS 接口类型的 HDD 和 SSD 类型硬盘。且需采用裸盘直通，并使用 Raid 卡管理硬盘。
* 所有存储节点之间的存储硬盘个数，硬盘容量以及硬盘型号，均需保持一致。
* 如需在物理层面将存储网络与物理网络进行隔离，则需两张双口网卡。若无需进行物理网络分离，一张双口网卡即可。
* 双口网卡可选：10Gbps/25Gbps。

=== 网关节点

* CPU线程数：32~56。
* 内存配置：96G~256G。
* CPU 线程数和内存的比例保持在 1:2.5 ~ 1:4 之间，若超出该比例区间，可增加网关节点。
* 系统盘：两块系统盘做 RAID 1。这里对系统盘容量大小不做要求。
* 索引盘：一块 1.92T 以上的 NVME SSD 用于存储索引。也可根据文件数量选择更大容量的 NVME 硬盘作为索引盘，或增加网关节点。
* 若无需配置外网访问，一张双口网卡即可。如需配置外网访问，则需两张双口网卡。双口网卡可选：10Gbps/25Gbps。

=== 存储节点

* CPU线程数：12~32。
* 内存配置：32G~64G。
* 系统盘：两块系统盘做 RAID 1。这里对系统盘容量大小不做要求。
* 存储盘：配备 12~36 块 4T~14T 的硬盘用做存储盘。支持 SATA 与 SAS 接口类型的 HDD 和 SSD 类型硬盘。且需采用裸盘直通，即无文件系统与数据。
* 所有存储节点之间的存储硬盘个数、硬盘容量以及硬盘型号，均需保持一致。
* 网卡：一张双口网卡即可，10Gbps/25Gbps 两款可选。


== 操作系统

目前 {product_name} 在 x86 服务器上支持的操作系统有 `centos 7.5`、`ubuntu 20.04`、`kylin 10-SP2-x86-Release-Build09-20210524`。ARM 服务器上支持的操作系统为 `ubuntu 18.04`。

{product_name} 推荐用户使用 `ubuntu 20.04`。安装镜像可以从 {product_name} 的 Publisher 服务器上获取。

[source,ftp]
----
ftp://ins.qingcloud.com
----


所有节点的 `root` 账户的初始密码必须一致，且无需创建额外的账户。

=== 硬件配置

|===
| 型号 | 节点数量   | CPU/节点 |  内存/节点 | 系统盘/节点 | 元数据盘/节点 | 数据盘/节点| 网络/节点

| 性能型 | 3-4096   | 2 * Intel Silver CPU |  128GB 及以上 | 2 * 480G SATA SSD 或 2 * 600G SAS HDD  | 1 * 1.6T 及以上 NVMe SSD   | 12 * HDD，单盘容量 4T / 6T / 8T / 10T / 12T  | 2 * 25Gb 双口网卡

| 容量型 | 3-4096   | 2 * Intel Silver CPU |  128GB 及以上 | 2 * 480G SATA SSD 或 2 * 600G SAS HDD | 1 * 1.6T 及以上 NVMe SSD   | 36 * HDD，单盘容量 4T / 6T / 8T / 10T / 12T  | 2 * 25Gb 双口网卡  

|===

**配置信息说明**

* {product_name} 暂无缓存架构，故不需要缓存盘。
* 以上推荐的 CPU、内存为最低配置，用户可根据实际需求做相应扩展。配置的节点均为全能型节点，即同时提供网关服务和存储服务。
* 集群能支持的 `文件或目录数量 = 总节点数/3 * 元数据盘单盘容量/500GB * 50亿`，如：3 节点、1.6T 的元数据盘，根据计算公式，能支持 100-150 亿个文件，也就是能支持百亿级别数量的文件。
* 若配置全闪，一般每节点配置 6-8 块 SSD 盘，最多可配置 12 块 SSD 盘，单盘容量可选：3.84T / 4T / 7.68T / 8T。
* 单块硬盘大小不超过 14T。
* 推荐选用 Mellanox CX5 25G 网卡和光模块。推荐使用 2 块双口网卡，用以实现存储网络和业务网络分离部署，若仅有一块双口网卡，则存储网络和业务网络共用同一个网络。
* 以上硬件配置可能因为市场和技术原因变化。
