---
title: 部署集群
weight: 3
---


节点服务器初始化完成后，可对集群进行安装部署操作。在部署集群之前，请再次确认配置文件是否正确配置完成。

//=== 操作步骤

. 执行如下命令行，登录 FirstBox。命令行中的 `<firstbox ip>` 可查看link:#_创建_firstbox[前文内容]。
+
[source,shell]
----
# ssh -p 6023 root@<firstbox ip>
----

. 进入脚本所在目录：

+
[source,shell]
----
# cd /root/qs-installer
----


. 执行如下命令行，对整个集群进行安装部署。
+
由于命令行包含较多任务，执行完成所需时间较长。若执行过程中报错，该命令会输出相应的错误信息并异常退出。请您根据错误信息进行排查并解决问题后，再次执行该命令行进行部署。
+
[source,shell]
----
# ./qs_installer deploy --all
----

. 命令行执行完成，且无错误信息或异常退出，说明集群已安装部署完成。

// === 命令行解析

// 安装部署使用的命令行 `./qs_installer deploy --all` 是一个聚合命令，包含以下执行步骤：

// [source,shell]
// ----
// # ./qs_installer deploy --configs
// # ./qs_installer deploy --registry
// # ./qs_installer deploy --databases
// # ./qs_installer deploy --hydro-node
// # ./qs_installer deploy --access-node
// ----

// 若用户较熟悉整个部署过程，也可分别执行上述命令行来部署整个集群。每个命令行执行的内容说明如下：

// * 下发网关服务及 Hydro 服务配置到对应角色的节点服务器
// +
// [source,shell]
// ----
// # ./qs_installer deploy --configs
// ----

// * 为所有节点安装 `docker` 与 `docker-registry` 服务
// +
// [source,shell]
// ----
// # ./qs_installer deploy --registry
// ----

// * 安装数据库，包含有：`etcd`、`redis-cache`、`tidb`、`tidb-schema`、`cassandra`、`tikv`、`memcached`
// +
// [source,shell]
// ----
// # ./qs_installer deploy --databases
// ----

// * 部署并初始化 Hydro 集群相关服务，包括有：`qs_index`、`hydro_confd`、`hydro_stor`
// +
// [source,shell]
// ----
// # ./qs_installer deploy --hydro-node
// ----

// * 安装基础服务、对象存储服务、Portal服务、Webconsole服务、监控告警服务、Account 服务以及 `docker_nas` 服务
// +
// [source,shell]
// ----
// # ./qs_installer deploy --access-node
// ----

// == 部署 NTP 服务

// 当前环境是虚拟机搭建的集群，则可跳过该章节相关内容。

// === 操作步骤

// . 执行如下命令行，登录 FirstBox。命令行中的 `<firstbox ip>` 可查看link:#_创建_firstbox[前文内容]。

// +
// [source,shell]
// ----
// # ssh -p 6023 root@<firstbox ip>
// ----

// . 登录 NTP 节点服务器：

// +
// [source,shell]
// ----
// # ssh ntp-server0
// ----

// . 查看 NTP 服务器当前时间：

// +
// [source,shell]
// ----
// # date
// Fri Oct 27 12:35:20 CST 2017
// ----

// . 修改系统时间为当前时间：

// +
// [source,shell]
// ----
// # date -s '2022-09-09 09:09:09'
// ----

// . 更新硬件时钟：

// +
// [source,shell]
// ----
// # hwclock -w
// ----

// . 退出 NTP 服务器，在 FirstBox 上执行如下命令部署 NTP 服务：

// +
// [source,shell]
// ----
// # cd /root/qs-installer
// # ./qs_installer deploy --ntp
// ----

// . NTP 服务成功部署后，需登录各节点服务器，检查各机器的 NTP 时间是否同步：

// +
// [source,shell]
// ----
// # ntpq -pn
// remote           refid   st t when poll reach   delay   offset  jitter
// =======================================================================
// *127.127.1.0     .LOCL.   10 l   38   64    7    0.000    0.000   0.000
// ----

// === 同步结果说明

// . 若 NTP 服务器没有设置上游，在 NTP 服务器上输出结果如下，即 `remote` 为 `127.127.1.0`，`st` 的值小于 16，为正常：

// +
// [source,shell]
// ----
// # ntpq -pn
// remote           refid   st t when poll reach   delay   offset  jitter
// =======================================================================
// *127.127.1.0     .LOCL.   10 l   38   64    7    0.000    0.000   0.000
// ----

// . 若 NTP 服务器没有设置上游，在非 NTP 服务器上输出结果如下，即 `remote` 为 NTP 服务器地址，`st` 的值小于 16 为正常：

// +
// [source,shell]
// ----
// # ntpq -pn
//   remote           refid   st t when poll reach   delay   offset  jitter
// =======================================================================
// *101.100.11.236  LOCAL(0)  11 u  285 1024  377    0.046    0.102   0.112
// ----

// . 若 NTP 服务器与公网或 IaaS 同步，在 NTP 服务器上输出结果如下，即 `remote` 为配置文件中所填写的 `ntp_servers` 地址；`st` 的值小于 16 为正常：

// +
// [source,shell]
// ----
// # ntpq -pn
//   remote           refid   st t when poll reach   delay   offset  jitter
// =======================================================================
// *172.31.47.40  192.168.7.29 4 u  616 1024  377    0.066    1.527   0.522
// +172.31.47.41  192.168.7.29 4 u   32 1024  377    0.079    2.156   0.740
// ----

// . 若 NTP 服务器与公网或 IaaS 同步，在非 NTP 服务器上输出结果如下，即 `remote` 为 NTP 服务器地址，为正常：

// +
// [source,shell]
// ----
// # ntpq -pn
//   remote           refid   st t when poll reach   delay   offset  jitter
// =======================================================================
// *101.100.11.23 172.31.47.40 5 u  897 1024  377    0.053    1.690   1.086
// ----

// === ntpq 调试

// `ntpq -pn` 输出结果异常，或状态异常时，如：`refid` 状态一直为 `.INIT.` 或 `st` 的值为 16，可操作如下步骤进行调试。

// . 若 NTP 与 IaaS 或公网同步，需检查节点与外部 NTP 源之间的网络是否正常，外部源的服务状态是否正常。若 NTP 没有设置上游，可跳过此步。

// . 执行如下命令行，登录 FirstBox。命令行中的 `<firstbox ip>` 可查看link:#_创建_firstbox[前文内容]。

// +
// [source,shell]
// ----
// # ssh -p 6023 root@<firstbox ip>
// ----

// . 登录 NTP 节点服务器，停止 `ntpd` 服务

// +
// [source,shell]
// ----
// # ssh ntp-server0
// # systemctl stop ntpd
// ----

// . 使用 `ntpdate` 命令调试，命令行中的 `<ip_address>` 需替换为 IaaS 的 NTP 服务器 IP 地址或者上游的 NTP 服务器 IP 地址。

// +
// [source,shell]
// ----
// # ntpdate -d <ip_address>
// ----
// +
// * 正常会显示：
// +
// [source,shell]
// ----
// transmit<xxx.xxx.xxx.xxx>
// receive<yyy.yyy.yyy.yyy>
// ----

// * 异常会显示：
// +
// [source,shell]
// ----
// receive: server not found
// ----

// . 如果收到的 `yyy.yyy.yyy.yyy` 跟上游的 NTP 服务器 IP 不同，执行如下命令行，命令行中的 `<yyy.yyy.yyy.yyy>` 使用上一步输出的 `yyy.yyy.yyy.yyy`

// +
// [source,shell]
// ----
// # ntpdate <yyy.yyy.yyy.yyy>
// ----

// . 若上一步没有报错，则修改配置文件中的 `ntp_servers` 为命令行中的 `yyy.yyy.yyy.yyy`。

// . 返回到 FirstBox 上再次执行如下命令行:

// +
// [source,shell]
// ----
// # ./qs_installer deploy --ntp
// ----

// === ntpq -pn 命令回显参数说明

// [cols="2,8a", options="header"]
// |===
// |参数名
// |说明

// |remote
// |响应该请求的 NTP 服务器名称

// |refid
// |NTP 服务器使用的上一级 NTP 服务器。当 `refid` 一直处于下列状态时，说明 NTP 同步可能存在错误

// * `.INIT.` : 关联初始化
// * `.DENY.` : 服务器拒绝访问
// * `.TIME.` : 关联超时
// * `.AUTH.` : 认证错误
// * `.AUTO.` : Autokey顺序错误

// |st
// |`remote` 服务器的级别，正常情况下一般处于 1~15 之间，当为 16 时需要检查是否正常同步

// |t
// |本地 NTP 服务器与远程 NT P服务器的通信方式。

// * `u`：单播
// * `b`：广播
// * `l`：本地

// |when
// |上一次成功请求至现在的秒数

// |poll
// |本地机和远程服务器多少时间进行一次同步，单位为秒。

// |reach
// |测试能否和服务器连接，每成功连接一次它的值就会增加

// |delay
// |表示本地与 `remote` 的延迟

// |offset
// |表示本地与 `remote` 的时间偏移量，单位为：毫秒。`offset` 越接近于 0，主机和 NTP 服务器的时间越接近

// |jitter
// |表示本地与 `remote` 同步时间源的平均偏差，绝对值越小，主机的时间就越精确。

// |===
