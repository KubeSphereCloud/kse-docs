配置文件 ip.ini 用于配置 {product_name} 集群中各节点的角色和网络相关信息。



== 示例

* 基础配置文件示例：
+
无特殊要求，参考示例，正确填写本文件内容即可。
+
[source,ini]
----

[role]
192.168.100.2 = oss nas cluster1 global
192.168.100.3 = oss nas cluster1 global
192.168.100.4 = oss nas cluster1 global

[keepalive_int]
192.168.100.2: vip=192.168.100.212 inf=eth0
192.168.100.3: vip=192.168.100.213 inf=eth0
192.168.100.4: vip=192.168.100.214 inf=eth0

[nas_vip]
192.168.100.2: vip=192.168.100.222 inf=eth0
192.168.100.3: vip=192.168.100.223 inf=eth0
192.168.100.4: vip=192.168.100.224 inf=eth0

----

* 高级配置文件示例：
+
如需分离部署，或提供外网访问等，可参考高级配置文件中的说明进行填写。此处仅列举部分内容进行说明。
+

[source,ini]
----

# extra vip config,no must,"keepalive_ext" do the prefix
[keepalive_ext]
192.168.100.2: vip=139.198.8.204 inf=bond0 mask=27
192.168.100.3: vip=139.198.8.205 inf=bond0 mask=27
192.168.100.4: vip=139.198.8.206 inf=bond0 mask=27

# replace keepalived,use load balancer config
[slb_int]
192.168.100.215
192.168.100.216

[slb_ext]
192.168.100.115
192.168.100.116

----

== 参数说明

* 基础配置文件：
+
[cols="1,4", options="header"]
|===
| 参数 | 说明 

| role a| 用于配置各物理节点的角色。其中各角色均可独立部署，也可部署于相同的节点服务器上。对象存储网关节点、文件存储网关节点以及对象存储服务的 Global 节点最少为 3 个。

在填写配置文件时，role 配置中的第一个节点 IP 必须为上传有安装包的节点服务器 IP。
填写格式参考示例：`节点 IP = 角色`。各角色之间使用空格隔开。

节点角色说明如下：

* `oss` 表示该节点为对象存储网关节点，需部署对象存储网关服务。
* `nas` 表示该节点为文件存储网关节点，需部署文件存储网关服务。
* `global` 表示该节点为对象存储服务的 Global 节点，需要部署对象存储的 Global 网关服务。
* `cluster1` 表示该节点为数据存储池 `cluster1` 的存储节点。数据存储池的配置可查看配置文件 link:../base[base.ini]。
//* 仅部署文件存储时，也需确保集群中有三个节点的角色为 `oss` 与 `global`。

| keepalive_int a| 用于配置节点虚 IP。填写格式参考示例：节点 IP: vip=VIP 地址 inf=网卡名。示例中各字段说明如下：

* `192.168.100.2:` 表示该节点的物理 IP，以 `:` 结尾。
* `vip=10.10.10.212` 表示该节点的 VIP 地址。根据前期规划进行填写。
* `inf=eth0` 网卡名。若当前节点有 `nas` 角色时，各节点网卡名必须一致。

| nas_vip a| 填写用于访问文件存储服务的 vip。填写格式参考示例：节点 IP: vip=VIP 地址 inf=网卡名。示例中各字段说明如下：

* `192.168.100.2:` 表示该节点的物理 IP，以 `:` 结尾。
* `vip=10.10.10.212` 表示该节点的 VIP 地址。根据前期规划进行填写。
* `inf=eth0` 网卡名，各节点网卡名必须一致。

|===

* 高级配置文件：
+
除基础配置文件中内容以外，还可配置如下参数。
+
[cols="1,4", options="header"]
|===
| 参数 | 说明 

| keepalive_ext a| 

* {product_name} 支持给节点服务器配置多组虚 IP 用做不同的用途。
* 多组不同的虚 IP 组命名格式为：`keepalive_ext_` 开头的字符串。
* 填写格式与 `keepalive_int` 相同。可选参数 `mask`，默认使用 `mask=24`，用户可根据实际情况进行修改。
* 节点虚 IP 按照客户需求选择 keepalive 或者负载均衡器 slb 其一种来配置。两种参数同时配置时，部署中会报错。{product_name} 推荐使用 keepalive 的相关配置。



| slb_int & slb_ext a| 


* 填写负载均衡器的 VIP。与 keepalive 字段互斥。两者选其一。
* 分别对应 keepalive 配置中的 `keepalive_int` 与 `keepalive_ext`。
* 填写格式参考示例。


|===
// === role

// 用于配置各物理节点的角色。其中各角色均可独立部署，也可部署于相同的节点服务器上。对象存储网关节点、文件存储网关节点以及对象存储服务的 Global 节点最少为 3 个。

// 需注意的是，在填写配置文件时，role 配置中的第一个节点 IP 必须为上传有安装包的节点服务器 IP。

// **示例**

// [source,ini]
// ----
// [role]
// 192.168.100.2 = oss nas cluster1 global
// 192.168.100.3 = oss nas cluster1 global
// 192.168.100.4 = oss nas cluster1 global
// ----

// ==== 填写格式说明

// [cols="1,4", options="header"]
// |===
// | 参数 | 说明 

// | 192.168.100.2 | 节点物理 IP
// | = | 连接符，前后保留空格
// | oss nas cluster1 global a| 节点角色。
 
// * `oss` 表示该节点为对象存储网关节点，需部署对象存储网关服务。
// * `nas` 表示该节点为文件存储网关节点，需部署文件存储网关服务。
// * `global` 表示该节点为对象存储服务的 Global 节点，需要部署对象存储的 Global 网关服务。
// * `cluster1` 表示该节点为数据存储池 `cluster1` 的存储节点。数据存储池的配置可查看配置文件 link:#_base_ini[base.ini]。

// |===



// === keepalive_int、keepalive_ext

// 业务网络 VIP。填写规则如下：

// * 如需将业务网络进行内外网分离，可同时配置 `keepalive_int`、`keepalive_ext`。
// * 若无需外网，则无需填写 `keepalive_ext` 字段。若需将外网访问根据用途不同，进行区分，可配置多个外网组。命名格式为：`keepalive_ext_` 开头的字符串。
// * 按照客户需求选择 keepalive 或者link:#_slb_intslb_ext[负载均衡器 slb ]其一种来配置。两种参数同时配置时，部署中会报错。{product_name} 推荐使用 keepalive 的相关配置。

// ==== 示例

// [source,ini]
// ----
// [keepalive_int]
// 192.168.100.2: vip=192.168.100.212 inf=eth0
// 192.168.100.3: vip=192.168.100.213 inf=eth0
// 192.168.100.4: vip=192.168.100.214 inf=eth0

// [keepalive_ext]
// 192.168.100.2: vip=10.10.10.212 inf=eth0 mask=27
// 192.168.100.3: vip=10.10.10.213 inf=eth0 mask=27
// 192.168.100.4: vip=10.10.10.214 inf=eth0 mask=27

// [keepalive_ext_bgp]
// 172.16.0.2: vip=172.16.0.212 inf=eth1 mask=27
// 172.16.0.3: vip=172.16.0.213 inf=eth1 mask=27
// 172.16.0.4: vip=172.16.0.214 inf=eth1 mask=27
// ----

// ==== 填写格式说明

// [cols="1,4", options="header"]
// |===
// | 参数 | 说明 

// | 192.168.100.2: | 节点物理 IP，以 `:` 结尾。
// | vip=10.10.10.212 |  为 VIP 地址。根据前期规划进行填写。
// | inf=eth0 |  为网卡名。
// | mask=27| VIP 的掩码。

// |===


// === slb_int、slb_ext

// 填写负载均衡器的 VIP。与link:#_keepalive_intkeepalive_ext[keepalive]字段互斥。两者选其一。

// ==== 示例

// [source,ini]
// ----
// [slb_int]
// 192.168.100.215
// 192.168.100.216

// [slb_ext]
// 192.168.100.115
// 192.168.100.116
// ----

// === nas_vip

// 填写用于访问文件存储服务的 vip。

// ==== 示例

// [source,ini]
// ----
// [nas_vip]
// 192.168.100.222: inf=eth0
// 192.168.100.223: inf=eth0
// 192.168.100.224: inf=eth0
// ----

// ==== 填写格式说明

// [cols="1,4", options="header"]
// |===
// | 参数 | 说明 

// | 192.168.100.2: | 为文件存储网关节点地址。可根据 `role` 中的配置进行填写。
// | vip=10.10.10.212 |  为 VIP 地址。根据前期规划进行填写。
// | inf=eth0 |  为网卡名。

// |===
