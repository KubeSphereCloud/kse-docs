== 对象存储高级功能特性一览


[cols="1a,1a,3a"]
|===
|分类 |高级特性 |描述

.5+^.^|数据安全 |版本管理 |版本管理是针对 Bucket 级别的数据保护功能。开启版本管理后，针对数据的覆盖和删除操作将会以历史版本的形式保存下来。用户在错误覆盖或者删除文件后，能够将文件恢复至任意时刻的历史版本。
|WORM |U10000 提供强合规策略，用户可针对 Bucket 设置基于时间的合规保留策略。当策略锁定后，用户可以在 Bucket 中上传和读取文件，在文件的保留时间到期之前，任何用户都无法删除文件和策略。
|日志归档 |用户可以通过 U10000 提供的日志归档功能将 Bucket 的访问日志按照固定命名规则，以小时为单位生成日志文件写入指定的 Bucket。用户可通过这些日志完成 Bucket 的访问统计、异常事件回溯和问题定位等工作。

|数据加密 |U10000 支持由用户提供密钥的加密方式进行服务器端加密，加密算法支持 AES256。

上传文件时，U10000 对接收到的文件进行加密，再将得到的加密文件持久化保存；下载文件时，U10000 自动将加密文件解密后返回给用户，并在返回的 HTTP 请求 Header 中，声明该文件进行了服务器端加密。

|自动一致性校验 |U10000 支持用户在上传对象的请求头中设置对象实体的 MD5 值，U10000 的服务端会自动根据参数检查该对象在传输过程中是否出错或被篡改。


.2+^.^|数据管理 | 分级存储 |U10000 支持在同一套架构和调度系统下，部署分层的存储集群。异构集群独立扩容互不影响。

U10000 提供 `标准存储`、`低频存储` 和 `冷存储` 三种存储分级。用户可将频繁访问的热数据存放在高性能的存储设备上，将冷数据存放在低成本的存储设备上，提升热点数据的访问性能，降低硬件成本。同时支持自定义分层存储策略，自动触发数据分层迁移，有效解决不常用数据的存储和迁移难题。

| 生命周期管理 |用户可以使用生命周期管理功能来配置一条或多条规则，对 Bucket 中的 Object 进行生命周期管理。通过设置生命周期策略，用户可以定期将 Bucket 内的多个 Object 转换为指定存储类型，或者将过期的 Object 和碎片删除，从而节省存储费用。

也支持将历史版本的 Object 进行删除或转换存储类型。


.2+^.^|数据容灾 | 跨区域复制 |U10000 允许用户开启跨不同的地域之间的 Bucket 自动异步的复制。

跨区域复制数据读取采用最终一致性模型，会将 Object 的创建、更新和删除等操作从源 Bucket 复制到不同区域的目标 Bucket，目标 Bucket 中的 Object 是源 Bucket 中 Object 的精确副本。

跨区域复制支持多站点间的单向复制、双向复制、一对多复制、链式复制等数据同步方式。

该功能用以满足 Bucket 跨区域容灾或用户数据复制的需求。

| 跨可用区冗余 |跨可用区冗余，提供机房级容灾能力。跨可用区冗余是将用户的数据同时存放在同一地域（Region）的 2 个可用区（Zone），当发生断网、断电或者灾难事件导致某个机房不可用时，对象存储仍然能够保障数据的正常访问。

|数据处理 | 图片处理 |支持对存储于 U10000 中的图片执行不同的操作，如格式转换，裁剪，翻转，水印等。
|===

== 文件存储高级功能特性一览


[cols="1a,1a,3a"]
|===
|分类 |高级特性 |描述

|数据管理 | 智能透明压缩 |支持透明智能数据压缩。在数据写入前完成对应的压缩处理后再进行存储，可以明显降低冗余数据的空间占用率，提高空间使用率。

开启数据压缩特性后，存储系统会使用高效压缩算法对新写入的数据进行整合压缩。同时用户可自定义压缩策略，如调整压缩比等，系统在进行数据压缩处理时，会根据用户设定的压缩策略对源文件进行预判断，不满足压缩策略条件的数据会选择性的放弃压缩。最终落盘的数据是否有经过压缩，对于存储系统是透明的。

未经压缩的数据可以直接读取，从而实现存储系统整体性能的最优化。

|数据安全 | WORM |支持租户文件系统级别的数据保留策略，当策略锁定后，用户可以在文件系统中上传和读取文件，但是在文件的保留时间到期之前，任何用户都无法删除文件和策略。

|===
