== U10000 使用方式

[cols="1a,1a,3a"]
|===
|类型 |特性 |说明

|存储访问协议和接口 |* S3

* NFS
* CIFS
* FTP
* HDFS

|支持 S3、NFS、SMB、FTP、HDFS 等多种访问协议和接口，并支持协议互通。

|客户端 |FUSE 客户端 |用户空间文件系统客户端，实现将 U10000 文件系统挂载至用户指定挂载点。

| Qscamel 工具 |用于在不同端点间迁移数据的命令行工具，具备断点续传、自动重试、多并发、支持多种迁移机制等特性。

.2+^.^|GUI 管理界面 | Portal 统一运维管理平台 |用于监控管理 U10000 资源和服务，并为用户提供可视化管理操作界面。
| Console 控制台 |为用户使用 U10000 提供可视化操作界面，直观易用。

|SDK 语言 |* Python

* Go
* Ruby
* C/C++
* Java
* JavaScript
* PHP

|U10000 提供了覆盖主流编程语言的 SDK，开发者可以利用这些 SDK 来快速便捷地接入 U10000。

== U10000 基础功能特性一览


[cols="1a,1a,3a"]
|===
|分类 |功能特性 |描述

.2+^.^|系统架构设计 |多租户 |支持多租户特性。以租户为粒度进行规划、分配统一管理的存储资源，不同租户之间的数据逻辑隔离，从而保障数据安全。
|智能负载均衡 |网关集群通过智能 DNS 服务实现访问会话的负载均衡，存储集群通过对各节点负载周期性采集结果的计算，自动协商选举合适的节点处理 IO 请求，有效减少 IO 请求热点的发生。

|数据冗余 |多副本、EC 纠删码 |支持三副本或 EC 纠删码等多种数据冗余保护机制。

|数据访问 |协议互通 |同时对外提供对象存储、文件存储、大数据存储的能力，支持 SMB/NFS/FTP/HDFS/S3 等多协议存储接口。


.5+^.^|对象存储基础功能 | Bucket 相关基础功能 |包括创建、删除、获取 Bucket 列表等功能。
| Object 相关基础功能 |包括上传、下载、删除对象，获取对象元数据，拷贝、移动、导入、查询对象，自定义元数据等。
| Object 追加写 |为了满足视频监控、视频直播等领域生成的实时视频流场景，U10000 提供了 Appendable 类型 Object，允许直接追加内容。通过追加上传，可在视频数据产生之后及时将数据上传至同一个 Object，而客户端只需要定时获取该 Object 的长度。每次追加上传的数据都能及时可读。
| 分段上传 |U10000 提供了 Multipart API 来实现 Object 分段上传。支持最大上传 50T 的大文件，还能实现客户端的断点续传，分段并发上传等高级特性。
| 配额 |支持租户的容量和存储空间数量配额管理、存储空间级别的容量配额。

.3+^.^|对象存储权限控制 | 访问控制（ACL） |提供了 Bucket 级别的读写权限设置，包括可读、可写和可读写。Bucket 拥有者默认拥有所有权限，另外可配置公开读或公开写(即不附带认证信息的匿名访问)，也可以针对特定青云用户来配置读写权限。
| 存储空间策略（Bucket Policy） |允许用户更细粒度的控制存储空间的访问，其策略可设置支持的 API 操作、生效的资源范围、允许还是拒绝、Referer、请求来源的 IP 地址范围。
例如防盗链设置：仅允许用户以指定站点为 Referer 下载存储空间中的某单个文件。

| 跨源资源共享 (CORS) |跨源访问是浏览器出于安全考虑而设置的一个限制，是用于隔离潜在恶意文件的关键安全机制，也叫同源策略。U10000 通过设置 CORS，可以对请求来源的域名和请求头进行控制，允许 Web 应用服务器进行跨源访问控制，确保跨源数据传输的安全性。


.3+^.^|文件存储基础功能 | 文件系统相关基础功能 |包括创建、删除、获取租户文件系统信息等功能。
| 文件系统共享 |支持通过 NFS、SMB 等协议共享租户文件系统。
| 配额|支持租户文件系统级别的容量配额管理，可通过设置合理的配额策略，限制非合理用户或业务的存储资源使用量，避免因为非正常用户大量的无效数据造成存储系统资源浪费，从而提高存储系统的有效利用。

|文件存储权限控制 | 访问控制（ACL） |支持租户文件系统级别的读写权限设置，包括只读、读写，同时支持权限设置对授权地址下 root 用户或所有用户生效，用户可根据实际情况设置 ACL 权限优先级。
|===
