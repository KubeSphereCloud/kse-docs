使用 {product_name} 的文件存储的第一步，就是需要用户创建一个文件系统，设置好该文件系统的容量配额以及读写数据块大小等。操作步骤如下。

. 成功登录 Portal。

. 点击**存储服务 > 文件存储 > 文件系统**，进入文件系统列表页面。
+
image::/images/u10000/stor_file_system_main.png[stor_file_system_main,100%]

. 在文件系统列表页面点击**创建文件系统**。

. 弹出**创建文件系统**对话框，用户根据页面提示信息填写相关参数后，点击**确定**。
+
image::/images/u10000/stor_file_system_create.png[stor_file_system_create,40%]
+
**参数说明**
+
[cols="1,3a", options="header"]
|===
| 参数 |说明

|配额 | 文件系统可用的存储空间大小，管理员可按需合理分配，避免出现资源争抢或过渡使用资源的情况。
|数据块大小 | 写入的数据块大小。建议使用 `自适应`。
|预读数据块大小 | 预先读取的数据块大小。建议使用 `自适应`。
|===
+
include::../../../../../../../../_components/note.adoc[]

若用户在比较明确业务场景和读写 IO 模型的前提下，可以对**数据块大小**与**预读数据块大小**两个参数进行指定，以提升性能。例如，用户的业务 IO 以 `4k` 操作为主，且大多是顺序读写，那最好就指定**数据块大小**和**预读数据块大小**均为 `4k`。
