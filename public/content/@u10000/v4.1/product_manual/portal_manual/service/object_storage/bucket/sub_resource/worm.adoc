WORM 为用户提供数据合规保留设置。用户根据需求，可选择不同级别锁：

* 针对 Bucket 设置桶级别锁，在保留期内，任何用户都无法删除和覆盖 Bucket 内的对象，实现保护数据或满足监管要求的目的。

* 设置对象版本锁，根据选择的模式受保护的对象版本不被覆盖或删除，此级别可兼容 S3 Object Lock。

include::../../../../../../../../_components/note.adoc[]

Portal 页面设置的 `对象版本锁` 为 Bucket 的默认配置。Bucket 启用 `对象版本锁` 后，后续上传的对象均使用该默认配置进行保护。若用户需差异化对象的保留模式与保留时间，可以使用 API 上传对象，并携带相应的请求头来指定保留模式与保留时间。

include::../../../../../../../../_components/table_footer.adoc[]

== 启用 WORM

. 在link:../../sub_resource/[子资源详情]页面，点击 **WOMR**，进入 WORM 功能设置页面。
+
image::/images/u10000/stor_bucket_worm.png[stor_bucket_worm,100%]
 
. 点击**启用**，根据页面提示信息，填写相关参数后，点击**启用**按钮。

** 数据保留级别选择**桶级别锁**
+
image::/images/u10000/stor_bucket_worm_1.png[,50%]

** 数据保留级别选择**对象版本锁**
+
image::/images/u10000/stor_bucket_worm_2.png[,50%]

** 参数说明
+
[cols='1,3a']
|===
| 参数名称 | 说明 

| 数据保留级别 | * 设置 `桶级别锁`，则存储于当前 Bucket 中的所有数据均受保护。

* 设置 `对象版本锁`，则存储于 Bucket 中的历史数据不受保护，后续上传至 Bucket 中的数据受保护。
* 推荐使用 `桶级别锁`。


| 数据保留时间 | 用于设置数据受保护的时间，在该期限内，数据不能被删除或修改。

* 设置 `桶级别锁` 且已 `锁定` 时，用户可延长数据保留时间。
* 设置 `桶级别锁` 且未 `锁定` 时，用户可以延长或缩短数据保留时间。
* 设置 `对象版本锁` 时，数据保留时间的修改，参考 `数据保留模式` 的设定。


| 默认保留配置 | 用于设置对象版本锁的默认保留配置。上传对象，若不另外指定保留配置，则使用默认保留配置。

| 数据保留模式 | 当设置 `对象版本锁` 时，需要同时指定数据保留模式。

* `治理模式` 下，对象可以修改数据保留模式为 `合规模式`，还可以延长或缩短数据保留时间。
* `合规模式` 下，对象不可以修改数据保留模式，可以延长但不能缩短数据保留时间。

|===

. `桶级别锁` 启用后，还需手动进行 `锁定`，未锁定时数据仍受保护但可以编辑和删除数据合规保留设置。


== 修改 WORM 策略

开启 `桶级别锁` 并锁定后，只能延长保留时间；未锁定时，可延长，也可缩短保留时间。`对象版本锁` 可修改默认保留时间与保留模式，该修改对后续上传的对象版本生效。


不支持 `桶级别锁` 与 `对象版本锁` 的转换。

== 删除 WORM 策略

Bucket 的 WORM 功能一旦开启后，就不能取消。WORM 作为 Bucket 的子资源，只能随 Bucket 的删除而删除。

. 待 Bucket 内的对象保护均到期后，逐一删除 Bucket 内对象。

. 删除 Bucket。

. 待 Bucke 成功删除后，创建同名 Bucket，以达到删除 Bucket 的 WORM 设置。
