:ks_db_product: OpenSearch

本节介绍如何在 DMP 数据库管理平台上创建 RadonDB {ks_db_product} 实例。


== 前提条件

* 在{ks_product_both}平台加入一个企业空间和一个项目。
* {ks_product_right}平台需要安装 RadonDB DMP 扩展组件。

== 操作步骤

. 以 platform-admin 角色登录{ks_product_left} Web 控制台并进入数据库管理平台。
. 在左侧导航栏选择 **{ks_db_product}**。
. 在页面右侧选择**新建 {ks_db_product} 实例**。
. 在**基本信息**页签，设置 RadonDB {ks_db_product} 实例的基本信息和位置，然后点击**下一步**。
+
--
[%header,cols="1a,4a"]
|===
|参数 |描述

|名称
|{ks_db_product} 实例的名称。名称只能包含小写字母、数字和连字符（-），必须以小写字母开头并以小写字母或数字结尾，最长 32 个字符。

|描述
|{ks_db_product} 实例的描述信息。描述可包含任意字符，最长 256 个字符。

|位置
|{ks_db_product} 实例所属的企业空间和项目。
|===
--

. 在**应用设置**页签，设置 {ks_db_product} 实例的参数。
+
--
[%header,cols="1a,4a"]
|===
|参数 |描述

|版本
|{ks_db_product} 应用的版本。

|数据库内核
|{ks_db_product} 数据库的内核版本。

|主节点/热节点/温节点/冷节点/控制面板节点/Logstash 节点设置

|**资源**：分配给每个节点的 CPU 和内存资源。

**主节点/热节点/温节点/冷节点/控制面板节点/Logstash 节点数量**：{ks_db_product} 实例各类型节点的数量。主节点的数量默认为 **3** 且不能修改，其他节点数量可根据需要设置。

**存储类**：节点使用的存储系统对应的存储类。默认值为 **local**。如果下拉列表中没有符合需要的存储类，请联系平台管理员创建存储类。

**卷**：每个节点使用的卷大小，单位为 GiB。每个主节点的卷大小默认为 20 GiB。控制面板节点和 Logstash 节点不能挂载卷。其他节点的卷大小可根据需要设置。
|===

点击**高级设置**，设置数据库的⽤⼾名和密码（默认用户名为 admin，密码为 RadonDB@123），启用或禁用**反亲和性配置**和 **node 亲和设置**。

[%header,cols="1a,4a"]
|===
|设置
|描述

|反亲和性配置
|开启反亲和性配置后， pod 将优先调度到不同的 node 上。

|node 亲和设置
|开启 node 亲和设置后，添加匹配 node 的键值对，可以将 pod 部署在特定条件的 node 上。

* 点击**添加**可以设置多个键值对。 
* 在已添加的键值对右侧点击image:/images/dmp/icons/trash-light.svg[trash,18,18]可删除键值对。
|===

--

. 点击**确定**。RadonDB {ks_db_product} 实例创建后，将显示在 {ks_db_product} 页面的实例列表中。
