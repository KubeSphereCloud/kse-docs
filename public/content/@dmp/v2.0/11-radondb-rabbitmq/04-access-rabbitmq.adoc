:ks_db_product: RabbitMQ

本节介绍如何访问 RadonDB {ks_db_product}。


== 前提条件

* 已获取{ks_product_both}平台登录账号和密码，且已获取平台操作管理权限。

* RadonDB {ks_db_product} 实例状态为**运行中**。

== 操作步骤

根据是否配置 TLS，访问 {ks_db_product} 的方式略有不同。

=== 未配置 TLS

. 以 platform-admin 角色登录{ks_product_left} Web 控制台并进入数据库管理平台。
. 在左侧导航栏选择 **{ks_db_product}**。
. 在 {ks_db_product} 实例列表中，点击一个实例名称打开其详情页面。
. 在详情页左侧，查看**访问地址**，示例如下。
+
image:/images/dmp/access-rabbitmq.png/[ ,70%]

. 在**参数管理**页签，查看默认的用户名 default_user（默认为 **guest**）和用户名密码 default_pass（默认为 **guest**）。

. 将鼠标悬停至 Web 控制台右下角的image:/images/dmp/icons/hammer.svg[hammer,20,25]，点击 **kubectl**，在弹出的对话框执行以下命令访问 {ks_db_product}。

. 若在集群内部访问 {ks_db_product}，执行以下命令：
+
--
[,bash]
----
rabbitmqadmin --host=<instance_name>.<namespace> --port=15672 --username=<user> --password=<password>
----

* <instance_name>：{ks_db_product} 实例的名称。
* <namespace>：{ks_db_product} 实例所在的命名空间（即项目名称）。
* port 默认为 15672，可以不指定 --port 参数。
* <user>：默认为 guest。如果通过命令行新增了 {ks_db_product} 用户，请替换为新用户名。
* <password>：默认为 guest。如果通过命令行新增了 {ks_db_product} 用户，请替换为新用户的密码。


示例命令如下：

[,bash]
----

rabbitmqadmin --host=radondb-feoed2.test --port=15672 --username=guest --password=guest

----
--

. 若在集群外部访问 {ks_db_product}，执行以下命令：
+
--
[,bash]
----
rabbitmqadmin --host=<instance_address> --port=<NodePort> --username=<user> --password=<password>
----

* <instance_address>：{ks_db_product} 实例的连接地址，如上图所示的 172.31.73.234。在实例详情页面的左侧区域查看对应的 IP 地址，并替换为实际地址。
* <NodePort>：{ks_db_product} 实例的端口号，请替换为实际端口号。可执行 kubectl get svc -n <namespace> 获取该实例对应的 NodePort。
* <user>：默认为 guest。如果通过命令行新增了 {ks_db_product} 用户，请替换为新用户名。
* <password>：默认为 guest。如果通过命令行新增了 {ks_db_product} 用户，请替换为新用户的密码。

--

=== 已配置 TLS

若已配置 TLS，访问 {ks_db_product} 的步骤同上，仅执行的命令不同。

* 配置 TLS 后，在集群内部访问 {ks_db_product}，执行以下命令：
+
--
[,bash]
----
rabbitmqadmin --host=<instance_name>.<namespace> --port=15672 --username=<user> --password=<password> --ssl --ssl-key-file=/path/to/client_key.pem --ssl-cert-file=/path/to/client_certificate.pem

----

* <instance_name>：{ks_db_product} 实例的名称。
* <namespace>：{ks_db_product} 实例所在的命名空间（即项目名称）。
* port 默认为 15672，可以不指定 --port 参数。
* <user>：默认为 guest。如果通过命令行新增了 {ks_db_product} 用户，请替换为新用户名。
* <password>：默认为 guest。如果通过命令行新增了 {ks_db_product} 用户，请替换为新用户的密码。
* 将 /path/to/client_certificate.pem 和 /path/to/client_key.pem 替换为实际的路径。
--

* 配置 TLS 后，在集群外部访问 {ks_db_product}，执行以下命令：

+
--
[,bash]
----
rabbitmqadmin --host=<instance_address> --port=<NodePort> --username=<user> --password=<password> --ssl --ssl-key-file=/path/to/client_key.pem --ssl-cert-file=/path/to/client_certificate.pem
----

* <instance_address>：{ks_db_product} 实例的连接地址，如上图所示的 172.31.73.234。在实例详情页面的左侧区域查看对应的 IP 地址，并替换为实际地址。
* <NodePort>：{ks_db_product} 实例的端口号，请替换为实际端口号。可执行 kubectl get svc -n <namespace> 获取该实例对应的 NodePort。
* <user>：默认为 guest。如果通过命令行新增了 {ks_db_product} 用户，请替换为新用户名。
* <password>：默认为 guest。如果通过命令行新增了 {ks_db_product} 用户，请替换为新用户的密码。
* 将 /path/to/client_key.pem 和 /path/to/client_certificate.pem 替换为实际的路径。
--
