== 问题一

**Q：为什么按流量收费的公网 IP，实际用量与单价乘积计算出的总价与页面显示总价不一致？**


A：由于按流量计费的公网 IP 是每小时扣除一次费用，且扣除的费用只保留小数点后 4 位，所以在一段时间内按流量计费的公网 IP 所产生的总费用的计算值和页面显示值会存在微量误差，这是由金额合并导致的精度缺失。

例如，公网 IP 单价为 0.96 元/GB，用量为 27.8164GB，则计算费用为 ¥26.703744，实际扣费为 ¥26.7037。故假设一天 24 小时的用量都一样，那么一天的用量 667.5936，用量费用计算值为 ¥640.889856，但是实际扣费为 ¥26.7037 x 24 = ¥ 640.8888。


== 问题二

**Q：为什么按需扣费的资源，在持续使用过程中，设置按天查看账单明细时，其用量显示会出现不为 24 小时的情况？**

A：按天查看账单明细时，针对资源用量显示不低于 23 小时，不高于 25 小时属于正常现象，这是由于计费的机制导致的。

按需计费的实际扣款方式是，每小时计费系统会提前扣除接下来一小时的费用，故此会出现以下 3 种情形：

* 若用户继续使用，会在一个小时候后，继续提前扣除下一小时的费用。
* 若用户在一个小时内删除资源，则系统会自动退回提前扣除的剩下时长的费用。
* 若用户在一个小时内进行了使资源价格发生变化的操作，则会根据实际用时扣除相关费用后，结束以当前价格扣费模式，以新的价格开始下一小时的扣费。

**示例**：

在账单明细页面查看 2023-11-14 当天平台内各项资源的用量及费用信息，其中实例 ID 为 i-uw3d8nzo 的云服务器，其用量为 23.8 小时。

image::/images/cloud_service/services/bill_center/faq_1.png[]

点击其 ID 进入相应的详情页面，在**租赁信息**卡片，点击image:/images/cloud_service/services/bill_center/faq_2.png[]，选择**查看详情**。如图所示，在 2023-11-14 00:32:27 该云服务器发生重启，其每日的资源用量统计周期从当日的 00:44:40 到次日的 00:40:40，改变为当日的 00:32:27 至次日的 00:32:37。

image::/images/cloud_service/services/bill_center/faq_3.png[,70%]

故此，在每日资源用量统计周期起止时间发生变化的前提下，其计算过程为：

2023-11-14 00:32:27 云服务器发生重启，此时并未到达前一天的用量统计及扣费周期的截止时间，故直至 00:44:40 开始第一小时扣费，截止到当日 24:00:00，总计使用时长为 23 小时 15 分钟 20 秒。

从 2023-11-14 24:00:00 到次日（2023-11-15）的 00:32:27，总计使用时长为 32 分钟 27 秒。

综上，在 2023-11-14 当天整个资源用量统计周期内，总计使用时长为 23 小时 47 分钟 47 秒，约 23.8 小时。
