DNAT 包含公网 DNAT 和私网 DNAT，详细的功能说明和使用场景如下表所示。

|===
| 类型 | 功能说明 | 场景

| 公网 DNAT
| 将公网 IP 映射为私有网络 IP，外部网络可以通过映射后的地址访问内部网络，从而使内部网络可以对外提供服务。
| 用户需要通过 Internet 访问内部服务。 +


| 私网 DNAT
| 将内部私有网络 IP 映射为一个指定私有网络IP（该指定私有网络 IP 在网络中不能存在冲突）。
a|* 需要云上隐藏真实云下内部私网 IP 。
* 分支之间本地网络存在冲突，搭配 SNAT 功能解决云上和云下有双向互访需求。
|===

== 前提条件

* 已创建连接云网。
* 已创建接入点。

== 添加 DNAT 规则

. 登录管理控制台。
. 选择**产品与服务** > *网络服务* > *SD-WAN（新版）*，进入**连接云网**页面。
+
image::/images/cloud_service/sd_wan/qs_cloud_network.png[]


. 在左侧导航栏中，点击**接入点**，进入**接入点**页面。
+
image::/images/cloud_service/sd_wan/qs_light_access.png[]

. 点击接入点名称，进入接入点详情页面。
. 选择**网络管理** > *DNAT*，进入 *DNAT* 页面。
+
image::/images/cloud_service/sd_wan/um_dnat_list.png[]


. 点击**添加规则**，弹出**添加 DNAT 规则**窗口。
+
image::/images/cloud_service/sd_wan/um_dnat_win.png[,60%]

+
参数说明，如下表所示。
+
|===
| 参数 | 参数说明

| 名称
| 设置 DNAT 的名称。

| 类型
a|* 公网 DNAT：将公网 IP 映射为内部网络地址。
* 私网 DNAT：将内部网络转换为指定私网 IP 地址。

| 外网端口
| 外网的端口号。

| 协议
| 可以选择为 TCP、UDP、ICMP 协议。

| 内网 IP
| 云服务器的私有网络 IP 地址。

| 内网端口
| 云服务器对公网公开的端口号，例如：80、443、22 等。
|===

. 点击**添加**，完成 DNAT 规则的添加操作。
+
若页面弹出**新增成功**提示信息，则说明新增 DNAT 规则成功。

. 点击**应用修改**，使配置生效。
+
添加 DNAT 规则后默认为**启用**状态。

== 注意事项

* 可以针对同一个内网云服务器的内网端口添加多条 DNAT 规则，只要采用不同的外网端口即可。
+
image::/images/cloud_service/sd_wan/um_dnat_note01.png[]

* 填写的源端口和内网端口可以不同。
* 可以针对同一个内网云服务器的多个内网服务添加多条 DNAT 规则。例如：SSH、HTTP、HTTPS 等。

* 需要放行 VPC 所在安全组源端口对应的下行规则，并**应用修改**安全组规则。例如：SSH、HTTP、HTTPS 等。
+
image::/images/cloud_service/sd_wan/um_dnat_note02.png[]
+
添加完成的安全组规则，如下图所示。
+
image::/images/cloud_service/sd_wan/um_dnat_note03.png[,80%]

* 若需要指定 IP 或者 IP 段访问源端口的服务，可以配置安全组的源 IP（相当于白名单）。
+
image::/images/cloud_service/sd_wan/um_dnat_note04.png[,80%]

+
添加的安全组规则，如下图所示。

== 测试连通性

可以登录云服务器，执行 telnet 命令，测试端口的连通性。
[,shell]
----
telnet ip 端口
----

== 相关操作

=== 禁用 DNAT 规则

. 进入**网络管理** > *DNAT* 页面。
+
image::/images/cloud_service/sd_wan/um_dnat_list.png[]

. 在已添加的静态路由所在行的**操作**列中，点击**禁用**。
+
若页面 DNAT 规则的状态变成为**禁用**，则说明禁用成功。

. 点击**应用修改**，使配置生效。

=== 修改 DNAT 规则

. 进入**网络管理** > *DNAT* 页面。
+
image::/images/cloud_service/sd_wan/um_dnat_list.png[]


. 在已添加的静态路由所在行的**操作**列中，点击**修改**，弹出**修改 DNAT 规则**窗口。
+
image::/images/cloud_service/sd_wan/um_modify_dnat.png[]


. 修改完成后，点击**确定修改**，完成 DNAT 规则的修改。
+
若页面弹出**编辑成功**提示信息，则说明修改 DNAT 规则成功。

. 点击**应用修改**，使配置生效。

=== 删除 DNAT 规则

. 进入**网络管理** > *DNAT* 页面。
+
image::/images/cloud_service/sd_wan/um_dnat_list.png[]

. 在已添加的静态路由所在行的**操作**列中，点击**删除**，弹出**删除 DNAT 规则**窗口。

. 点击**确定**，完成 DNAT 规则的删除操作。
+
若页面弹出**删除成功**提示信息，则说明删除 DNAT 规则成功。

. 点击**应用修改**，使删除生效。
