本章节介绍如何快速登录云服务器。

== 登录方式

您可以根据云服务器的操作系统选择登录云服务器的方式。

[cols="2*a", options="header"]
|===
| 登录方式 | 满足条件

| Web 终端登录（Windows & Linux）
| 登录用户名和密码

| SSH 登录（Linux）
| * 登录用户名
* 云服务器link:../../../ssh/manual/ssh/[已绑定 SSH 密钥]
* 云服务器已开放 TCP 22 端口
* 云服务器link:../app_public_ip/[已绑定公网 IP 地址]
|===

== Web 终端登录

. 登录 {console_name}。
. 选择 *产品与服务* > *计算* > *云服务器*，进入云服务器列表页面。
+
image::/images/cloud_service/compute/vm/qs_login_vm_1.png[]

. 点击 *Web 终端*，弹出 *Web 终端* 窗口。
+
image::/images/cloud_service/compute/vm/qs_login_vm_2.png[]

. 在 Web 终端窗口中，输入您创建云服务器时设置的账号和密码，按 Enter，即可登录到云服务器。
 ** *Windows*
+
image::/images/cloud_service/compute/vm/qs_login_vm_3.png[]

 ** *Linux*
+
image::/images/cloud_service/compute/vm/qs_login_vm_4.png[]

== SSH 登录（Linux）

* 如果创建云服务器时，设置为 SSH 密钥方式登录，那么在使用 SSH 登录之前，需要先打开安全组规则的 TCP 22 端口。

* 由于操作系统 CentOS 6.8 的 OpenSSH 版本过低，从较高版本的 OpenSSH 主机访问时可能出现错误，可参考link:../../faq/troubleshooting/connect_manage/no_matching_host_key[常见问题]相关内容进行解决。

=== 本地是 Linux 或 Mac

通过本地 SSH 密钥（如：kp-1234abcd ）直接使用远程命令 SSH 安全地登录云服务器。

[source,shell]
----
chmod 600 /path/to/kp-1234abcd
ssh -i /path/to/kp-1234abcd root@ip_address
----

例如：SSH 密钥 `kp-1234abcd`，连接云服务器 `172.31.45.132`。

[source,shell]
----
chmod 600 /path/to/kp-1234abcd
ssh -i /path/to/kp-1234abcd root@172.31.45.132
----

=== 本地是 Windows

使用 Xshell 5 及以上版本登录。

例如：在 Windows 下使用 SSH 密钥 `kp-1234abcd`，云服务器IP 地址 `172.31.45.132`。

. 打开 Xshell。
. 创建一个新会话，协议是 SSH，输入云服务器的 IP 地址。
+
image::/images/cloud_service/compute/vm/qs_login_vm_5.png[]

. 点击 *用户身份验证*，进入 *用户身份验证* 页面。
+
image::/images/cloud_service/compute/vm/qs_login_vm_6.png[]
+
参数说明，如下表所示。
+
[cols="2*a", options="header"]
|===
| 参数 | 参数说明

| 方法(M)
| 选择 `Public Key`。

| 用户名(U)
| 输入 `root`。

| 用户密钥(K)
| 点击 `浏览(B)`，导入绑定云服务器的 SSH 密钥公钥文件对应的私钥文件。
|===

. 点击 *连接*，即可连接到云服务器。
