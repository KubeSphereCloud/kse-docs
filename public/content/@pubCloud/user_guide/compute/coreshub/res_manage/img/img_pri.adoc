{coreshub_name_cn}支持用户基于预置镜像或 Dockerfile 构建自定义镜像。

== 前提条件

* 已经获取 {console_name}账户和密码。
* 已完成个人实名认证且账户余额大于 0 元。

== 创建镜像

. 登录 {console_name}。
. 在顶部导航栏，点击**产品与服务** > **{coreshub_name_cn}** > **{coreshub_name_cn}**，进入其**总览**页面。
. 在左侧导航栏，选择**镜像仓库**，默认进入**预置镜像**列表页面。
. 选择**自定义镜像仓库**页签，点击**新建镜像**。
+
image::/images/cloud_service/compute/coreshub/res_manage_img_pri_1.png[]
. 在弹出**新建镜像**窗口中输入**镜像名称**和**描述**，点击**确定**即可。
+
include::../../../../../../_components/note.adoc[]
* **镜像名称**要求不少于 3 个英文字符，且只能以小写字母或数字开头，支持使用“小写英文字母/数字/_/-/.”。 
* **描述**为选填项，用户根据实际情况进行填写即可。
include::../../../../../../_components/table_footer.adoc[]

== 构建镜像

. link:#_创建镜像[镜像创建]完成后，点击待构建的**镜像名称**，进入其详细信息页面。
+
image::/images/cloud_service/compute/coreshub/res_manage_img_pri_2.png[]

. 在**标签列表**页，点击**构建镜像**。
. 在弹出的构建镜像窗口中，配置各项参数，点击**确定**。
+
[options="header",cols="1,3"]
|===
|参数 |说明
|标签
|当前构建的镜像的标签，用户自定义。

|镜像类型
a|支持**基于预置镜像**或**基于 Dockerfile**

* 若选择**基于预置镜像**，需配置以下参数。
** 预置镜像：用户需根据实际需要，选择待配置依赖的**预置镜像**。
** 依赖：用户根据实际需要，输入相应依赖的安装命令，命令行间用换行符分割。例如，``pip install tqdm``。

* 若选择**基于 Dockerfile**，需配置以下参数。
** Dockerfile：自动化构建 Docker 镜像的文本文件，需用户自行编写。Dockerfile 中不支持 ADD 和 COPY 关键字。

|===

. 构建成功的镜像，显示在镜像的详情页的**标签列表**内，且状态为``成功``。用户可在当前页面获取相应镜像的**拉取命令**。
