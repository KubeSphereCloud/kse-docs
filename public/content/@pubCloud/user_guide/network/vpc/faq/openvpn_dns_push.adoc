== 案例场景

客户希望 VPC 内网配置的域名解析，通过 VPN 拨入到本地生效，无需手动去配置 hosts 解析。

== 操作步骤
. 配置内网解析的域名如下。
+
image::/images/cloud_service/network/vpc/openvpn_dns_push_01.png[,60%]

. 选择 **VPN服务** > *OpenVPN* > *修改*，在附加配置里增加以下内容。
+
[source]
----
push "route 172.20.0.0 255.255.0.0"
push "dhcp-option DNS 172.20.255.254"
----
+

include::../../../../../_components/note.adoc[]

push 的路由需要按照指定的格式。

include::../../../../../_components/table_footer.adoc[]


. 本地拨入 VPN 以后查看是否生效，若没有生效，刷新 DNS 缓存。
+
[source]
----
ipconfig /flushdns
----
+
如下图，解析已经生效。
+
image::/images/cloud_service/network/vpc/openvpn_dns_push_02.png[]

. 如内网 DNS 配置没有生效，需要离开私有网络重新加入私有网络时手动指定内网 IP 地址，或者手动更改 /etc/resolv.conf 的配置，格式如下。
+
[source,shell]
----
[root@i-tqzv58ti ~]# cat /etc/resolv.conf
# Generated by NetworkManager
search localdomain pek3.qingcloud.com
nameserver 172.20.255.254
----
