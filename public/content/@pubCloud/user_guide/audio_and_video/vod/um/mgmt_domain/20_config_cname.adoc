本章节介绍如何配置 CNAME 解析。

== 前提条件

* 已获取管理控制台的账号和密码。
* 已开通云点播服务且link:../10_add_domain/[添加自有域名]。

== 操作步骤

如下**以在{platform_shortname}的云解析 DNS 服务中配置 CNAME 值为例**进行 CNAME 配置说明。

. 登录管理控制台。
. 选择**产品与服务** > *音视频服务* > *云点播*，进入**云点播**的**音视频管理**页面。
. 在左侧导航栏中，点击**域名管理**，进入**域名管理**页面。
+
image::/images/cloud_service/audio_and_video/vod/qs_domain_cname.png[]

. 在域名所在行的 CNAME 列中，点击image:/images/cloud_service/audio_and_video/vod/icon_copy.png[20px,20px]，获取域名的 CNAME 值。
. 选择**产品与服务** > *域名与网站* > *云解析 DNS* ，进入**云解析 DNS 服务**页面。
+
image::/images/cloud_service/audio_and_video/live_cdn/um_dns_list.png[]

. 点击**添加**，将**已注册备案**的域名添加到云解析 DNS 服务列表中。
+
image::/images/cloud_service/audio_and_video/live_cdn/um_add_domain.png[]

. 点击已添加的域名的名称，进入域名**解析记录**页面。
+
image::/images/cloud_service/audio_and_video/live_cdn/um_add_parsing.png[]

. 点击**添加记录**，分别添加推流域名和播流域名对应的 CNAME 记录值。
+
image::/images/cloud_service/audio_and_video/live_cdn/um_add_domainlist.png[]
+
配置说明，如下表所示。
+
[cols="1,2"]
|===
| 参数 | 参数说明

| 云服务器
a| 云服务器名就是域名前缀。

* *www：解析后的域名为：www.example.xyz。
* @：直接解析主域名 example.xyz。
* *：泛解析，匹配其他所有域名 *.example.xyz
* mail：将域名解析为：mail.example.xyz，通常用于解析邮箱服务器。

| 线路
| 选择**全网默认**。

| 类型
| 选择 *CNAME*。

| 记录值
| 配置**步骤 3** 获取的 CNAME 值。

| 其他参数
| 您可以根据自己的实际情况填写。
|===

. 配置完成后，点击**添加**。完成 CNAME 值的配置。
+
配置的 CNAME 生效后，您的自有域名的所有请求都将通过青云的云点播服务进行加速播放。

== 验证 CNAME

=== Windows

. 在 Windows 系统的运行对话框中，输入 *cmd*，按 *Enter*，打开 DOS窗口。
+
image::/images/cloud_service/audio_and_video/live_cdn/um_cname_cmd_win.png[,50%]

. 执行以下命令，验证 CNAME 是否配置成功。
+
若回显信息中，成功解析配置的 CNAME，则 CNAME 配置成功。
+
[source]
----
nslookup <域名>
----


=== Linux/Mac

. 打开 Linux/Mac 窗口。
+
image::/images/cloud_service/audio_and_video/live_cdn/um_cname_cmd_linux.png[,50%]

. 执行以下命令，验证 CNAME 是否配置成功。
+
若回显信息中，成功解析配置得 CNAME，则 CNAME 配置成功。
+
[source]
----
dig <域名>
----
