|===
|运维事项	|描述
|Zookeeper 集群可容忍故障的节点数	
|由于 Zookeeper 是 NeonSAN 的仲裁和选主服务，故当 Zookeeper 节点存活数小于 Zookeeper 集群规模的一半时，应尽快恢复。

|扩容要求	
|NeonSAN 集群容量使用量过高会影响存储系统性能或带来数据超额风险，故当容量使用至 90% 时，必须进行扩容；使用率超 70%，可以提前扩容。

|扩容节点数	
|考虑容量的前提下，保持各故障域节点数和容量相当。

|扩容方案	
|RG 少于 12 节点，新机器或硬盘可扩容到原 RG； +
RG 大于 12 节点，新机器扩容到新建 RG。

|元数据库集群可容忍故障的节点数	
|小于数据库集群的一半，且需要尽快恢复。

|单资源组内可容忍故障的的存储节点数	
|小于卷副本数。

|单资源组内可容忍故障的磁盘数	
|小于卷副本数。

|在线升级	
|业务低峰期进行，遵守升级流程。
|===
