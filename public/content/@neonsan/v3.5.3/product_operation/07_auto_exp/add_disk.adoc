== 注意事项

* 仅对支持热插拔的SSD与服务器有效。
* 新增磁盘需尽量保证**硬盘型号**、**容量**与原存储节点上的保持一致。若无法保证硬盘容量一样，必须保证**硬盘型号相同**。

== 操作步骤

. 将新磁盘插入原集群的存储节点上。

. 对新插入盘的盘头前 20G 进行写零操作。
+
[,shell]
----
dd if=/dev/zero of=/dev/nvme4n1 bs=10M count=2048 oflag=direct
----
+
include::../../../../_components/note.adoc[]
``of`` 用于指定的参数为新插入盘的盘符，需根据实际情况进行修改。
include::../../../../_components/table_footer.adoc[]

. 测试磁盘 fio 性能。
+
[,shell]
----
fio -name=test_10 -filename=/dev/nvme4n1 -bs=4k -direct=1 -ioengine=libaio -group_reporting --eta-newline=1 -runtime=600 -time_based -rw=randwrite -iodepth=32 -numjobs=2
----
* 若 4k 随机写的 fio 测试性能不足，则需要暂停扩容，排查具体原因并解决后，再进行扩容。
* 实际使用过程中，磁盘 4k 随机写的 fio 测试性能至少 200k+的 IOPS。

. 将盘添加到对应节点。
+
[,shell]
----
neonsan add_ssd -dev <dev_name> -store_id <store_ID>
----
+
相关说明：
+
* ``-dev`` 参数用于指定盘符，必须与节点上原有的磁盘命名格式保持一致。
* 该命令执行成功后，该盘插入的对应存储节点的 store.conf 文件中会自动新增该磁盘的信息如下。
+
[,shell]
----
cat /etc/neonsan/store.conf
... ...
//添加新插入盘符相关信息
[ssd.4]
trim_on_reclaim=1
zero_on_reclaim=0
dev="/dev/nvme1n1"
type="SSD"
... ...
----
* store.conf 文件中 ``trim_on_reclaim`` 参数需根据磁盘类型进行决定。若为 iCAS 盘，可手动修改上述配置文件将该参数设置为 0，无需重启 neonstore 服务。
* 该命令格式根据 NeonSAN 版本不同，参数有所不同。可使用 ``neonsan add_ssd -h`` 查看帮助信息。

. 检查卷的状态，SSD 状态，以及 store 节点的状态。
+
[,shell]
----
neonsan list_volume -detail
neonsan list_ssd
neonsan list_store
----

. 执行如下命令对集群内的 SSD 进行负载均衡。
+
[,shell]
----
neonsan rebalance_ssd
----
