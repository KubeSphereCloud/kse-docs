QingStor NeonSAN 分布式存储系统在实际生产环境中，可能存在集群节点服务器所使用的架构不一致的情况，即节点服务器包括 x86 和 ARM 两种架构。然而，由于不同架构所用使用 Installer 安装包不同，故采用常规的 NeonSAN 安装方法会导致环境部署失败。 +
因此，本文针对上述情况，详细介绍了当待部署环境中同时存在 x86 和 ARM 两种架构节点时，NeonSAN 分布式存储系统的安装方法，主要包括安装前的准备工作、安装与验证步骤等。 +
简单来说，针对环境中不同架构的节点服务器，推荐的部署方案是：先将 x86 架构的节点按照 NeonSAN 安装部署指南进行部署完成后，再将 ARM 架构节点以扩容的方式，作为存储节点添加至集群中。

集群各节点的系统要求以及环境部署规划参考link:https://box.yunify.com/s/McLnzcYbDccawAW[《NeonSAN 自动化安装部署指南》]即可。

include::../../../../_components/attention.adoc[]
* 待部署的集群中 x86 架构和 ARM 架构的节点均至少 3 个。
* 两种架构的节点不能属于一个存储池。
