NeonSAN 的网络类型包括：

* 业务网络（前端网络）：QBD 或 iSCSI 等客户端与存储节点间的网络，负责将上层业务的 IO 分发到具体的存储节点上。
* 管理网络：NeonSAN Portal 管理平台或云平台等与 ZooKeeper 节点、管理节点间的网络，通过查询 ZooKeeper 集群找到当前集群主管理节点，将相应的 API 请求分发到主管理节点。
* 存储网络（后端网络）：NeonSAN 存储节点间副本或纠删码数据同步的网络。
* 网关网络（复制网络）：仅 NeonSAN 异步复制功能所需，主站点集群与备站点集群进行数据异步复制的网络。

默认情况下，采用**两网部署**模式。业务网络与存储网络分离。业务网络使用一片双口 10Gb/25Gb 网卡做 bond 配置，管理网络复用业务网络所在网卡。存储网络使用另一片双口 10Gb/25Gb 网卡不做 bond。

若客户有等保或安全需求，可选择**三网部署**部署模式，即每个节点管理网络和业务网络各用一片双口网卡（做 bond），存储网络使用一片双口网卡（不做 bond）。

NeonSAN 在网络部署方面需充分考虑高可用，全冗余设计，即：

* 存储节点配置两个双口网卡，若在全闪环境，则须支持 RDMA。
* 存储集群内部节点为冗余链路，交换机需支持 PFC 和 ECN 功能。
* 存储与外部之间为冗余链路。
