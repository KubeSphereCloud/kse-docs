|英文词汇|中文/缩略词 |解释说明|应用
|Pool   
|卷组     
|用于管理卷，创建卷时需指定卷组。其与存储资源无关联，一个卷组下可以有多个卷，且这些卷可以分布在不同存储池，也可分布于同一个存储池。 
|创建卷时均需要制定某一卷组，否则将会创建卷失败。
示例，创建一个 3 副本大小为10G 的卷，其所处的卷组为 pool1： +
创建卷组：`neonsan create_pool -pool pool1` +
在卷组 pool1 创建卷：`neonsan create_volume -volume vol1 -pool pool1 -size 10G -recount 3`

|volume 
|卷 
|{product_name} 在客户端申请的卷，相当于一个逻辑盘，可使用 QBD 挂载出去供用户使用。
|创建一个 3 副本10G 的精简置备卷：`neonsan create_volume -volume vol1 -pool pool1 -size 10G -recount 3` +
使用 QBD 进行挂载： `qbd -m pool1/vol1`

|shard 
|分片
| {product_name} 客户端创建的卷，按照 64G 划分为若干个Shard；每个 shard 根据指定的副本数，保存为若干个 replica。
|-

|- 
|精简置备卷 
|精简置备卷可预先分配超实际可用的存储空间，以灵活的方式预先规划存储空间，优化存储利用率。
|{product_name} 创建的卷默认均为精简置备卷。

|- 
|厚置备卷 
|全置备卷可立即分配即时可用的存储空间，为关键业务提供实时存储空间满足存储需求。
|创建卷是指定 -think_prov 参数即可。即 `neonsan create_volume -volume vol2 -pool pool1 -size 10G -recount 3 -think_prov`

|replica 
|副本 
|是数据的一份，三副本则表示同样的数据存放了三份；副本分为主副本（primary replica）和从副本（slave replica）
|创建卷时通过-recount 参数指定卷的副本数，支持 1 ~ 3 副本。

|primary replica 
|主副本 
|主副本主要用于数据的读写。 每个卷只有一个主副本。
|可通过执行如下命令，查看指定卷的主副本分布情况：`neonsan list_replica_location pool1/vol1`,返回结果中参数 IS PRIMARY 结果为 true 的副本为主副本。

|salve replica 
|从副本 
|卷的从副本，除了主副本外的其他副本。
|可通过执行如下命令，查看指定卷的主副本分布情况：`neonsan list_replica_location pool1/vol1`,返回结果中参数 IS PRIMARY 结果为 false 的副本为主副本。

|- 
|多副本策略 
|数据保护策略，将数据副本存储在多个不同节点上，满足数据安全冗余的需求。
|-

|repcount
|副本数  
|卷的一个属性，创建后可修改。{product_name} 的卷支持创建 1 ~ 3 个副本。
|副本数在创建卷的过程中，通过 -repcount 参数进行指定。

|最小副本数
|min repcount
|当OK状态的副本数小于该值，则该卷变为只读。若当 OK 状态的副本数为 0，卷的状态为 ERROR，则客户端不能对该卷进行读写操作。
|创建卷时指定 -min_repcount 参数来设定，取值范围为 1 ~ 3，默认值为1。`neonsan create_volume -volume vol2 -pool pool1 -size 10G -recount 3 -min_repcount 2`

|policy 
|卷副本分布策略 
|{product_name} 支持如下副本策略：
* default: 默认策略 

* vol_name_aware:对 Volume 名敏感

* clone:克隆 

* stretch_cluster:拉伸集群卷

* mutex:互斥卷，单副本的 Volume 不分布于同一个存储节点上

* most_balanced:使副本均匀分布于指定存储池内所有存储节点上 
|创建卷时通过指定 -policy 参数，设置当前卷的副本分布策略：`neonsan create_volume -volume vol2 -pool pool1 -size 10G -recount 3 -policy most_balance`

|resource group 
|存储池（rg） 
|为了资源合理利用以及管理的方便性，可将集群的存储节点划分为多个存储池，每个存储池最多支持 24 个存储节点。
目前 {product_name} 支持3 ~ 4096 个存储节点（Store 节点）
|创建存储池：`neonsan create_rg -rg test_rg` +
存储池添加节点：`neonsan add_rg_node -rg test_rg -store_id 1,2,3`

|snapshot 
|快照（Snap） 
|卷的数据在某一个时刻的完整拷贝或镜像，是一种重要的数据容灾手段，当数据丢失时，可通过快照将数据完整的恢复到快照时间点。
|创建快照：`neonsan create_snapshot -snapshot pool/6@snap`

|snapshot_regular
|定时快照 
|设定固定的时间间隔，对集群内的所有卷或指定卷创建快照。
|创建定时快照任务：`neonsan create_snapshot_regular_job -interval 5 -volume pool/vol`

|rollback
|回滚 
|将某一卷的数据恢复到指定状态的行为，需用到快照。
|-

|rollback_snapshot
|快照回滚
|将卷回滚至指定快照
|将卷回滚至指定快照：`neonsan rollback_snapshot -snapshot pool/7@snap`

|label
|标签
|针对某些存储池（rg）的合集，可通过打标签的方式对集群内的存储池进行分类。一个存储池可添加多个标签，且一个标签下也可添加多个存储池。
|给存储池添加：`neonsan add_rg_label -rg test_rg -label test_label`

|===
