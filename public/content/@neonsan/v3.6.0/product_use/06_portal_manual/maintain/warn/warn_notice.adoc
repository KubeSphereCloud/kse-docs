Portal 管理平台的告警通知方式支持如下几种：

* 支持通过 SMTP 发送告警信息至指定接收人。
* 支持通过 SNMP 的方式发送告警至指定服务器。
* 支持用户设置 Webhook 的方式，发送告警至指定 URL 链接。

== SMTP 设置

SMTP 即邮件通知方式。在配置邮件发送告警信息时，需先配置link:../warn_set/#_告警模板[告警模板]。SMTP 配置包括邮件发送服务器信息以及邮件接组相关信息。

. 登录 link:../../../login[Portal 管理界面]。

. 选择左侧导航栏**运维管理** > **告警通知**，进入告警通知设置页面，选择 **SMTP** 页签。
+
image::/images/neonsan/portal_warn_notice_1.png[]
+

include::../../../../../../_components/note.adoc[]
* ``SMTP 发件邮箱设置``仅支持配置一个发件邮箱，设置成功后，可对其进行**修改**或**删除**操作。
* ``告警接收组``需要关联link:../warn_user[告警用户]和link:../warn_set/#_告警模板[告警模板]，该列表中会显示环境中所有可以接收到告警通知的接收组信息，包括``接收组名称``、``成员数量``、``关联告警模板``以及``备注``。

include::../../../../../../_components/table_footer.adoc[]

. 在 ``SMTP 发件邮箱设置``页面部分，点击**配置邮箱**，弹出**配置 SMTP 发件邮箱**设置窗口，输入各项参数后，点击**保存**。
+
image::/images/neonsan/portal_warn_notice_2.png[,60%]
+
**参数说明**
+
[cols="1,2", options="header"]
|===
| 参数 |说明

|邮箱地址
|发件箱地址，仅支持配置一个。设置成功后支持对其进行修改或删除操作

|邮箱服务器
|发件箱服务器

|端口
|发件箱端口

|密码
|发件箱密码
|===

. 配置告警接收组。点击**告警接收组**页面部分的**创建告警接收组**，进入创建告警接收组页面，输入各项参数后，点击**确定**即可。
+
image::/images/neonsan/portal_warn_notice_3.png[]
+
**参数说明**
+
[cols="1,2", options="header"]
|===
| 参数 |说明

|接收组名称
|即为告警信息接收群组，与其他信息接收群组含义相同，群组里可有多个联系人，也可以仅包含一个联系人。群组里所有联系人，均可收到告警信息邮件。

|备注
|非必填项，可填写当前接收组相关信息

|关联告警用户
|选择告警邮件接收人。用户可根据需求选择一个或多个用户添加至该告警群组。若当前页面无可选告警用户，或当前页面列举的告警用户不满足需求，用户可先自行link:../warn_user[创建告警用户]后，再做关联。

|关联告警模板
|选择告警邮件发送模板。用户可根据需求选择一个或多个告警模板。若当前页面无可选告警模板，或当前页面列举的告警模板不满足需求，用户可先自行link:../warn_set/#_告警模板[创建告警模板]后，再做关联。
|===

. 配置后的 SMTP 发件箱信息与告警接收组信息均支持修改与删除。
+
* 点击 ``SMTP 发件邮箱设置``页面部分的**修改**或**删除**按钮，在弹出的窗口中，根据提示信息完成 SMTP 发件邮箱的删除或修改。

* 点击指定``告警接收组``所在行右侧的更多操作，选择**修改**或**删除**，在弹出的窗口中，根据提示信息完成告警接收组的删除或修改。

+
image::/images/neonsan/portal_warn_notice_4.png[]

== SNMP 设置

. 登录 link:../../../login[Portal 管理界面]。

. 选择左侧导航栏**运维管理** > **告警通知**，进入告警通知设置页面，选择 **SNMP** 页签，进入 **SNMP 服务配置**页面后，开启 SNMP 服务配置开关。
+
image::/images/neonsan/portal_warn_notice_5.png[]
+
*参数说明*
+
[cols="1,2", options="header"]
|===
| 参数 |说明

|SNMP 服务主机
|通过 SNMP 方式接收告警的服务器节点 IP 地址

|端口
|通过 SNMP 方式接收告警的端口号

|对象标识符
|Portal 所在节点的 OID

|版本号
|SNMP 服务的版本号，默认为 2c
|===

. SNMP 服务配置完成后，如需修改，重复以上步骤即可。如无需 SNMP 服务，可直接关闭。

== Webhook 设置

Portal 管理平台支持用户配置 Webhook 来接收相应的告警通知。若设置或者删除 Webhook 消息接收端后，该配置默认生效时间为 10 秒。

. 登录 link:../../../login[Portal 管理界面]。

. 选择左侧导航栏**运维管理** > **告警通知**，进入告警通知设置页面，选择 **Webhook** 页签，进入 **Webhook 服务配置**页面后，开启 Webhook 服务配置开关。

. 点击**添加 Webhook 接收端 URL**，填写接收端 URL 信息后，点击**配置**即可。
+
image::/images/neonsan/portal_warn_notice_6.png[]
+

include::../../../../../../_components/note.adoc[]
* ``地址``填写接收端的 URL 地址，需输入以 ``http://`` 或 ``https://`` 开头的 URL。
* 可添加最多 5 个 Webhook 接收端。
* 如需验证该 URL 是否设置成功，用户可在当前存储环境中触发一条告警后，再到设置的 Webhook 消息接收端确认告警消息已经收到即可。

include::../../../../../../_components/table_footer.adoc[]

. 若需修改指定的接收端 URL，可直接在修改其相应的地址和备注信息即可，修改完成后，需点击**配置**按钮，使其生效。

. 点击接收端 URL 的删除图标，即可删除相应的 Webhook 消息接收端。已删除的 Webhook 消息接收端不再收到相应的告警消息。
+
image::/images/neonsan/portal_warn_notice_7.png[]

. 点击 Webhook 服务配置开关右侧的 **Webhook 发送消息示例**，弹出示例窗口。
+
image::/images/neonsan/portal_warn_notice_8.png[,80%]
