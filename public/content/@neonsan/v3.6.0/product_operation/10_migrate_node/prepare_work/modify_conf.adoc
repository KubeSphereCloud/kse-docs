新节点只作为管理节点使用，故需移除其 ``/etc/rc.local`` 文件中关于 intelcas 的相关内容。

. 逐一登录至各个新管理节点。

. 修改 ``/etc/rc.local`` 文件，即删除以下内容。
+
--
[,shell]
----
# if use intelcas, need to wait it ready
which casadm && test -f /etc/intelcas/intelcas.conf
if [ $? -eq 0  ]; then
    count=120
    while [ ${count} -gt 0  ]
    do
        casadm -L | grep -q Active
        if [ $? -eq 0  ]; then
            break
        fi
        sleep 1
        count=$((count-1))
    done
fi
----

修改后的 rc.local 文件主要内容如下。

[,shell]
----
#!/bin/sh
... ...
# supervisord
if [ ! -S /var/run/supervisor.sock ]; then
    # safe supervisord
    ulimit -c unlimited
    ulimit -n 65535
    ulimit -l unlimited
    /usr/bin/supervisord
fi

# QingStor
# Iterate and execute user-defined rc.local files.
for rc_local in $(ls /etc/rc.local.qingstor.* | sort); do
    if [ -x $rc_local ]; then
        eval $rc_local >> /tmp/rc_local.log 2>&1
    fi
done
source /etc/profile

exit 0
----
