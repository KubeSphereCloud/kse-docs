{product_name} 管理服务在线迁移是指在保证用户环境不停机、业务不中断的前提下，将 {product_name} 集群的管理服务从原集群的节点上迁移至独立的物理节点上，其中集群的管理服务主要包括 Zookeeper 服务、 数据库管理服务以及 Portal 服务。

== 读者对象

本文档主要适用于以下工程师：

* 实施工程师

* 运维工程师

* 技术支持工程师等

== 术语解释

[cols="2,2,6a",options="header"]
|===
|英文词汇|中文/缩略词 |解释说明
|QingStor-zero-web
|-
|{product_name} 存储集群 Web 管理平台组件，可部署在集群的三台存储节点上，互为主备，可以提供 GUI 管理配置、监控告警、运维升级。

|Zookeepr
|-
|一个分布式协调服务，其底层提供的主要功能包括管理（存储、读取）用户程序提交的数据以及为用户程序提交数据节点监听服务。

|NeonCenter
|{product_name} 管理组件
|控制层面组件，主 Center 处理所有集群层面的控制请求。

|NeonStore
|{product_name} 数据组件
|数据层面组件，负责处理数据收发、副本间复制、IO 读写以及本地磁盘管理。

|Prometheus
|-
|负责监控存储系统的各项数据，根据告警规则向 Alerter 发送告警信息。

|Alerter
|-
|青云自研的告警信息管理模块，{product_name} 通过 Prometheus 获取监控数据，匹配相应的告警规则后，由 Alerter 生成相应的告警信息，并将告警信息发送给指定用户。

|Monitor
|监控组件
|加载于每个节点的监控组件，负责物理机硬件、服务状态以及相关数据的监控。硬件异常事件通过 Monitor 上报到主 Center，触发对应的故障处理流程。

|-
|混合部署
|NeonSAN 的存储服务和管理服务同时部署于同一个节点上。
|===
