---
title: "NeonSAN v3.5.0 及之后版本操作步骤"
weight: 1
---

. 关闭 NeonSAN 集群的自动恢复与自动均衡功能。
+
[,shell]
----
neonsan set_parameter -parameter auto_recovery -value 0
neonsan set_parameter -parameter auto_balance -value 0
----

. 查看集群中剩余空间。
+
[,shell]
----
neonsan list_store
----
+
include::../../../../../../../_components/note.adoc[]
* 集群剩余空间需大于集群总容量的 20%。
* 除待迁移数据磁盘所在节点之外的其他节点的剩余空间，需大于待迁移数据磁盘已使用空间的两倍。
* 若存在状态不为 OK 的节点，需调查原因后，再继续数据迁移操作。
include::../../../../../../../_components/table_footer.adoc[]

. 检查集群中待迁移数据的磁盘 UUID。
+
[,shell]
----
neonsan list_ssd
----
+
include::../../../../../../../_components/note.adoc[]
若待迁移数据的磁盘状态为 ERROR 且无法恢复，可参考“物理设备运维 > SSD 下线及换盘操作”相关内容将相应磁盘下线。
include::../../../../../../../_components/table_footer.adoc[]

. 执行如下命令，迁移指定磁盘上的数据。
+
[,shell]
----
neonsan evict_ssd -ssd_uuid <ssd_uuid>
----
+
* ``-ssd_uuid`` 指定的为步骤 3 中查询到的待迁移数据的磁盘的 ``UUID``，需根据实际情况进行更改。
* 可通过 ``neonsan list_job`` 查看任务执行状态。
* 任务执行完成后，迁移数据的磁盘状态应为 `OFFLINE`，否则需排查原因，解决问题后再次执行后续步骤。
* 检查集群中卷的状态，所有卷的状态均需为 `OK`。
+
[,shell]
----
neonsan list_volume
----
* 该命令可重复执行，直至数据迁移完成。
* 若数据迁移失败，需根据命令返回的提示信息进行相应的处理，或联系 NeonSAN 研发小组，分析失败原因，解决问题。

. 开启 NeonSAN 集群的自动恢复与自动均衡功能。
+
[,shell]
----
neonsan set_parameter -parameter auto_recovery -value 1
neonsan set_parameter -parameter auto_balance -value 1
----
