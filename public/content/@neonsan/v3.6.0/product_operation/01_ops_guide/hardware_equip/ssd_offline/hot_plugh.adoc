---
title: "SSD 支持热插拔"
weight: 1
---

以下操作仅对支持热插拔的 SSD 与服务器有效。

. 参考前文内容，link:../../move_disk/[移除待下线的磁盘]。
+
include::../../../../../../_components/note.adoc[]
移除磁盘的过程中，**仅需迁移待下线磁盘数据**即可。
include::../../../../../../_components/table_footer.adoc[]
. 对待下线盘进行下电操作。
.. 查看系统中磁盘信息。
+
[,shell]
----
ls -l /sys/block/
----
+
image:/images/neonsan/ops_guide_19.png[]

.. 确认待下线磁盘的 ``bus device function``。
+
[,shell]
----
lspci | grep -i non
----
+
image:/images/neonsan/ops_guide_20.png[]

.. 根据待下线磁盘的 ``bus device function``，确认待下线磁盘的槽位
+
[,shell]
----
lspci -vvv -s 84:00.0
----
+
image:/images/neonsan/ops_guide_21.png[]

.. 依次执行如下命令，对待下线磁盘进行下电操作。
+
--
[,shell]
----
cat /sys/bus/pci/slots/17/power
----
回显示例。
[,shell]
----
1
----
进行下电操作。
[,shell]
----
echo 0 > /sys/bus/pci/slots/17/power
----
再次检查，返回结果为 ``0``，则表示下电成功。
[,shell]
----
cat /sys/bus/pci/slots/17/power
----

--

. 待下线磁盘信息从配置文件中删除。
+
[,shell]
----
vi /etc/neonsan/store.conf

... ...
# 待下线盘符相关信息
[ssd.4]
trim_on_reclaim=1
zero_on_reclaim=0
dev="/dev/nvme1n1"
type="SSD"
... ...
----

. 拔盘。
+
* 若该磁盘不再使用且无需更换新的磁盘，可直接执行步骤 9 及后续操作，**完成 SSD 下线**操作。
* 若需更换新的磁盘则需执行步骤 5 及后续操作，完成 SSD 换盘操作。

. 插入新盘。
. 对新插入盘的盘头前 20G 进行写零操作。
+
[,shell]
----
dd if=/dev/zero of=/dev/nvme4n1 bs=10M count=2048 oflag=direct
----
+
* of 指定的参数为新插入盘的盘符，需根据实际情况进行修改。
* 写零操作之前，确认一下新插入盘的序列号，与步骤 4 中的序列号对比，避免盘符输入错误，导致对其他盘进行了写零操作。

. 将新盘添加到对应节点。
+
--
* NeonSAN v3.0.0 及以后版本。
+
[,shell]
----
neonsan add_ssd -dev <dev_name> -store_id <storeid>
----

* NeonSAN v3.0.0 之前版本。
+
[,shell]
----
neonsan add_ssd -dev <dev_name> -store_ip <storeIp> 
----

include::../../../../../../_components/note.adoc[]
``-dev`` 参数指定的 dev_name 需要与原有的磁盘盘符格式保持一致。
include::../../../../../../_components/table_footer.adoc[]
--

. 查看配置文件。
+
[,shell]
----
vi /etc/neonsan/store.conf

... ...
# 添加新插入盘符相关信息

[ssd.4]
trim_on_reclaim=1
zero_on_reclaim=0
dev="${dev_name}"
type="SSD"
... ...
----
+
include::../../../../../../_components/note.adoc[]
* NeonSAN v3.0.0 及之后版本，``add_ssd`` 命令执行成功后，配置文件中会自动添加新插入磁盘的信息，无需手动添加。
* ``trim_on_reclaim`` 参数需根据磁盘类型来指定，若为 iCAS 盘，该参数的值需为 0。
include::../../../../../../_components/table_footer.adoc[]


. 确认卷的状态，SSD 状态，以及 Store 的状态。
+
[,shell]
----
neonsan list_volume -detail
neonsan list_ssd
neonsan list_store
----
+
include::../../../../../../_components/note.adoc[]
若磁盘信息中存在无效磁盘记录，可参考。link:../../../fault_process/disk/clear_disk_record[清理磁盘记录]进行处理。
include::../../../../../../_components/table_footer.adoc[]

. 执行如下命令对集群内的 SSD 进行负载均衡。
+
[,shell]
----
neonsan rebalance_ssd
----
+
include::../../../../../../_components/note.adoc[]
该命令为异步执行，可以通过 ``neonsan list_job`` 查看进程。
include::../../../../../../_components/table_footer.adoc[]
