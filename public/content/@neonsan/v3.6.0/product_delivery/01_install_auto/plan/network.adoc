NeonSAN 的网络类型包括：

* 业务网络（前端网络）：QBD 或 iSCSI 等客户端与存储节点间的网络，负责将上层业务的 IO 分发到具体的存储节点上。
* 管理网络：NeonSAN Portal 管理平台或云平台等与 ZooKeeper 节点、管理节点间的网络，通过查询 ZooKeeper 集群找到当前集群主管理节点，将相应的 API 请求分发到主管理节点。
* 存储网络（后端网络）：NeonSAN 存储节点间副本或纠删码数据同步的网络。
* 网关网络（复制网络）：NeonSAN 异步复制功能，主站点集群与备站点集群进行数据异步复制的网络。

默认硬件部署方式，业务网络与存储网络分离，分别使用一片双口 10Gb/25Gb 网卡做 bond 配置，管理网络复用业务网络所在网卡，存储网络使用另一片网卡。NeonSAN 在网络部署方面需充分考虑高可用，全冗余设计，即：

* 存储节点配置两个双口网卡，且支持 RDMA。
* 存储集群内部节点为冗余链路，交换机需支持 PFC 和 ECN 功能。
* 存储与外部之间为冗余链路。
