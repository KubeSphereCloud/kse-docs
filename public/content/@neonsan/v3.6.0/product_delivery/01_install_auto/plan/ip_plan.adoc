* 按网络类型规划
+
NeonSAN 默认是两网融合部署模式，即每个节点管理网络和业务网络共用一片双口网卡（做 bond）、存储网络单独使用一片双口网卡（不做bond），每个节点需要规划 3 个 IP。QingCloud Installer 自动化部署工具默认支持该模式配置。 +
若采用三网分离的部署模式，即每个节点管理网络和业务网络各用一片双口网卡（做 bond），存储网络使用一片双口网卡（不做 bond），每个节点需要规划 4 个 IP。QingCloud Installer v2.0.18 及以后版本支持配置该模式。 
+

* 按照服务组件规划
+
NeonSAN 默认 Common 服务（包括 ZooKeeper、MySQL 以及 Portal 等）、NeonCenter 和 NeonStore 服务组件的 data_port 共用一片双口网卡（做 bond），NeonStore 服务组件的 repl_port 使用一片双口网卡（不做 bond），每个节点需要规划 3 个 IP。 +
各个组件也可以采用独立的网卡和 IP，需要保证组件所用的 IP 其 2 层或 3 层能 ping 通可通信。采用 QingCloud installer 自动化部署暂不支持这种方式，如有需要可采用手动部署方式，每个节点需要的 IP 数量依据需要独立组件数量确定。
