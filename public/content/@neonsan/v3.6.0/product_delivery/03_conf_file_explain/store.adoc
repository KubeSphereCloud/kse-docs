**注意事项**：

* ZooKeeper 集群信息可查询如下配置内容，需换成当前环境对应的 zookeeper 目录。
+
[source,shell]
----
cat /opt/zookeeper/conf/zoo.cfg
----


* 同步灾备的情况下，ZooKeeper 集群需剔除仲裁节点。例如：由 a，b，c 组成的一个ZooKeeper集群。其中，c 是仲裁站点，那么，ZooKeeper 集群应设置如下。
+
[source,conf]
----
[zookeeper]
ip="a:2181,b:2181"
cluster_name="cluster1"
----

* NeonSAN 系统默认配置的 disk 为 SSD 类型，如果环境中配置了 cas 盘，即混插模式，配置应做如下修改。
+
[source,conf]
----
type="CachedHDD"
objectsize=4
----

* 磁盘清盘，如果磁盘支持 trim，可首选 trim 模式，即配置如下。
+
[source,conf]
----
zero_on_reclaim=0
trim_on_reclaim=1
----
* 磁盘不支持 trim 模式，则选用清零配置。
+
[source,conf]
----
zero_on_reclaim=1
trim_on_reclaim=0
----
* 由于磁盘清零耗时较长，混插环境，建议私有云如下配置。
+
[source,conf]
----
zero_on_reclaim=0
trim_on_reclaim=0
----
* 关于绑核部分内容，可参考link:https://box.yunify.com/s/jmd7i7dPccZkp3k[《NeonSAN 产品性能测试指南》]。

* 异步灾备环境必须设置如下参数，且值为 60；同步灾备环境无需配置。
+
[source,conf]
----
# metro client timeout
[metro]
timeout=60
----
* 混插模式下，需设置 ssd_io_timeout 为 60，rep_io_timeout 为 65，conn_close_wait_time 的值为 rep_io_timeout 的 10 倍，即 650。
+
[source,conf]
----
ssd_io_timeout=60 
conn_close_wait_time=650
rep_io_timeout=65
----

* 在网络硬盘场景下，为解决网络频繁闪断导致磁盘不可用的问题，可通过以下参数，设置当磁盘无法访问且无 IO 时重试访问磁盘的超时时间，单位秒，需根据实际环境配置进行修改。在非网络硬盘场景下，无需设置该参数。
+
[,conf]
----
... ....
[store]
ssd_access_timeout=10
... ...
----
