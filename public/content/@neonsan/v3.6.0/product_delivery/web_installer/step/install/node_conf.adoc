完成集群信息配置后，点击**下一步**，进入**节点配置**阶段。节点配置主要包括**收集节点信息**和**校验与配置节点**两步。

image::/images/neonsan/web_installer_node_conf_1.png[,90%]

## 收集节点

. 在**节点配置**界面，点击**开始收集**，进入**收集节点**界面。
+
image::/images/neonsan/web_installer_node_conf_2.png[,60%]

. 部署人员根据页面提示，填写相应参数。参数说明如下。
+
[options="header",cols="1,2"]
|====
|参数|说明
|登录节点
a|登录各物理服务器的用户名或密码，用于登录各节点服务器。

* 若在节点操作系统配置过程中，使用的青云默认用户名密码，此处选择``默认用户名密码``，后台将使用 yop/zhu1241jie 来访问各物理服务器。
* 若在节点操作系统配置过程中，部署人员曾自定义过用户名密码，则此处选择``自定义用户名密码``，并根据实际情况填写用户名和密码即可。

|IP 列表
a|填写所有物理节点的 IP 地址。SANC 融合部署场景下，NeonSAN 管理虚拟机所在的宿主机的节点 IP 也需填写在列。

* 一行一个 IP 地址或 IP 区间，使用回车换行。至少包含 3 个节点。
* 支持填写 IP 区间，如 192.168.100.2-3，表示包括 192.168.100.2 和 192.168.100.3 两个节点 IP。
* 支持同时填写独立 IP 地址和 IP 区间。
|====

. 点击**开始收集**，等待节点信息收集完成即可。
+
image::/images/neonsan/web_installer_node_conf_3.png[,60%]
+
include::../../../../../../_components/note.adoc[]
若节点收集失败可返回重新收集。
include::../../../../../../_components/table_footer.adoc[]

## 校验并配置节点

. 节点收集完成之后，点击**校验并配置节点**。

. 进入**节点配置**详情页面，**节点库**列表内罗列了所有物理节点信息。点击节点所在行内的的**详情**，查看相应的详细信息是否收集正确。
+
include::../../../../../../_components/attention.adoc[]
* 重点检查各节点的**硬盘信息**是否准确。
* 若检查某一节点收集的信息有误，可进入其**详情**，点击**重新收集信息**。
* 若收集到预期外的节点信息，可点击指定节点所在行的**删除**，将其从节点库中剔除。
include::../../../../../../_components/table_footer.adoc[]

+
image::/images/neonsan/web_installer_node_conf_4.png[,95%]

. 选择管理节点。
+
* **SANC 独立部署**
+
在**节点库**中，勾选待作为**管理节点**的节点，并点击**批量设为管理节点**。所有选定的节点，将显示在**管理节点**列表内，且在节点库中，相应节点将被标注**管理节点**。
+
image::/images/neonsan/web_installer_node_conf_5.png[,95%]

* **SANC 融合部署**
+
.. 在**节点库**中，将指定物理节点（云平台的 KS 节点）**设定为管理节点**，在**管理节点**区域，点击**配置管理虚机**。
+
image::/images/neonsan/web_installer_node_conf_6.png[,95%]

.. 此处必须配置 3 个管理虚机。若企业云平台只有 2 个 KS 节点，则需新增虚机，点击某一**管理节点**上的更多操作，选择**增加虚机**。根据实际规划，配置管理虚机的相关信息。
+
include::../../../../../../_components/note.adoc[]
管理虚机需分布在不同的 KS 节点上。若仅有 2 个 KS 节点，则其中一个 KS 节点上拉起 2 给管理虚机，另一个 KS 节点上拉起 1 个管理虚机。
include::../../../../../../_components/table_footer.adoc[]
+
image::/images/neonsan/web_installer_node_conf_7.png[,95%]
+
**参数说明**：
+
** ``配置虚机 IP``：管理虚机的 IP 地址，需要与管理网络在同一网段中，未被使用的 IP 地址。
** ``Gateway``：部署企业云平台时，规划并配置好的 gateway 网关地址，需在云平台端进行查看。
** ``Hostname``：当前管理虚机的 Hostname，根据实际情况进行填写。

. 点击**下一步**，进入下一阶段。
