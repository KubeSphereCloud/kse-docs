安装示例组件

更新时间：2023-07-24 07:10:31

本节以安装 DevOps 为例，演示如何在 KubeSphere 企业版 Web 控制台安装组件。有关组件使用的更多信息，请参阅扩展组件使用指南。

DevOps 组件是专为 Kubernetes 中的 CI/CD 工作流设计的，它提供了一站式的解决方案，帮助开发和运维团队用非常简单的方式构建、测试和发布应用到 Kubernetes。

前提条件

您需要已经安装好 KubeSphere Core，并可以登录到 KubeSphere 企业版 Web 控制台。

安装准备

  1. 点击 Web 控制台左上角的扩展市场，进入扩展市场页面。

  2. 搜索查找 DevOps，点击订阅。

  3. 如果您未登录 KubeSphere 云账号，请先登录，然后进入订阅流程。支付完成后，点击对话框中的立即安装，进入组件安装流程。

安装步骤

在单集群上安装 DevOps

  1. 在组件安装对话框的安装检测页签，选择 DevOps 的版本号，并安装好所有必装组件，点击下一步。

    说明

    安装检测时，会识别 DevOps 的依赖组件中哪些是必装组件，哪些是选装组件。

    • 若必装组件的状态为未就绪，您需要先行安装正确版本的必装组件，以确保 DevOps 的正常使用。而选装组件不会影响 DevOps 的安装。

    • 若扩展组件不存在依赖组件，则不会显示必装组件和选装组件。

  2. 在扩展组件安装页签，修改 DevOps 的配置后（可选），点击开始安装即开始安装 DevOps。

    安装过程中，可在扩展市场的 DevOps 组件详情页查看安装状态。安装完成后，点击顶部导航栏上的扩展组件图标，可以使用该组件，并添加到快捷访问。

在多集群上安装 DevOps

若当前为多集群环境，按照上述步骤完成组件安装后，还需要为指定的主集群和成员集群配置 Agent。

  1. 在集群选择页签，根据名称，标识，标签选择集群。

  2. 在差异化设置页签，分别编辑选中集群的 YAML 配置，也可不修改，使用初始默认配置。点击确定，开始配置集群 Agent。
